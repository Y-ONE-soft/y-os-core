# 134. feat: 마스터 소속 그룹 지정 및 프로젝트 생성 그룹 기본값

## 작업 요약

- **마스터가 내 정보에서 소속 그룹을 직접 지정**할 수 있게 했다 (스탭은 지금처럼 관리자 관리, 읽기 전용).
- **프로젝트 생성의 그룹 드롭다운 기본값을 내 소속 그룹으로** 채웠다. 마스터는 다른 그룹도 고를 수 있고, 스탭은 자기 그룹 하나로만 만들어진다(변화 없음).

## 배경

기존에는 마스터가 프로젝트를 만들 때 그룹 드롭다운이 빈 "그룹 선택"으로 시작해 매번 골라야 했고, 자기 소속 그룹이라는 개념이 없었다. 소속 그룹은 스탭·마스터 모두 관리자 관리(읽기 전용)였다.

바뀐 흐름:
- 마스터가 내 정보에서 자기 소속 그룹을 한 번 정한다.
- 프로젝트 생성은 그 그룹을 드롭다운 기본값으로 쓴다 — 마스터는 필요하면 다른 그룹으로 바꿔 만든다.
- "소속 없음"은 허용하지 않는다 — 항상 실제 그룹 하나를 갖는다.

## 변경 내용

### 마스터 그룹 지정 (내 정보)

**`src/types/auth.ts`** — `ProfilePatch`에 `groupId` 추가.

**`src/app/api/auth/me/route.ts`** — `groupId`는 **`current.role === "MASTER"`일 때만** 반영한다. 스탭이 보내면 조용히 무시. 빈 값은 400(소속 없음 금지), 존재하지 않는 그룹도 400(FK 위반 500 대신 의미 있는 에러). `db.projectGroup.findUnique`로 실재를 확인한다.

**`src/server/auth/service.ts`** — `updateProfile`가 `groupId` 패치를 그대로 반영한다(마스터 여부는 라우트가 이미 걸렀다). 주석을 갱신.

**`src/components/features/me/my-info-page.tsx`**
- 마스터는 프로필 폼에 **소속 그룹 `<select>`** 를 둔다 — 빈 옵션 없이 항상 한 그룹을 고른다. `groups`(프로젝트 스토어, 마스터는 전체 그룹)에서 채운다
- 좌측 계정 요약 카드의 소속 그룹 읽기 전용 행은 **스탭에게만** 보인다 — 마스터는 폼에서 고르므로 중복 표시하지 않는다
- 안내 문구도 역할별로: 마스터 "아이디·역할은 관리자가 관리합니다.", 스탭 "아이디·역할·소속은 관리자가 관리합니다."
- 저장 시 바뀐 그룹만 patch에 담고(`setUser`로 세션 갱신), 마스터가 아니거나 빈 값이면 보내지 않는다

### 프로젝트 생성 그룹 기본값

**`src/components/features/projects/project-create-dialog.tsx`**
- 그룹 상태 초기값을 `""` → **`user?.groupId ?? ""`** 로. 마스터 드롭다운이 열릴 때 자기 소속 그룹이 이미 잡혀 있다
- 드롭다운은 그대로 — 마스터는 다른 그룹도 선택 가능
- 빈 "그룹 선택" 옵션은 **내 그룹이 스토어에 아직 없을 때만** 자리 표시로 남기고, 평소엔 실제 그룹들만 보인다
- 서버(`api/admin/projects/route.ts`)는 변경 없음 — 마스터는 body.groupId, 스탭은 세션 그룹을 그대로 쓴다

| 시나리오 | 결과 |
| --- | --- |
| 마스터 `groupId: g-lab` 변경 | 200, groupId=g-lab ✓ |
| 없는 그룹 `g-nope` | **400** ✓ |
| 빈 그룹 `"  "` | **400** ✓ (소속 없음 금지) |
| 스탭이 `groupId` 전송 | **무시** — 그룹 g-soft 그대로, 400(바꿀 값 없음) ✓ |
