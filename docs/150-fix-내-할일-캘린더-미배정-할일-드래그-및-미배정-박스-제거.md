# fix: 내 할일 캘린더 미배정 할일 드래그 및 미배정 박스 제거

## 증상

내 할일 캘린더에서 **프로젝트 없는(미배정) 할일**의 칩을 다른 날짜로 드래그해도 예정일이 바뀌지 않았다. 또 미배정 할일을 프로젝트처럼 박스(회색 배경)로 묶어 그렸는데, 그럴 필요가 없었다.

## 원인

캘린더 칩을 끌어 옮길 때 예정일을 계산하는 `nextTaskSchedule`이 **`myProjects`(프로젝트 단계 소속 할일)만 순회**하고 미배정 목록은 보지 않았다. 그래서 미배정 칩을 드래그하면 `null`을 반환해 아무 일도 일어나지 않았다(백로그→캘린더 HTML5 드롭에는 미배정 분기가 있었지만, 캘린더 안에서 칩을 옮기는 경로가 빠져 있었다).

미배정 박스는 `UNASSIGNED_BOX`라는 가짜 프로젝트 키로 칩들을 묶고 `meta`에 "미배정" 이름의 프로젝트로 등록해, 렌더러가 회색 박스를 그리던 것이다.

## 수정

### `my-work-calendar-panel.tsx` — 드래그 버그

`nextTaskSchedule`에 미배정 순회를 추가하고, 반환을 명시적 union으로 만들어 "단계가 있으면 프로젝트도 있고, 미배정이면 둘 다 `null`"이라는 상관관계를 유지했다.

`handleDrag`의 할일 분기에 미배정(단계 없음) 처리를 넣었다 — 단계 커버(stretch) 로직을 건너뛰고 예정일만 `updateTask(null, null, ...)`로 옮긴다.

### `my-work-calendar-source.ts` — 박스 제거

미배정 칩은 그대로 만들되(`UNASSIGNED_BOX` 키로 배치돼 위치만 잡힘), `meta[UNASSIGNED_BOX]` 등록을 없앴다.

### `my-work-calendar.tsx` — 렌더러

`meta`에 없는 묶음의 박스는 그리지 않는다.

```tsx
const project = projects[box.project];
if (!project) return null; // 미배정 등 프로젝트 없는 묶음은 박스(배경) 없이 칩만
```

## 최신 main 재적용

작업 중 캘린더 대개편(일/주/월 뷰, 인라인 추가 등)이 main에 머지돼 이 파일들이 크게 바뀌었다. 확인 결과 그 개편에는 **미배정 드래그·박스 처리가 들어 있지 않아**(nextTaskSchedule은 여전히 `myProjects`만, source는 여전히 `meta[UNASSIGNED_BOX]` 등록) 이 수정이 유효했다. 리베이스 충돌을 푸는 대신 최신 main에서 같은 변경을 다시 얹고, 그 위에서 재검증했다.

## 후속

- `ProjectBoxItem`에 남아 있는 `UNASSIGNED_BOX` 분기(`isProject`로 링크 대신 span)는 이제 미배정 박스가 아예 안 그려지므로 도달하지 않는 dead code다. 이번엔 그 개편 코드를 최소한으로만 건드리려 남겨 뒀다.
- docs 번호는 작성 시점 main 최대(149) + 1로 150을 썼다.
