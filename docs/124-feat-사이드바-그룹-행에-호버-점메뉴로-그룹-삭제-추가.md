## 1. 작업 내용 요약

마스터 사이드바의 **그룹 헤더 행**에 호버 시 나타나는 ⋮ 버튼을 달아, 프로젝트·단계·할일과 같은 방식으로 그룹을 삭제할 수 있게 했다.

기존 그룹 행은 **우클릭 컨텍스트 메뉴**(`그룹 삭제`)만 있었다 — 발견하기 어렵다. 프로젝트 행은 이미 호버 ⋮(`RowActions`)를 갖고 있는데 그룹만 없어서, 같은 진입점을 그룹에도 맞췄다. 우클릭 메뉴는 그대로 둔다.

그룹 목록은 원래 마스터에게만 렌더되므로(사이드바 `isMaster && groups.map`) 이 ⋮ 삭제도 자연히 마스터 전용이다.

## 3. 설계 결정

**호버 대상을 그룹 헤더로 한정했다.** 그룹 `<li>`는 헤더 + 하위 프로젝트 목록을 함께 담는다. `<li>` 전체를 `group`으로 잡으면 하위 프로젝트에 마우스를 올려도 그룹 ⋮가 떠 버린다. 그래서 헤더만 `<div className="group relative flex items-center">`로 감싸고, 프로젝트 `<ul>`은 그 밖 형제로 두었다. 프로젝트 행이 쓰는 `group`(각자의 `<li>`)과는 별개 서브트리라 서로 간섭하지 않는다.

**개수 배지는 호버 시 사라져 ⋮ 자리를 내준다.** 그룹 행 우측에는 프로젝트 개수(`group.projects.length`)가 있다. 프로젝트 행은 개수가 없어 ⋮를 그냥 얹으면 됐지만, 그룹은 개수와 ⋮가 같은 자리라 겹친다. 개수 span에 `group-hover:opacity-0`을 줘서 호버하면 개수가 사라지고 ⋮가 나타나는 **교체**가 되게 했다.

**⋮는 그룹 헤더 버튼 밖(형제)에 둔다.** 헤더 버튼은 접기/펴기 토글이라, ⋮를 그 안에 넣으면 버튼 중첩이자 누를 때 토글이 실행된다. `RowActions`는 이미 클릭·드래그·포인터다운을 stopPropagation 하므로 형제로 두면 토글과 분리된다(프로젝트 행과 동일 패턴).

## 4. 검증

`npm run lint` ✓ · `npx tsc --noEmit` ✓ · `npm run build` ✓

**브라우저 실검증** — dev 서버(워크트리 전용 포트 3061, 점유 프로세스 확인). 마스터·스탭을 브라우저 컨텍스트로 분리해 대조하고, 임시 그룹으로 삭제 왕복을 확인했다.

| # | 검증 | 결과 |
|---|---|---|
| 1 | 그룹 ⋮ 버튼 존재 | 그룹마다 1개, 기본 `opacity:0` |
| 2 | 호버 대상 한정 | 헤더 래퍼(`div.group.relative`)가 하위 프로젝트 `<li>`를 포함하지 않음 |
| 3 | 호버 시 표시 (CDP `:hover` 강제) | **⋮ opacity 1 / 개수 opacity 0** — 교체 확인 |
| 4 | ⋮ 클릭 → 메뉴 | 열림, 항목 `그룹 삭제` |
| 5 | 그룹 삭제 실행 | `DELETE /api/admin/groups/{id}` → 200, 낙관 제거 + 서버에서 사라짐 확인 |
| 6 | 스탭 사이드바 | 그룹 ⋮ 없음(그룹은 마스터 전용, 프로젝트 ⋮만 노출) |

임시 그룹은 스크립트가 만들고 삭제했다.

## 5. 알려진 사항

- 그룹에 프로젝트가 들어 있어도 삭제가 막히지 않는다 — 서버 `deleteGroup`이 그대로 지운다(하위 프로젝트도 함께 정리됨). 우클릭 메뉴의 기존 동작과 같다.
- 삭제 확인 다이얼로그는 없다 — 프로젝트·단계·할일 삭제와 동일하게 즉시 실행이다.
