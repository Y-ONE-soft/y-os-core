# feat: 내 할일 캘린더 주 행 높이를 키워 빈 클릭 영역 확보

## 배경 / 목적

"캘린더에서 날짜를 누르면 할일을 추가"하려면, 각 날짜 칸에 **눌러서 새 할일을 놓을 빈 공간**이 있어야 한다. 지금은 주 행 최소 높이가 `104px`로 빠듯해, 프로젝트 박스가 칸을 채우면 클릭할 빈 자리가 거의 남지 않는다. 다음 태스크(날짜 클릭 → 할일 추가)의 토대로, **빈 공간이 어느 정도 보이도록 달력을 키운다.**

이 커밋은 높이(기하)만 다룬다. 클릭 → 추가 인터랙션은 다음 태스크에서 얹는다.

## 변경 내용

파일: `src/components/features/my-work/my-work-calendar.tsx`

1. **주 행 최소 높이 상향** — `WEEK_MIN_PX` `104` → `140`.
   - `140 = 날짜숫자(24) + 약 4레인(28×4=112) 여유` 수준. 박스가 없는 날도 넉넉한 클릭 영역을 갖는다.
2. **콘텐츠 아래 여유 레인 추가** — 새 상수 `WEEK_PAD_LANES = 1`.
   - 주 최소 높이 계산을 `DATE_ROW_PX + laneCount * LANE_PX + 8` → `DATE_ROW_PX + (laneCount + WEEK_PAD_LANES) * LANE_PX + 8`로 변경.
   - 프로젝트 박스가 칸을 꽉 채운 주에도, 콘텐츠 아래로 **레인 1단(28px)**만큼 "프로젝트 없음" 할일을 놓을 빈 공간이 항상 남는다.

## 값 결정 이유

- **왜 140인가** — 기존 104는 "날짜 숫자 + 할일 두 줄 남짓"으로 잡힌 값이라 빈 클릭 여백이 없었다. 4레인 여유(140)면 빈 달에서도 클릭 영역이 눈에 띈다. 과하게 키우면 6주가 세로로 너무 길어지므로, 페이지 스크롤(#127에서 도입)이 받아 주는 선에서 절제했다.
- **왜 콘텐츠 기준 +1레인인가** — `WEEK_MIN_PX`만 올리면 콘텐츠가 많은 주(박스가 여러 겹)에서는 여전히 빈 자리가 없다. 콘텐츠 높이에 비례해 항상 1레인을 더 얹어야, 바쁜 주에도 "프로젝트 없음" 영역이 보장된다. `LANE_PX`(28)와 묶여 움직이므로 별도 px가 아니라 레인 수로 표현했다.
- 레인/박스 기하(`LANE_PX`·`BOX_PAD_PX`·`BOX_GAP_PX`)는 손대지 않았다 — 박스 겹침 규칙에 영향을 주지 않기 위해 높이만 늘렸다.
