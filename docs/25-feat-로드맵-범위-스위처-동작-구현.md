# 25. 로드맵 범위 스위처 동작 구현

- **예정 커밋 메시지**: `feat: 로드맵 범위 스위처 동작 구현`
- **작업일**: 2026-07-22
- **작업 브랜치**: `로드맵-범위-스위처` (워크트리 `.claude/worktrees/roadmap-range`, base: main `f8a89d5`)

---

## 1. 작업 내용 요약

작업 현황 페이지 로드맵의 **오늘 / 일 / 주 / 개월 / 분기** 스위처를 시각 자리표시에서 **실제 동작**으로 구현했다. 그동안 로드맵 창이 `ROADMAP` 상수(2026-07-15 시작 4주)로 고정돼 있어 스위처를 눌러도 아무 변화가 없었고, 시드 날짜가 지나면 화면이 무의미해지는 문제도 함께 해소했다.

- **일**: 7일 창 (일 단위 눈금, `7/22`)
- **주**: 4주 창 (주 단위 눈금) — 기존 기본값과 동일한 배치
- **개월**: 3개월 창 (월 눈금, `7월`)
- **분기**: 4분기 창 (분기 눈금, `26 Q3`)
- **오늘**: 현재 기간으로 복귀 (이미 오늘 창이면 비활성)
- 창 이동을 위한 **‹ ›(이전/다음 기간) 버튼** 추가 — 없으면 "오늘"이 동작할 여지가 없어 함께 도입

## 2. 주요 구현 결정

| 항목 | 결정 | 이유 |
|---|---|---|
| 창 계산 모듈 분리 | `roadmap-window.ts` — `buildRoadmapWindow(range, today, page)` | 순수 함수로 분리해 컴포넌트는 렌더만. 프로젝트 상세 로드맵도 나중에 같은 모듈로 전환 가능 |
| 기준일 | **오늘**(실시간) 기준, 상수 제거 | 고정 창은 날짜가 지나면 무의미. `useState(todayISO)` 지연 초기화 — 이 컴포넌트는 세션 로딩 후 클라이언트에서만 첫 렌더되므로 하이드레이션 불일치 없음 |
| 창 배치 규칙 | 각 범위 모두 "직전 1단위 + 이후"로 잡아 **오늘이 두 번째 칸**에 오게 함 | 기존 주간 뷰 배치(7/15 시작·오늘 7/22)와 동일한 감각 유지 |
| 눈금 렌더 | grid → **절대 위치(%)** | 월·분기는 칸 길이가 달라(28~31일, 90~92일) 균등 grid로 표현 불가 |
| 짧은 막대 | `minWidth: 26px` | 분기 뷰에서 2~3일 단계가 1px로 사라지는 것 방지 |
| 범위 전환 시 page | 0으로 초기화 | 단위가 바뀌면 이전 페이지 위치가 의미를 잃음 |
| 프로젝트 상세 로드맵 | 이번 범위에서 제외 (여전히 고정 창) | 사용자 요청이 작업 현황 페이지 한정 — 후속 태스크로 남김 |
| 변수명 | 창 객체를 `view`로 (`window` 아님) | 전역 `window` 섀도잉 방지 |
