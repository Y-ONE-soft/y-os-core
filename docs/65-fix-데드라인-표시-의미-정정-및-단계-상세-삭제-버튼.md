- **예정 커밋 메시지**: `fix: 데드라인 표시 의미 정정 및 단계 상세 삭제 버튼 추가`

- **작업 브랜치**: `데드라인-의미-정정-단계삭제` (워크트리 `.claude/worktrees/deadline-semantics`, base: main `3a2c1c1`)

## 1. 작업 내용 요약

사용자 요청 3건을 처리했다.

1. **데드라인 표시 의미 정정 (핵심 수정)** — "이건 **캘린더에 마감일 라벨을 붙이냐 마냐**지, 로드맵에 나오냐 안 나오냐가 아니다"
2. 데드라인 토글을 더 보기 좋게 재배치
3. 단계 상세에 **단계 삭제 버튼** 추가 (작업 상세와 동일 위치)

## 2. 데드라인 표시 의미 정정

`showDeadline`이 원래 의도(캘린더 마감 라벨)와 다르게 **로드맵·보드의 기간 표시까지 좌우**하고 있었다. 그 결과 데드라인을 끄면 단계 막대가 통째로 사라져, 기간이 있는 단계인데도 로드맵에서 보이지 않았다.

| 위치 | 이전 | 이번 |
|---|---|---|
| 작업 현황 로드맵 | `showDeadline && startDate`일 때만 막대 | **`startDate`만 있으면 항상 막대** |
| 프로젝트 단계 로드맵 | 〃 | 〃 |
| 보드 컬럼 헤더 기간 | `showDeadline`일 때만 `07/20~07/24` 표기 | **기간이 있으면 항상 표기** |
| 내 작업 캘린더 마감 라벨 | `showDeadline`에 따라 표시 | **변경 없음 — 이것이 유일한 용도** |

- 캘린더 쪽(`my-work-calendar-source.ts`)은 이미 올바르게 쓰고 있어 손대지 않았다.
- 즉 이번 수정으로 `showDeadline`의 영향 범위가 **캘린더 마감 라벨 하나로 좁혀졌다.**

## 3. 데드라인 토글 재배치

직전 작업(docs/59)에서 헤더 우측(작업 상세의 이슈 등록 자리)에 뒀는데, 의미가 "캘린더 마감 라벨"로 정정되면서 **기간(시작일·종료일) 바로 아래**가 맥락상 맞는 자리가 됐다.

| | docs/59 | 이번 |
|---|---|---|
| 위치 | 헤더 우측 버튼 | **세부 사항 · 종료일 바로 아래** |
| 형태 | `☑ 데드라인 표시` 텍스트 버튼 | **카드형 체크박스** — 체크 사각형 + 제목 + 설명 2줄 |
| 설명 | "로드맵·보드에 마감 표시" (틀린 설명) | **"캘린더에 이 단계의 마감일 라벨을 붙입니다"** |
| 켜짐 표시 | 버튼 색 강조 | 체크 채움 + `border-primary bg-primary/5` 카드 강조 |

- 단계 **추가** 오버레이도 동일한 카드형으로 통일(생성·수정 경험 일치).
- 기간 설정 바로 아래에 있어 "이 기간의 마감을 캘린더에 표시할지" 인과가 자연스럽다.

## 4. 단계 삭제 버튼

- 병렬 세션이 머지한 `deleteStage`(서버·API·스토어, docs/57)를 그대로 사용.
- 작업 상세의 `작업 삭제`와 **같은 위치**(사이드 패널 하단, `variant="destructive"` 전체 폭) — 두 상세 화면이 완전히 대칭이 됐다.
- 삭제 시 오버레이를 닫는다. 단계 안의 작업은 서버 규칙대로 **삭제되지 않고 백로그로 이동**한다.

| 시나리오 | 결과 |
|---|---|

| 상세 사이드 순서 | 데드라인 카드 → 공동 작업자 → 단계 삭제 ✓ (헤더에는 데드라인 없음) |
| 데드라인 문구 | "캘린더에 이 단계의 마감일 라벨을 붙입니다" ✓ |
| 토글 저장 | DB `showDeadline: true → false` 확인 ✓ |
| 단계 삭제 | 오버레이 닫힘 + DB에서 단계 제거 확인 ✓ |

## 7. 알려진 사항

- 데드라인을 켠 단계는 **종료일 당일** 캘린더에 마감 라벨이 붙는다(`my-work-calendar-source.ts`의 기존 규칙 그대로).
- 이번 정정으로 로드맵·보드는 `showDeadline`과 무관해졌으므로, 기간을 숨기고 싶으면 기간 자체를 비우면 된다.
