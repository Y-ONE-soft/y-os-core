# feat: 내 할일 일 뷰를 시간대 그리드로 전환

## 작업 요약

내 할일 캘린더의 **일 뷰**를 하루짜리 칸에서 **0~23시 시간대 그리드**로 바꿨다. 예정 시각(`scheduledTime`)이 있는 할일은 그 시각 행에, 없는 할일은 맨 위 "시각 미정" 밴드에 놓인다. 할일을 시간대 사이로 끌어 시각을 지정하거나 밴드로 끌어 해제할 수 있다.

이로써 사용자 요청 두 가지가 함께 해결된다.

1. **"일 뷰에서 오늘만"** — 여러 날에 걸친 단계 막대는 시간대 그리드에 그리지 않는다. 그 날 예정된 할일만 보인다.
2. **"시간 뷰로"** — 시각 축이 생겨 하루 일정을 시간대로 본다.

## 주·월 뷰는 그대로

일 뷰만 새 렌더러(`MyWorkDayHours`)로 분기하고, 주·월 뷰는 기존 칸 그리드(`MyWorkCalendar`)를 그대로 쓴다.

```tsx
{view === "day" ? <MyWorkDayHours ... /> : <MyWorkCalendar ... />}
```

시간대 그리드는 세로축이 시간이라 칸 그리드와 렌더 모델이 근본적으로 다르다. 억지로 한 렌더러에 합치기보다 뷰별로 갈랐다.

## 지나가는 단계 막대가 사라지는 원리

시간대 그리드는 **할일만** 그린다. 데이터를 소스 오버레이에서 뽑되 `kind === "task"`만 취한다.

```ts
source.overlays.filter((overlay) => overlay.kind === "task")
```

단계 막대(`kind === "stage"`)는 애초에 목록에 들어오지 않으므로, "오늘을 지나가는 단계 막대"가 구조적으로 나타날 수 없다. 이것이 요청 1을 만족시키는 방식이다. (마감 라벨 오표시는 앞 커밋 docs/170에서 이미 별도로 고쳤다.)

## 데이터 — 오버레이 재사용 + 소속·시각 join

"그 날 예정된 내 할일"이라는 필터는 `buildCalendarSource`(일 그리드)가 이미 적용한다(담당자·프로젝트 범위·그 날짜). 이 결과(`kind:"task"` 오버레이)를 그대로 쓰고, 시간대 그리드에 필요한 **예정 시각·소속(project/stage)** 만 보드에서 이어 붙였다.

```ts
// taskId → {projectId, stageId, scheduledTime}
const meta = new Map(...);  // previewBoards 단계·백로그 + unassigned
source.overlays.filter(kind==="task").map(o => ({ id, name, color, done, late, ...meta.get(o.taskId) }))
```

필터 로직을 다시 구현하지 않아 **캘린더와 표시 대상이 항상 일치**한다(같은 소스). `projectId/stageId`가 필요한 이유는 시각을 저장할 때 `updateTask(projectId, stageId, taskId, {scheduledTime})`에 그 위치가 필요하기 때문이다.

## 상호작용

- **시각 지정** — 할일을 다른 시간대 행으로 끌어 놓으면 그 시각(`HH:00`)으로 저장된다. HTML5 드래그(`task-drag`의 공용 헬퍼)를 써 캘린더의 드래그 규약과 통일했다.
- **시각 해제** — "시각 미정" 밴드로 끌어 놓으면 `scheduledTime`이 `null`이 된다.
- **완료 토글 / 상세 열기** — 칩의 체크박스는 그 자리에서 완료를, 이름을 누르면 상세를 연다(캘린더 칩과 같은 규약).
- **드롭 하이라이트** — 끌고 있는 행에 배경을 깔아 어디 놓이는지 보인다.
- **현재 시각 줄** — 오늘이면 붉은 선으로 지금 시각을 표시한다.

시각은 **시(hour) 단위로 스냅**한다(분은 안 받는다). 하루 일정을 훑는 용도라 시 단위면 충분하고, 분까지 드래그로 맞추긴 어렵다. 필요하면 상세 오버레이에 시각 입력을 붙이는 것이 자연스럽다(범위 밖).

## 안전 처리 두 가지

- **현재 시각 줄의 setState** — 첫 값을 효과 본문에서 동기로 넣으면 `react-hooks/set-state-in-effect`에 걸린다(태스크 1의 그 규칙). `requestAnimationFrame`으로 한 프레임 미뤄 렌더 중 동기 setState를 피했다. 오늘이 아니면 같은 방식으로 줄을 지운다.
- **시각 없는 할일** — `scheduledTime`이 없거나 형식이 어긋나면(`hourOf`가 `null`) "시각 미정" 밴드로 보낸다. 잘못된 값이 시간대 행을 깨지 않는다.

### 미검증 — 실제 화면·드래그

**핵심 상호작용은 눈으로 못 봤다.** 데이터 행이 클라이언트 스토어에서 그려져 SSR HTML에 없고(그래서 렌더 스모크로 갈음), "일" 탭 전환·드래그로 시각 지정·현재 시각 줄 위치는 브라우저에서 확인해야 한다. 확인이 필요한 것:

1. "일" 탭에서 시간대 그리드가 뜨고 오늘 예정 할일이 제 시각 행에 있는가
2. 할일을 다른 시각 행으로 끌면 그 시각으로 옮겨지는가 / "시각 미정" 밴드로 끌면 해제되는가
3. 여러 날에 걸친 단계 막대가 **안 보이는가**
4. 현재 시각 줄이 지금 시각에 있는가

## 알려진 이슈

- **시각은 시(hour) 단위 스냅.** 분은 지원하지 않는다. 드래그로 정밀 시각을 맞추기 어렵고, 필요하면 상세 오버레이에 시각 입력을 붙이는 편이 낫다.
- **새 할일을 시간대 그리드에서 바로 만들 수 없다.** 칸 그리드처럼 빈 칸을 눌러 추가하는 흐름은 시간대 그리드엔 아직 없다. 백로그·다른 뷰에서 만든 뒤 끌어 시각을 잡는다. 다음 개선 후보.
- **PATCH 시각 미검증**(docs/171의 알려진 이슈)은 그대로다. 이 UI는 항상 `HH:00`만 보내므로 실사용엔 영향 없다.
