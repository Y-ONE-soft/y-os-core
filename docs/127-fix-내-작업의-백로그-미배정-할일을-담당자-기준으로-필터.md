# 127. fix: 내 작업의 백로그·미배정 할일을 담당자 기준으로 필터

## 증상

내 작업 화면의 백로그에 **내가 만들지 않은 할일**이 나왔다. 남이 만든 백로그·미배정 할일이 그대로 섞여 보였다.

## 원인

워크스페이스 API는 **모든 사용자의 할일을 다 내려주고** 클라이언트가 거른다. 그런데 내 작업 화면의 백로그·미배정은 담당자 필터가 없었다.

- **백로그 aside**(`my-work-backlog.tsx`) — 필터가 아예 없이 `전 프로젝트 백로그 + 전체 미배정`을 그렸다
- **캘린더**(`my-work-calendar-source.ts`) — 프로젝트 소유(`ownerId === 나`)로 단계는 걸렀지만, 그 프로젝트 백로그와 미배정 할일은 담당자와 무관하게 다 그렸다
- **타임라인**(`my-work-timeline-panel.tsx`) — 같은 문제

실제 개발 DB에서 확인한 사례: step01의 내 작업 백로그에 **master가 만든 미배정 할일 2건**이 떠 있었다.

## 판정 기준

`Task.assigneeId`로 판정한다. 할일 생성 시 서버가 만든 사람을 기본 담당자로 넣으므로(`assigneeId: owner ?? createdById`), 내가 만든 할일은 내 `assigneeId`를 갖는다. 사용자 지시 "백로그도 결국에는 작업자가 나라고 생각해서 나오면 될듯"과 맞는다.

**두 기준이 성격에 맞게 갈린다.**

- **단계 막대·단계 안 할일** — 프로젝트 단위라 **프로젝트 소유**(`ownerId === 나`)로 판정. 그대로 둔다
- **백로그·미배정 할일** — 낱개 작업이라 **담당자**(`assigneeId === 나`)로 판정. 이번에 추가

## 변경 내용

### `src/components/features/my-work/my-work-scope.ts` (신규)

세 화면이 같은 판정을 쓰도록 공통 헬퍼를 뒀다.

```ts
export function isMyTask(task: BoardTask, userId: string | undefined) {
  return userId != null && task.assigneeId === userId;
}
```

### `src/components/features/my-work/my-work-backlog.tsx`

`useSession`으로 사용자를 받아, 미배정 + 전 프로젝트 백로그를 합친 목록을 `isMyTask`로 거른다.

### `src/components/features/my-work/my-work-calendar-source.ts`

`buildCalendarSource`에 `viewerId` 인자를 추가. 백로그 할일과 미배정 할일 루프에서 `isMyTask`로 거른다. **단계 막대와 단계 안 할일 칩은 건드리지 않았다** — 그건 프로젝트 소유 기준이라 대상이 다르다.

### `src/components/features/my-work/my-work-calendar-panel.tsx`

`buildCalendarSource`에 `user?.id`를 넘긴다.

### `src/components/features/my-work/my-work-timeline-panel.tsx`

프로젝트 백로그를 `isMyTask`로 거른 뒤 행을 만든다.

### `src/components/features/projects/board-store.tsx`

`addUnassignedTask(name, assigneeId?)` — 낙관적 항목에 담당자를 미리 박는다. 서버도 같은 기본값(`createdById`)을 넣지만, 낙관적 항목에 없으면 **방금 만든 할일이 담당자 필터에 걸려 서버 응답 전까지 깜빡인다.** 백로그 aside가 생성 시 `user?.id`를 넘긴다.

### 실데이터로 필터 결과 확인

개발 서버에서 워크스페이스 응답의 `assigneeId` 분포를 확인하고, 필터를 적용한 결과를 사람별로 셌다.

당시 데이터:
- 백로그 1건 — 담당자 step01
- 미배정 2건 — 담당자 master

필터 적용 결과:

| 사용자 | 내 작업 백로그 | 비고 |
| --- | --- | --- |
| step01 | 1건 | 전에는 master 미배정 2건까지 **3건** |
| master01 | 2건 | step01 백로그 1건 제외 |

**교차 격리가 확인됐다.** step01에게는 master가 만든 미배정이 더 이상 보이지 않고, master에게는 step01 백로그가 보이지 않는다.

**미검증** — 실제 화면 렌더와 방금 만든 할일의 깜빡임 여부는 프리뷰에서 확인해야 한다. 캘린더는 클라이언트 렌더라 SSR로는 안 보이고, 브라우저 자동화 도구가 없다.

## 바꾸지 않은 것

- **단계 막대·단계 안 할일 칩** — 프로젝트 소유 기준 그대로. 사용자 지시가 백로그·미배정에 한정됐고, 단계는 프로젝트 단위라 성격이 다르다
- 요청 알림·필터 바(`my-work-filter-store`) — 별개 기능
