# fix: 캘린더 막대를 프로젝트·단계 순서로 정렬

## 요청

> 캘린더 뷰 막대바 순서를 **프로젝트 우선, 그리고 단계 우선**으로 정렬해줘. 지금 단계 섞여보여서 이상함.

## 문제

- 프로젝트는 이미 박스 1개로 묶여 있어 "프로젝트 우선"은 성립한다.
- 문제는 **박스 안 단계 막대의 세로 순서.** 기존 `assignStageLanes`는 단계를 **시작일 순(같으면 긴 것 우선) 그리디**로 레인에 배정했다. 그래서 **번호가 늦은 단계가 먼저 시작하면 위 줄로 올라와**, 위에서부터 1·2·3… 이 아니라 순서가 섞여 보였다.
  - 예: 단계1(1/10~15), 단계2(1/1~20) → 시작일 순이면 단계2가 위(줄0), 단계1이 아래(줄1)로 뒤집힘.

## 해결

`src/components/features/my-work/my-work-calendar-layout.ts`의 `assignStageLanes`를 **단계 번호(stageNo) 순 + 겹침 기준 레인 배정**으로 바꿨다.

- 단계별 전역 구간을 모을 때 **stageNo도 함께** 보관.
- 프로젝트별 단계 목록을 `stageNo` 오름차순으로 정렬.
- 번호 순으로 훑으며 **기간이 겹치지 않는 첫 레인**에 앉힌다:
  - 앞 번호 단계와 겹치면 아래 줄로 내려가 **번호 순서가 유지**된다(위=작은 번호).
  - 서로 안 겹치는 단계는 같은 줄을 나눠 써(연속 단계는 한 줄에 1·2·3 왼→오) **박스가 불필요하게 두꺼워지지 않는다**.
- 기존 `laneEnd`(마지막 찬 지점) 방식은 시작일 순 정렬을 전제로 해서, 번호 순으로 바꾸면 안 맞아 실제 구간 겹침(`placed.start < stage.end && stage.start < placed.end`) 검사로 교체.

## 변경 파일

- `src/components/features/my-work/my-work-calendar-layout.ts` — `assignStageLanes` 정렬 기준을 시작일 → 단계 번호로, 레인 배정을 겹침 검사 방식으로 교체. (레인 수 계산·최소 1줄 규칙은 그대로.)

## 영향 / 범위

- 단계 막대의 **세로 순서만** 바뀐다. 박스 위치·색·드래그·클릭 동작은 그대로.
- 연속(비겹침) 단계는 여전히 한 줄에 압축되고, 겹치는 단계만 번호 순으로 줄을 나눈다.
- 미배정 칩 배치·프로젝트 박스 세로 순서는 이 커밋에서 건드리지 않는다.

## 검증

- `npm run lint` 통과 / `npx tsc --noEmit` 통과(exit 0).
- 순수 배치 로직 변경(레인 배정 규칙)이라 타입·린트로 검증. 겹치는 단계가 번호 순으로 쌓이는지의 시각 확인은 머지 후 프로덕션/사용자 확인(태스크 ②와 함께).
