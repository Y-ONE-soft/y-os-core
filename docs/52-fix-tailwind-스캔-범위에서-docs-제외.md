# fix: Tailwind 스캔 범위에서 docs 제외

## 작업 요약

Tailwind v4가 `docs/`의 작업 결과 문서까지 스캔해, **소스에서 지운 클래스가 문서 언급만으로 CSS 번들에 계속 남는** 문제를 고쳤다. `@source not "../../docs"` 한 줄로 문서를 스캔 대상에서 제외했다.

## 어떻게 발견했나

docs/44(막대 이동 커서를 `move`로 교체) 작업 직후, 프로덕션에 변경이 반영됐는지 확인하려고 배포된 CSS 번들에서 커서 클래스를 뽑아봤다.

```
.cursor-ew-resize
.cursor-grab        ← 소스에서 지웠는데 남아 있다
.cursor-grabbing    ← 위와 같음
.cursor-move
```

소스(`git grep cursor-grab -- src/`)에는 **0건**인데 번들에는 있었다. 처음에는 배포가 오래된 것으로 의심해 프로덕션 별칭이 가리키는 배포, 응답 헤더의 `X-Vercel-Cache`, 캐시 우회 재요청까지 확인했으나 모두 최신이었다.

결정적 확인은 **로컬 빌드 산출물 비교**였다. 같은 커밋을 로컬에서 빌드한 `.next/static/chunks/*.css`에도 `.cursor-grab`이 똑같이 들어 있었다. 배포 문제가 아니라 **빌드가 원래 그렇게 만들고 있었던 것**이다.

원인은 `docs/42`와 `docs/44`가 산문에서 클래스명을 그대로 언급한 것이었다.

> `grab`/`grabbing`은 "물건을 집어 든다"는 은유다 … `cursor-grab touch-none select-none active:cursor-grabbing`

Tailwind는 마크다운을 코드로 구분하지 않는다. 파일 안에서 클래스처럼 보이는 문자열을 찾을 뿐이라, 문서에 적힌 클래스명이 그대로 유틸리티 생성 근거가 된다.

## 왜 자동 스캔이 docs까지 갔나

[`src/app/globals.css`](../src/app/globals.css)는 `@import "tailwindcss";`만 있고 `@source` 지정이 없었다. Tailwind v4는 이 경우 **프로젝트 루트를 기준으로 소스를 자동 탐지**한다(`.gitignore` 대상은 제외). `docs/`는 gitignore 대상이 아니므로 스캔에 포함된다.

이 저장소는 **태스크마다 작업 결과 문서를 남기는 규칙**이 있고, 그 문서에 Tailwind 클래스명이 자주 등장한다. 즉 문서를 쓸수록 죽은 CSS가 쌓이는 구조였다.

## 해결 방식 선택

| 방식 | 판단 |
| --- | --- |
| `@source not "../../docs"` | **채택.** 자동 탐지는 그대로 두고 docs만 뺀다. 영향 범위가 좁아 회귀 위험이 가장 낮다 |
| `@import "tailwindcss" source(none);` + `@source "../"` | 제외. 자동 탐지를 끄고 화이트리스트로 전환하는 방식이라 정확하지만, 나중에 `src/` 밖에 스캔이 필요한 파일이 생기면 조용히 클래스가 누락된다 |
| `docs/`를 `.gitignore`에 추가 | 논외. 문서는 저장소에 남아야 한다 |

경로는 CSS 파일 기준 상대 경로다. `src/app/globals.css` → `../../docs`.

`@source not`은 Tailwind 4.1+ 기능이며 이 저장소는 **4.3.3**이라 지원 범위 안이다.

## 변경 파일

### `src/app/globals.css`

```diff
  @import "tailwindcss";
  @import "tw-animate-css";
  @import "shadcn/tailwind.css";
+
+ /* Tailwind v4는 @source 지정이 없으면 프로젝트 루트를 자동 스캔한다.
+    docs/의 작업 결과 문서는 산문에 클래스명을 그대로 적기 때문에, 소스에서
+    지운 클래스가 문서 언급만으로 계속 CSS에 남는다(실제 사례: cursor-grab).
+    문서는 스캔 대상에서 제외한다. */
+ @source not "../../docs";
```

발견 경위를 주석으로 남겼다. 이유를 모르면 "왜 굳이 제외했지" 하고 되돌리기 쉬운 종류의 설정이다.

## 알려진 이슈 · 한계

- **이번에 제거된 10개는 이미 쌓여 있던 분량이다.** 문서가 늘수록 재발했을 문제이므로, 이번 조치는 누적을 막는 쪽에 의미가 있다. 768 B 자체는 크지 않다.
- **`docs/` 외의 마크다운은 여전히 스캔된다.** `README.md`·`CLAUDE.md`·`AGENTS.md` 등이 루트에 있고, 이들도 클래스명을 적으면 같은 일이 생긴다. 현재는 해당 사례가 없어 범위를 `docs/`로만 좁혔다. 필요해지면 `@source not`을 추가하면 된다.
- **이 문제 때문에 배포 검증에 혼선이 있었다.** "소스에 없는 클래스가 번들에 있다"는 신호를 배포 지연으로 오인해 별칭·캐시·배포 상태를 한참 확인했다. 앞으로 같은 상황에서는 **로컬 빌드 산출물과 먼저 비교**하는 것이 빠르다.
