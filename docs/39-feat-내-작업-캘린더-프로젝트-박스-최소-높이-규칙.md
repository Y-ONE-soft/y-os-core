# 39. feat: 내 작업 캘린더 프로젝트 박스 최소 높이 규칙

## 작업 요약

내 작업 페이지(`/projects/my-tasks`) 월 캘린더의 프로젝트 박스에 **높이 하한 규칙**을 넣었다.

- **단계 줄은 단계 유무와 무관하게 항상 확보** — 할일이 단계 자리로 올라오지 않는다
- **박스 최소 2줄** — 단계 막대 두 개 두께
- 할일이 2줄 이상이면 그만큼 두꺼워진다

캘린더 전체 높이가 커지는 것은 의도된 결과다(W4 84px → 136px).

## 배경

docs/32에서 프로젝트 박스가 단계와 할일을 함께 감싸도록 만들었지만 두 가지가 남아 있었다.

1. **단계 없는 프로젝트의 박스에 단계 줄이 없었다.** W5의 YOS는 할일만 있어 `stageLanes = 0`이 되고, 할일이 박스 맨 위(단계 자리)에 붙었다. 프로젝트는 항상 단계를 포함하는 단위이므로 자리가 비더라도 줄은 잡혀 있어야 한다.
2. **박스가 1줄까지 얇아졌다.** 할일 없는 프로젝트는 박스가 단계 막대 하나 두께였다.

## 배치 규칙 (변경분)

`src/components/features/my-work/my-work-calendar-layout.ts`

```
1. 프로젝트 하나가 박스 하나 — 단계와 할일을 모두 감싼다
2. 단계 줄은 단계 유무와 무관하게 항상 1줄 확보한다      ← 신규
3. 단계 막대는 그 한 줄로만 표현한다 (겹쳐도 나누지 않음)
4. 할일 칩은 단계 줄 아래에 붙고, 서로 겹칠 때만 줄을 나눈다
5. 박스는 최소 2줄 — 단계 막대 두 개 두께               ← 신규
6. 서로 다른 프로젝트는 열 범위가 겹칠 때만 위아래로 갈라진다
```

박스 높이 계산:

```ts
const STAGE_LANES = 1;    // 단계가 없어도 자리를 잡는다
const MIN_BOX_LANES = 2;  // 단계 막대 두 개 두께

const height = Math.max(MIN_BOX_LANES, STAGE_LANES + taskLanes.length);
```

할일이 1줄이면 최소치(2줄) 안에 들어가고, 2줄부터 박스가 두꺼워진다. 단계 줄 아래에 빈 줄이 남지 않는다.

## 변경 파일

### `src/components/features/my-work/my-work-calendar-layout.ts`

- 규칙 주석 갱신 (4개 → 6개 항목)
- `STAGE_LANES`·`MIN_BOX_LANES` 상수 추가
- `stageLanes = stages.length > 0 ? 1 : 0` **제거** — 단계 줄은 항상 1
- 박스 높이가 `Math.max(MIN_BOX_LANES, STAGE_LANES + 할일 줄 수)`
- 할일 레인 계산이 `STAGE_LANES`를 기준으로 내려간다
- `ProjectBox.stageLanes` → **`ProjectBox.taskLanes`** — 항상 1인 값 대신 구분선 렌더에 실제로 필요한 할일 줄 수를 넘긴다

### `src/components/features/my-work/my-work-calendar.tsx`

- 박스 안 구분선을 `box.lanes - 1`개가 아니라 **`box.taskLanes`개**만 그린다. 할일 없는 박스(2줄)에 빈 구분선이 남던 문제를 막는다

## 결과

| 주차 | 전 | 후 | 구성 |
| --- | --- | --- | --- |
| W3 (7/12~18) | 1레인 58px | **2레인 84px** | CMS 박스 2줄 (할일 없음) |
| W4 (7/19~25) | 2레인 84px | **4레인 136px** | CMS 2줄 + YOS 2줄 |
| W5 (7/26~8/1) | 2레인 84px | 2레인 84px | YOS 2줄 / YOC 2줄 (열이 안 겹쳐 나란히) |

프로젝트마다 최소 2줄을 보장하므로 W4가 눈에 띄게 높아진다. 캘린더가 높아지더라도 이 규칙으로 간다는 것이 사용자 결정이다.

W5의 YOS 박스는 단계가 없지만 단계 줄을 비운 채 확보하고, 할일 `PRD`·`설계서 검토`가 그 아래 줄에 놓인다.

```bash
npx prisma generate       # 새 워크트리라 생성물이 없어 선행 필요

npm run dev -- -p 3009    # 워크트리 포트 분리
```

dev 서버 렌더 HTML로 배치를 확인했다 (`master01` 로그인 후 `/projects/my-tasks`).

- **주차 높이** — W3 `84px`, W4 `136px`, W5 `84px`
- **박스 높이** — 5개 전부 `height:56px`(= 2 × 26 + 4). 최소 2줄 규칙이 모든 박스에 적용됨
- **박스 위치** — W4 CMS `top:21px`(레인 0), YOS `top:73px`(레인 2) — 2줄짜리 블록이 연속 레인에 앉음
- **단계 한 줄** — W4 CMS 단계 2건 모두 `top:24px; height:22px`
- **단계 줄 항상 확보** — W5 YOS 박스는 단계가 없는데도 할일이 `top:50px`(박스 두 번째 줄)에 놓임. 단계 자리로 올라오지 않음
- **구분선** — W5 YOS·YOC 박스에만 `top:27px` 1개씩. 할일 없는 W3·W4 박스에는 없음

## 알려진 이슈 / 후속

- 막대 **클릭 동작은 여전히 없다.** 크기·배치까지가 이번 범위이고, 단계 상세 연결은 후속 과제.
- 단계가 3개 이상 겹치면 중간 막대의 노출 폭이 좁아질 수 있다(docs/32와 동일). 자리표시 데이터에는 해당 사례 없음.
- 캘린더가 세로로 길어지면서 스크롤 영역 처리(월 뷰 전체 높이 대 컨테이너)는 실데이터 연결 후 재평가 대상.
