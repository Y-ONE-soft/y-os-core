# feat: 단계 없음을 보드 컬럼으로 분리

요청 사이클 `단계없음-보드-컬럼`의 단일 태스크.

## 배경

"프로젝트에서 '단계 없음'을 우측 백로그랑 같이 두는 게 아니라, '단계 추가'의 보드맵으로 따로 빼줘" 요청.

프로젝트 상세 우측 `ProjectBacklog` 패널은 `stageId=null` 할일을 예정일 유무로 두 섹션으로 갈라 보여줬다.

- **단계 없음**: `stageId=null` + 예정일 있음 (캘린더에 뜨는 할일)
- **백로그**: `stageId=null` + 예정일 없음 (미정)

"단계 없음"을 우측 패널에서 빼서, 보드 영역(단계 컬럼들이 나열되는 곳)에 별도 컬럼으로 옮긴다.

## 결정 사항 (사용자 승인)

| 항목 | 값 |
|---|---|
| 컬럼 위치 | **단계들 다음, '단계 추가' 앞** |
| 우측 백로그 패널 | **'백로그'만 남김** (단계 없음 섹션 제거) |
| 카드 스타일 | **단계 컬럼과 동일** |
| 빈 상태 | 항상 표시 (빈 컬럼 + 안내) — '단계 추가' 앞 고정 자리라 있다 없다 하면 레이아웃이 흔들린다 |

## 변경 내용

### 1. `board-store.tsx` — `moveToStageless` 액션

"단계 없음" 컬럼에 할일을 놓을 때 쓴다. 단계에서 빼되(`stageId=null`) 예정일은 남긴다 — 이게 백로그와 "단계 없음"을 가르는 기준이다.

```ts
moveToStageless(projectId, taskId) {
  const board = cache.getSnapshot().boards[projectId];
  const task = board?.stages.flatMap((s) => s.tasks).find((t) => t.id === taskId)
    ?? board?.backlog.find((t) => t.id === taskId);
  const keep = task?.scheduledDate ?? todayISO();
  boardActions.assignTask(projectId, taskId, projectId, null, keep);
}
```

기존 예정일이 있으면 유지(단계에서 끌어온 경우), 없으면(백로그에서 끌어온 경우) 오늘로 잡는다. `assignTask`가 `scheduledDateOverride`를 받으므로 그 경로를 재사용한다.

### 2. `project-board.tsx`

**카드 컴포넌트 추출** — 단계 컬럼의 인라인 할일 카드를 `BoardTaskCard`로 뽑아 단계 컬럼과 "단계 없음" 컬럼이 공유하게 했다(사용자 결정 "단계 컬럼과 동일"). `stageId`를 prop으로 받아 단계 없음은 `null`을 넘긴다 — `toggleTask`가 이미 `stageId=null`(백로그)을 처리한다.

**"단계 없음" 컬럼** — 단계들 뒤·'단계 추가' 앞에 추가. `backlog` 중 예정일 있는 할일(`stageless`)을 담는다.

- 진짜 단계가 아니므로 **순서 변경·단계 삭제는 없다.** 할일 드래그만 받는다(`isTaskDrag`).
- 드롭하면 `moveToStageless` 호출 — 예정일 유지.
- 헤더는 muted 색(단계 색 점이 아니라 회색 점 + "단계 없음").
- 비어 있으면 안내 문구를 보여 자리를 유지한다.

**할일 추가 입력은 두지 않았다.** 단계 컬럼과 달리 새 할일에 예정일을 어떻게 줄지가 정의되지 않는다(단계는 시작일이 있지만 "단계 없음"은 없다). 새 할일은 백로그·단계에서 만들고 드래그로 옮기거나, 캘린더에서 예정일을 잡으면 여기 나타난다.

### 3. `project-backlog.tsx`

우측 패널에서 **"단계 없음" 섹션을 제거**하고 "백로그"만 남겼다. `stageless` 계산·`BacklogRow` 호출을 지웠다. 패널 드롭 동작(할일을 놓으면 예정일 해제 → 백로그)은 그대로다.

## 검증

`npm run build` 성공(타입 검사 포함), `npm run lint` 경고 0.

dev 서버(포트 3161)로 실제 화면을 확인했다. 픽스처: 프로젝트 + 예정일 있는 백로그 할일("단계없음 할일") + 예정일 없는 백로그 할일("진짜 백로그 할일").

스크린샷과 DOM 판정으로 확인:

```
보드 '단계 없음' 컬럼: 있음 | 컬럼 안 '단계없음 할일' 카드: 있음
우측 패널 '단계 없음' 섹션: 없음 (제거됨)
```

- **보드**: `프로젝트 생성`(단계) → **`단계 없음`(카운트 1, "단계없음 할일" 카드)** → `단계 추가`(점선) 순서. 요청한 위치.
- **우측 패널**: "백로그" 섹션만, "진짜 백로그 할일"(백로그 배지)이 그대로 표시.

검증용 프로젝트는 삭제했다.

## 알려진 이슈 / 주의점

### '단계 없음' 컬럼엔 할일 추가 입력이 없다

단계 컬럼과 미세하게 다르다. 예정일 지정 기준이 없어 새 할일을 여기서 바로 만들지 않는다(위 설명). 필요하면 "추가 시 오늘 예정일" 규칙으로 입력을 붙일 수 있다.

### 컬럼은 항상 보인다

예정일 있는 백로그 할일이 하나도 없어도 빈 컬럼 + 안내 문구로 자리를 유지한다. '단계 추가' 앞 고정 자리라 숨기면 레이아웃이 흔들린다. 빈 컬럼이 거슬리면 접기/숨기기를 별도로 검토할 수 있다.

### 드래그로 예정일이 붙는다

백로그(예정일 없음) 할일을 '단계 없음' 컬럼으로 끌면 오늘 예정일이 붙어 캘린더에 뜨게 된다. 이게 "단계 없음"의 정의(예정일 있음)라 의도된 동작이지만, 사용자가 "그냥 단계만 뺐다"고 여기면 예정일이 생긴 걸 모를 수 있다. 반대로 우측 백로그 패널로 끌면 예정일이 해제된다.

### 단계 없음 카드에 예정일 배지가 없다

"단계 컬럼과 동일" 스타일이라 예정일을 카드에 표시하지 않는다. 구 백로그의 "단계 없음" 행에는 날짜 배지가 있었다. 컬럼 전체가 "단계 없음"이라 구분은 되지만, 카드별 예정일은 상세를 열어야 보인다. 필요하면 카드에 작은 날짜 배지를 더할 수 있다.
