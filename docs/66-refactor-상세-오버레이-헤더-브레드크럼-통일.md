- **예정 커밋 메시지**: `refactor: 상세 오버레이 헤더 브레드크럼 통일`

- **작업 브랜치**: `상세-헤더-브레드크럼-통일` (워크트리 `.claude/worktrees/overlay-breadcrumb`, base: main `2cac16b`)

---

## 1. 작업 내용 요약

두 상세 오버레이의 헤더 경로 표기를 **계층 브레드크럼으로 통일**했다 (사용자 요청).

| 화면 | 이전 | 이번 |
|---|---|---|
| 단계 상세 | `Y.OS core · 로드맵` | **`프로젝트 › 단계`** |
| 할 일(작업) 상세 | `Y.OS core · 백로그` | **`프로젝트 › 단계 › 할 일`** |

### "이상한 `Y.OS core · 로드맵`"의 정체

사용자가 지적한 그 표기는 **단계 상세가 단계명 대신 `로드맵`이라는 고정 문자열을 하드코딩**하고 있어서 나온 것이다. 어떤 단계를 열어도 항상 "로드맵"이 찍혔다. 단계명으로 교체하면서 해소됐다.

작업 상세도 `프로젝트 · 단계`까지만 있고 **정작 보고 있는 작업 이름이 빠져** 있었다. 이제 마지막 항목으로 들어간다.

## 2. 구현

- **`overlay-breadcrumb.tsx` 신규** — 두 오버레이가 함께 쓰는 표시 컴포넌트.
  - 구분자 `›`, 항목 사이 일정 간격
  - **마지막 항목(현재 보고 있는 대상)은 진하게**(`font-medium text-foreground`) — 경로 중 어디에 있는지 즉시 인지
  - 각 항목 `truncate` + `title` 속성 — 이름이 길어도 헤더가 밀리지 않고, 마우스를 올리면 전체 이름 확인
  - `nav[aria-label="위치"]`로 감싸 스크린리더에도 경로임을 알린다
- 두 화면 모두 좌측 식별 칩(`단계 N` / `작업 N`)은 그대로 유지 — 칩은 순번, 브레드크럼은 경로로 역할이 다르다.

## 5. 알려진 사항

- 작업이 백로그에 있으면 단계 자리에 `백로그`가, 프로젝트 미배정이면 `미배정`이 표시된다(기존 `stageLabel`/`projectLabel` 규칙 그대로).
- 브레드크럼 항목은 아직 **클릭 이동이 되지 않는다** — 상위(프로젝트·단계)로 바로 이동하는 링크는 필요해지면 후속 작업으로.
