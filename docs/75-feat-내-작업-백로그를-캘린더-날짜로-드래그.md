# 75. 내 작업 백로그를 캘린더 날짜로 드래그

- **예정 커밋 메시지**: `feat: 내 작업 백로그를 캘린더 날짜로 드래그`
- **작업일**: 2026-07-22
- **작업 브랜치**: `내작업-캘린더-드롭` (워크트리 `.claude/worktrees/mywork-calendar-drop`, base: main `1fd4a65`)
- **요청 사이클 태스크 2/2** — 태스크 1(docs/74) 미배정 묶음 위에 사용자 조작을 얹는다.

---

## 1. 작업 요약

내 작업 페이지의 **백로그 항목을 캘린더 날짜 칸으로 끌어다 놓으면 일정이 잡힌다.** 화면에 계속 있던 안내 문구("백로그를 날짜 칸으로 드래그하면 일정이 잡힙니다")가 실제로 동작한다.

## 2. 동작 규칙 (사용자 확인)

떨어뜨린 날짜를 **덮는 단계가 있으면 그 단계로 편입**하고, 없으면 예정일만 잡는다.

| 떨어뜨린 대상 | 결과 |
|---|---|
| 같은 프로젝트의 단계가 그 날짜를 덮음 | **그 단계로 편입** + 예정일 = 떨어뜨린 날짜 |
| 덮는 단계 없음 | 백로그에 남고 **예정일만** 잡힘 → 그 프로젝트 박스 안 칩으로 표시 |
| 미배정 작업(프로젝트 없음) | 편입할 단계가 없으므로 **예정일만** → "미배정" 묶음(docs/74)에 표시 |

편입 시 예정일은 기존 규칙(`max(단계 시작일, 오늘)`)이 아니라 **사용자가 지정한 그 날짜**다. 이를 위해 `assignTask`에 예정일 override 인자를 더했다 — 인자를 주지 않으면 기존 계산 그대로다.

## 3. 구현

프로젝트 상세에서 만든 드래그 규약(`task-drag.ts`, docs/67)을 **그대로 재사용**했다. 전용 MIME으로 외부 드롭을 배제하고, `dragover`에서는 타입만 보고 하이라이트를 판정한다.

| 파일 | 역할 |
|---|---|
| `my-work-backlog.tsx` | 백로그 행을 `draggable`로, 안내 title·grab 커서 |
| `my-work-calendar.tsx` | 날짜 칸에 `onDragOver`/`onDragLeave`/`onDrop`, 놓일 칸 하이라이트(`bg-primary/10` + inset ring), `onDropTask` prop |
| `my-work-calendar-panel.tsx` | `handleDropTask` — 덮는 단계 탐색 후 편입 또는 예정일만 |
| `board-store.tsx` | `assignTask`에 `scheduledDateOverride` 추가 |

칸의 날짜는 `shiftISO(grid.gridStart, 주×7 + 열)`로 구한다 — 그리드가 이미 첫 칸 날짜를 들고 있어 별도 계산이 필요 없다.

> 캘린더에는 이미 포인터 기반 드래그(단계 막대·할일 칩 이동)가 있는데, 이번 것은 **HTML5 DnD**라 이벤트가 겹치지 않는다. 목록 → 칸 이동에는 브라우저 기본 DnD가, 날짜 축 위 미세 조정에는 포인터 드래그가 각각 맞다.

## 6. 알려진 사항 / 후속 과제

- 캘린더는 **내가 소유한 프로젝트**만 그린다. 백로그 패널은 전 프로젝트를 보여주므로, 남의 프로젝트 백로그 작업을 떨어뜨리면 예정일은 저장되지만 이 캘린더에는 보이지 않는다
- 드롭다운으로 백로그를 다시 지정하면(`assignTask` → 백로그) 예정일이 해제된다. 드롭으로 잡은 날짜와 규칙이 어긋나 보일 수 있어, 예정일 해제 규칙 자체를 재검토할 여지가 있다
- 캘린더 → 백로그 역방향(일정 해제)은 아직 없다
