- **예정 커밋 메시지**: `feat: 프로젝트 상세 백로그 티켓 형태 통일`
- **작업일**: 2026-07-22
- **작업 브랜치**: `프로젝트-백로그-티켓-형태` (워크트리 `.claude/worktrees/project-backlog-ticket`, base: main `cce57f3`)

---

## 1. 작업 내용 요약

프로젝트 상세 페이지의 백로그 항목을 **내 작업 페이지의 백로그 티켓과 같은 형태**로 통일했다 (사용자 요청).

| 요소 | 이전 (프로젝트 상세) | 이번 |
|---|---|---|
| 작업명 | 단순 텍스트 | **클릭 → 작업 상세(티켓) 오버레이** |
| 소속 배지 | 없음 | **드롭다운 배지** — 백로그 ↔ 단계 지정 |
| 완료 체크 | 있음 | 유지 |

**"단계나 이런 것만 지정"** 요구대로, 내 작업 화면과 달리 **프로젝트는 바꾸지 않는다**. 이 화면은 이미 프로젝트 스코프 안이므로 드롭다운 선택지는 **해당 프로젝트의 백로그 + 단계 목록**뿐이다.

## 2. 주요 구현 결정

| 항목 | 결정 | 이유 |
|---|---|---|
| 티켓 상세 | 기존 `TaskDetailOverlay` 재사용 (내 작업과 동일 컴포넌트) | "동일한 형태" 요구 충족 + 중복 구현 방지. 티켓 번호(YOS-757)·이슈 등록 등 기능 그대로 |
| 소속 변경 | 기존 `boardActions.assignTask` 재사용, `toProjectId`를 현재 프로젝트로 고정 | 새 액션 불필요. DB 저장 경로도 동일 |
| 드롭다운 구성 | `백로그`(현재, 체크) + 단계들(색 도트 + 이름) | 내 작업은 프로젝트별 그룹 구조가 필요하지만 여기선 단일 프로젝트라 평면 목록이 맞음 |
| 배지 라벨 | `백로그` (프로젝트명 생략) | 내 작업은 `{프로젝트} · 백로그`가 필요하지만, 여기선 프로젝트가 자명해 중복 표기 |
| 안내 문구·입력창 | 기존 유지 | 요구 범위 밖 |

## 5. 참고 — 검증 중 만난 병렬 환경 이슈

- dev 서버 포트 3009가 다른 세션(`mywork-box-minheight`) 서버에 이미 점유되어, 내 서버가 뜨지 못한 채 **그 세션 화면을 테스트할 뻔했다**. 포트 점유자를 확인하고 3012로 옮겨 해결.
- 교훈: 워크트리 dev 서버는 **띄운 뒤 포트 점유자를 확인**하는 편이 안전하다 (`Get-NetTCPConnection`으로 워크트리명 확인).
- Radix 드롭다운은 `element.click()`으로 열리지 않는다(pointerdown 기반) — 브라우저 검증 시 실제 마우스 클릭(`elementHandle.click()`)을 써야 한다.

## 6. 알려진 사항 / 후속 과제

- 드래그 앤 드롭(백로그 ↔ 단계)은 여전히 미구현 — 안내 문구만 존재. 드롭다운이 그 역할을 임시 대체
- 보드 카드(단계 안의 작업)는 아직 티켓 형태가 아님 — 클릭 시 상세로 가려면 별도 작업 필요
