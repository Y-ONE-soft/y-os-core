## 1. 작업 내용 요약

같은 성격의 두 화면인데 생김새가 달랐던 **단계 상세 오버레이**를 **작업 상세 오버레이 기준으로 맞췄다** (사용자 요청: "지금 다르잖아").

작업 상세가 더 나중에·정교하게 만들어졌으므로 그쪽을 기준으로 삼고, 단계 상세를 끌어올리는 방향으로 통일했다.

| 요소 | 이전 (단계 상세) | 이번 (= 작업 상세) |
|---|---|---|
| 다이얼로그 | `h-[87dvh] w-[92vw]`, 기본 라운드 | **1280×780 고정(뷰포트 여백 24px), `rounded-[16px]`** |
| 닫기 버튼 | shadcn 기본(우상단 작은 X) | **헤더 우측 커스텀 ✕ 버튼**(`size-9`), 기본 버튼 숨김 |
| 헤더 칩 | `bg-muted`만 | **`border` + `bg-muted` + muted 텍스트**, 내용도 `단계 N`으로 순번 표기 |
| 헤더 구분자 | `·` 단독 | `&nbsp;&nbsp;·&nbsp;&nbsp;` (작업 상세와 동일 간격) |
| 제목 | `text-2xl font-bold tracking-tight` | **`text-[30px] leading-tight font-semibold`** |
| 완료 체크박스 | `size-[22px]` | 동일 + **`[&_svg]:size-4`** |
| 섹션 헤딩 | `text-[13px]` | **`text-sm`(14px)**, 카운트도 `&nbsp;` 간격 규칙 적용 |
| 내용 입력 | 커스텀 `<textarea>` + 별도 `DescriptionEditor` 컴포넌트 | **shadcn `Textarea` + `defaultValue`/`onBlur`** (작업 상세와 같은 패턴, 보조 컴포넌트 제거) |
| 댓글 아바타 | `bg-primary` + 흰 글자 | **`bg-muted`** |
| 댓글 본문 | `text-[13px]` | **`text-sm`** |
| 본문/사이드 구분 | `border-l` | **`w-px bg-border` 분리 요소** |
| 사이드 패널 | 배경 없음 | **`bg-muted`** |
| 사이드 필드 | `Label` + 기본 Input | **`text-xs text-muted-foreground` 라벨 + `h-9 rounded-[8px] bg-background` 입력** |
| 프로젝트 표시 | 텍스트 한 줄 | **입력 필드와 같은 높이의 박스**(작업 상세의 Select와 동일한 시각 무게) |
| 공동 작업자 버튼 | lucide 아이콘 + 기본 스타일 | **`👥` 이모지 + `w-full rounded-[8px] bg-background`** (작업 상세와 동일) |
| 하단 메타 | `mt-auto` | **`flex-1` 스페이서 + `text-[11px] leading-[1.6]`** |

## 2. 통일하지 않은 것 (의도적)

| 항목 | 이유 |
|---|---|
| 티켓 번호 칩(`YOS-655`) | 단계는 티켓이 아니다. 대신 같은 자리에 `단계 N` 순번을 넣어 **시각적 리듬만** 맞췄다 |
| `🐞 이슈 등록` 버튼 | 이슈는 작업 단위로 등록한다. 단계에는 대응 개념이 없어 헤더 우측은 닫기만 둔다 |
| 작업 유형·난이도 셀렉트 | 작업 고유 속성. 단계에는 기간·데드라인 표시가 그 자리를 대신한다 |
| 프로젝트 **변경** Select | 단계는 프로젝트에 종속돼 이동 개념이 없다. 표시만 같은 모양으로 |
| 산출물 업로드·티켓 연결 실동작 | 각각 별도 도메인 작업. 단계 상세에서는 안내 문구로 두되 **레이아웃은 동일** |
| **삭제 버튼** | 작업 상세에는 있으나 **단계 삭제 API가 아직 없다**(`deleteStage` 부재). 서버 작업이 필요해 이번 UI 통일 범위에서 제외 — 후속 과제 |

## 5. 참고 — 병렬 세션 정리

- 같은 작업을 준비하던 다른 세션(`detail-overlay-align`, pid 30408)이 20분간 소스 변경 없이 멈춰 있었다. 사용자 지시로 해당 프로세스를 종료하고 이 세션이 이어받았다.
- 그 워크트리(`.claude/worktrees/detail-overlay-align`)와 브랜치(`단계-작업-상세-화면-일치`)는 **커밋이 없는 빈 상태로 남아 있다** — 제거는 권한 제한으로 보류. 필요 시 `git worktree remove --force` + `git branch -D`로 정리하면 된다.
