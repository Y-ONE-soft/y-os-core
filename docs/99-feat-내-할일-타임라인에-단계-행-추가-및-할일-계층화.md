# 99. feat: 내 할일 타임라인에 단계 행 추가 및 할일 계층화

## 작업 요약

내 할일 타임라인 뷰(docs/82)에 **단계 행**을 넣고, 할일을 소속 단계 아래로 계층화했다.

## 문제

프로젝트 아래에 **모든 할일이 평평하게** 나열됐다. 단계는 어디에도 보이지 않았고, 단계에 속한 할일과 백로그 할일이 섞여 있었다.

```
▸ 프로젝트                    [전체 47% ────────]
    ☐ 할일 A                        ▪ 7/22
    ☐ 할일 B                        ▪ 7/25
    ☐ 백로그 할일                    (날짜 없음)
```

할일이 어느 단계 소속인지, 그 단계가 언제부터 언제까지인지 알 수 없었다. 프로젝트 상세 로드맵은 단계를 보여주는데 이 화면만 빠져 있었다.

## 변경 후

```
▸ 프로젝트                    [전체 47% ────────]
  ● 단계 1          1/3      [단계 1 ──────]
      ☐ 할일 A                    ▪ 7/22
      ☐ 할일 B                    ▪ 7/25
  ● 단계 2          0/2      [단계 2 ────]
      ☐ 할일 C                    ▪ 7/28
    백로그          0/1
      ☐ 예정 없는 할일
```

## 변경 내용

### `src/components/features/my-work/my-work-timeline-panel.tsx`

**1. 단계 행 추가**

라벨은 단계 색 점 + 이름 + `완료/전체`, 막대는 `startDate~endDate`를 단계 색으로 그린다. 프로젝트 상세 로드맵의 단계 행과 같은 구성이고 같은 `RoadmapBar`를 쓴다.

시작일이 없는 단계는 막대 없이 행만 나온다 — 단계가 있다는 사실 자체가 정보다.

**2. 할일을 소속 단계 아래로**

`stages.map(...)` 안에서 그 단계의 할일만 그린다. 들여쓰기가 세 단계가 됐다.

| 층 | 왼쪽 여백 |
| --- | --- |
| 프로젝트 | `pl-4` |
| 단계 · 백로그 | `pl-7` |
| 할일 | `pl-10` |

**3. 백로그 그룹**

단계에 속하지 않은 할일은 마지막에 `백로그` 그룹 행으로 묶었다. 단계 행과 같은 층·같은 형식이되 색 점과 막대가 없다.

**4. 할일 행을 컴포넌트로 분리**

단계 아래와 백로그 아래 두 곳에서 같은 모양을 써야 해서 `TimelineTaskRow`로 뺐다. 기존 마크업·동작(체크박스, 이름 클릭 시 상세, 예정일 막대 드래그, 막대 오른쪽 날짜 표기)은 그대로다.

**5. 단계 클릭 시 단계 상세 오버레이**

단계 이름과 막대를 누르면 `StageDetailOverlay`가 열린다. 같은 페이지의 캘린더 뷰가 이미 단계 막대 클릭으로 상세를 열고, 이 타임라인도 할일은 클릭하면 상세가 열린다. 단계만 안 열리면 어긋난다.

오버레이가 프로젝트 이름·색을 함께 받으므로 열린 단계의 소속 프로젝트를 찾아 넘긴다. 프로젝트 수가 적어 매 렌더 훑어도 부담이 없다.

**6. 헤더 라벨** — `프로젝트 · 작업` → `프로젝트 · 단계 · 할일`

## 검증

```bash
npx tsc --noEmit          # 통과 (출력 없음)
npm run lint              # 통과
npm run build             # 성공 — Compiled successfully
```

**시각 확인은 프리뷰로 사용자가 한다.** 저장소에 브라우저 자동화 도구가 없고, 워크스페이스 캐시의 `getServerSnapshot()`이 항상 빈 값이라 SSR HTML로는 본문이 렌더되지 않는다.

## 진행 중 있었던 것

소속 프로젝트 조회를 처음에 `useMemo` + `for...of` + 조기 반환으로 짰더니 `react-hooks/preserve-manual-memoization`에 걸렸다. React Compiler가 그 형태의 메모이제이션을 보존하지 못한다. `find`로 바꾸고 `useMemo`를 걷어냈다 — 컴파일러가 알아서 처리하는 영역이라 수동 메모가 필요 없다.

## 알려진 이슈 / 후속

- **단계 막대는 드래그할 수 없다.** 할일 막대는 끌어서 예정일을 옮길 수 있지만 단계 막대는 클릭만 된다. 프로젝트 상세 로드맵은 단계 드래그가 되므로 여기서도 되는 게 자연스럽지만, 이번 요청 범위 밖이라 남긴다.
- 프로젝트 행의 `전체 N%`와 진행률 계산 방식은 그대로다.
