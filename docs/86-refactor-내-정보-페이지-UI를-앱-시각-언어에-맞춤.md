# refactor: 내 정보 페이지 UI를 앱 시각 언어에 맞춤

요청 사이클 `내-정보-페이지-UI-정합`의 단일 태스크. "UI 좀 제대로 해줘" 지적에 따라 docs/84에서 만든 `/me` 화면을 다시 짠다. 기능·API 변경은 없고 화면 구성만 바꾼다.

## 왜 투박했나

**Figma에 내 정보 화면 디자인이 없다.** 디자인 파일(`jimmy-design-system`, fileKey `Vmv6OlcsEHpHnf5vNDVDoX`)의 `🚀 Y.OS Core` 페이지 최상위 프레임을 확인한 결과 8개뿐이다.

```
64:140  Y.OS Shell
74:13   Y.OS Shell — Collapsed Sidebar
74:65   Y.OS Shell — User Menu Open
78:11   Y.OS Shell — Projects · Master
79:48   Y.OS Shell — Projects · Staff
82:11   Y.OS Shell — Projects · Master · Adding
82:100  Y.OS Shell — Projects · Staff · Adding
92:158  Y.ONE Logo
```

다른 화면들은 전부 Figma 노드를 근거로 만들어졌다(코드 주석에 노드 id가 남아 있다). **내 정보 화면만 근거 없이 즉흥적으로 만들어져** 앱의 시각 언어에서 벗어났다.

구체적인 문제는 이랬다.

| 문제 | 이전 |
|---|---|
| 카드가 페이지에 붕 떠 있음 | `max-w-[560px]` 한 장이 왼쪽에 몰리고 우측 2/3이 빈 공간 |
| 읽기 전용 필드 배치가 어긋남 | 2열 그리드에 항목 3개라 세 번째가 홀로 남음 |
| 구획이 없음 | 프로필·계정 정보·편집 필드가 한 상자에 평평하게 나열 |
| 앱 패널 스타일 미적용 | 다른 패널은 `border + shadow-[0px_1px_3px_0px_rgba(0,0,0,0.05)]`인데 테두리만 |
| 저장 액션이 미완성처럼 보임 | 좌측 하단 버튼 + 맨 텍스트 상태 |

## 변경 내용

### 좌우 2단 구성

좌측 계정 요약(고정폭 280px) + 우측 편집 폼(유연, 최대 560px).

```tsx
{/* 좌: 계정 요약(고정폭) · 우: 편집 폼. 한 장짜리 카드는 넓은 화면에서
    왼쪽에 몰려 페이지가 비어 보인다 — 백로그가 있는 화면들과 같은 구성. */}
```

프로젝트 상세·내 작업이 모두 "본문 + 고정폭 aside" 구성이라 낯설지 않다. `flex-wrap`이라 좁은 화면에서는 자연히 세로로 쌓인다.

### 앱 패널 규격 통일

```ts
/** 이 화면의 패널 — 백로그·로드맵 등 앱 전반과 같은 규격 */
const PANEL =
  "rounded-[8px] border bg-background shadow-[0px_1px_3px_0px_rgba(0,0,0,0.05)]";
```

`project-backlog.tsx`·`my-work-backlog.tsx`·`project-roadmap.tsx`가 쓰는 값과 동일하다. 상수로 뽑아 두 패널이 어긋날 여지를 없앴다.

### 읽기 전용 필드를 목록형으로

이전에는 라벨 위·값 아래의 세로 필드였고 2열 그리드에서 3개가 어긋났다. 좌우 정렬 목록(`라벨 … 값`)으로 바꿔 개수와 무관하게 정렬이 유지된다.

```tsx
/** 계정 정보처럼 바꿀 수 없는 값 — 입력처럼 보이면 고칠 수 있다고 오해한다 */
function ReadOnlyRow({ label, value }) { … }
```

입력 필드로 보이지 않게 하는 원칙은 그대로 유지했다.

### 액션을 푸터 밴드로

```tsx
{/* 액션은 muted 밴드로 내려 본문과 분리한다 — 폼 안에 버튼이 그냥
    떠 있으면 저장 지점이 눈에 안 들어온다.
    mt-auto로 바닥에 붙여 좌측 계정 패널과 높이를 맞춘다. */}
```

저장 버튼을 우측 정렬하고 상태 메시지를 그 옆에 뒀다. `프리셋 저장 다이얼로그`가 푸터를 분리하는 방식과 같은 계열이다. `items-stretch` + `mt-auto` 조합으로 두 패널 높이가 맞아 들쭉날쭉함이 사라졌다.

### 중복 표기 제거 (스크린샷을 보고 고침)

1차 개편 스크린샷에서 이름 "스탭" 바로 아래 역할 칩 "스탭"이 붙어 **같은 말이 두 번** 보였다.

```tsx
{/* 이름 아래에는 아이디를 둔다. 역할·소속은 아래 계정 목록에 있으므로
    여기에 칩으로 또 넣으면 "스탭 / 스탭"처럼 같은 말이 두 번 보인다. */}
```

이름 아래를 `@step01`(아이디)로 바꿨다. 그러자 이번엔 계정 목록의 "아이디 step01" 행과 겹쳐서, **그 행도 제거**했다. 최종적으로 계정 목록은 `역할`·`소속 그룹` 두 줄이다.

### 그룹 로딩 자리표시

그룹 이름은 프로젝트 스토어 로드 후에야 알 수 있다. 이전 구현은 빈 문자열이라 `—`로 보였는데, 이는 "소속 없음"으로 읽힌다.

```tsx
{/* 그룹 이름은 스토어 로드 후에야 안다. 그동안 "—"를 보여주면
    "소속 없음"으로 읽히므로 자리만 잡아둔다. */}
```

로드 전에는 `Skeleton`으로 자리만 잡는다. 페이지 전체 스켈레톤도 실제 레이아웃(좌 280px + 우 유연 2단)에 맞춰 다시 그렸다.

## 검증

`npm run build` 성공, `npm run lint` 경고 0. **기능·API는 건드리지 않았고** 화면 구성만 바꿨다.

UI 작업이라 헤드리스 Chrome + CDP로 **매 단계 스크린샷을 보며** 진행했다. 실제로 이번 변경의 절반(중복 표기 2건, 그룹 자리표시)은 코드가 아니라 **스크린샷을 보고 발견한 것**이다.

저장 흐름이 그대로인지도 확인했다.

```
드롭다운 항목: ["알림2","내 정보","프리셋 관리","설정","로그아웃"]
이동한 경로: /me
저장 결과: 저장했습니다.
헤더 표시 이름: 검검증용 이름       ← 아바타 이니셜 + 이름, 즉시 반영
저장 버튼 비활성(변경 없음): true
서버 값: 검증용 이름 / 책임
```

소속 그룹이 `Soft`(id가 아닌 이름)로 표시되는 것도 확인했다. 검증 중 바꾼 계정 정보는 시드 값(`스탭 / 사원 / step01@y-os.local / 010-0000-0002`)으로 복구했다.

## 알려진 이슈 / 주의점

### Figma 원본이 여전히 없다

이번 개편은 **기존 화면들에서 규격을 역으로 추출해 맞춘 것**이지 디자인 원본을 옮긴 게 아니다. 나중에 내 정보 화면 디자인이 나오면 다시 맞춰야 할 수 있다. 같은 처지의 화면(프리셋 관리 등)이 늘고 있으므로, **Figma에 없는 화면을 위한 디자인 규약을 문서로 정하는 편이 낫다** — 페이지 셸(`px-6 pb-6 pt-5`, `text-[22px]` 헤더), 패널(`rounded-[8px] border + shadow`), 폼 필드(`h-9 rounded-[8px]`, 라벨 `text-xs text-muted-foreground`) 정도는 이미 사실상 굳어져 있다.

### 넓은 화면에서 아래쪽 여백이 크다

편집 항목이 4개뿐이라 세로가 짧다. 비밀번호 변경·알림 설정 등이 붙으면 자연히 채워진다. 억지로 늘리지 않았다.

### 스켈레톤이 실제 높이와 정확히 일치하지는 않는다

`h-[236px]`로 근사했다. 로드 전후 미세한 흔들림이 있을 수 있다.
