# 125. fix: 내 할일 캘린더 단계 배지를 할일 수에서 단계 순번으로

## 작업 요약

내 할일 캘린더의 단계 막대 원형 배지가 **할일 개수**를 보여주던 것을 **단계 순번**으로 바꿨다. 프로젝트 상세 로드맵·작업 현황 로드맵과 같은 기준이다.

## 문제

캘린더 단계 막대의 배지는 `stage.tasks.length`(그 단계의 할일 개수)를 넣고 있었다.

그런데 같은 막대 UI를 쓰는 다른 화면은 **단계 순번**을 넣는다.

- `project-roadmap.tsx:274` — `badge={index + 1}`
- `workload-roadmap.tsx:331` — `badge={stageIndex + 1}`

`RoadmapBar`의 `badge` prop 주석도 "단계 순번 — 내 할일 캘린더와 같은 원형 배지로 표시"라 순번이 원래 의도였다. 캘린더만 어긋나 있었다.

## 변경 내용

### `src/components/features/my-work/my-work-calendar-source.ts`

- `stageSegments`에 `stageIndex`(0-based) 인자 추가
- 오버레이의 `count: stage.tasks.length` → **`stageNo: stageIndex + 1`**
- 호출부를 `for...of` → `stages.forEach((stage, stageIndex) => …)`로 바꿔 index를 넘김

순번은 **그 프로젝트 보드의 단계 순서** 기준이다 — `startDate` 유무나 이 달 표시 여부와 무관하게 보드 순서로 매긴다. 로드맵과 완전히 같은 기준이라, 같은 단계가 두 화면에서 같은 번호를 갖는다.

### `src/components/features/my-work/my-work-calendar-layout.ts`

- `CalOverlay`의 `stage` 종류에서 `count: number` → `stageNo: number`. 필드 의미가 "할일 수"에서 "단계 순번"으로 바뀌므로 이름도 바꿔 혼동을 없앴다

### `src/components/features/my-work/my-work-calendar.tsx`

- 배지 렌더 `{overlay.count}` → `{overlay.stageNo}`

## 바꾸지 않은 것

헤더의 `이 달 N건`은 이미 **단계 수**(`stageCount`)라 맞다. 이건 "이 달에 걸친 단계가 몇 개인가"이고 배지는 "이 단계가 몇 번째인가"라 서로 다른 값이다. 건드리지 않았다.

## 검증
