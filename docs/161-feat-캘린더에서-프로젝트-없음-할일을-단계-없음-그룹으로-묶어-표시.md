# feat: 캘린더에서 프로젝트 없음 할일을 단계 없음 그룹으로 묶어 표시

## 요청

> 프로젝트 없음 하나 있고, 똑같이 "단계 없음"을 유지한다. 프로젝트 없음은 프로젝트 상세엔 안 보이고 내 할일 백로그에만 보임. 이게 캘린더에 들어가면 이것만 특수하게 단계에 포함되지 않고 **할일만** 보여주면 된다. 프로젝트 없음 할일이 캘린더로 드래그되거나 날짜를 확정하면 **"단계 없음"으로 표현되어 캘린더/뷰에 표시**되게.

## 배경

- 앞서 #143(`미배정 칩 별도 영역 제거`)로, 프로젝트 없음(미배정) 할일이 캘린더에서 **박스 없이 낱개 회색 칩**으로 흩어져 배치됐다.
- 이번엔 그것을 **하나의 "프로젝트 없음" 그룹 박스**로 다시 묶는다. 단, 실제 프로젝트 박스와 달리 **단계 막대는 없고(=단계 없음) 할일 칩만** 담는다.
- 데이터·소속은 그대로다. 예정일이 있는 미배정 할일만 캘린더에 뜨는 규칙도 그대로(드래그·날짜 확정으로 예정일이 생기면 이 박스에 나타난다).

## 구현

### 배치 — `my-work-calendar-layout.ts`

- 그동안 `UNASSIGNED_BOX`를 박스 배치(`active`)에서 **제외**하고 낱개 칩을 빈 레인에 자유 배치했다. 이 제외와 낱개 배치 루프를 없앴다.
- 이제 `UNASSIGNED_BOX`도 `active`에 포함해 **프로젝트 박스와 같은 경로로 하나의 박스**가 된다. 단계 오버레이가 없으므로 단계 줄 수는 기본 1(빈 단계 줄), 그 아래 할일 칩 줄 — 즉 "단계 없음 + 할일만".
- 정렬에서 **항상 실제 프로젝트 박스들 뒤(아래)** 로 가도록 `UNASSIGNED_BOX`를 정렬 키 끝에 붙였다("프로젝트 우선").

### 이름표 — `my-work-calendar-source.ts`

- 미배정 칩이 하나라도 있으면 `meta[UNASSIGNED_BOX]`를 **`{ name: "프로젝트 없음", color: 회색 }`** 으로 넣는다. 이 메타가 있어야 박스에 이름표가 붙는다.

### 렌더 — `my-work-calendar.tsx` (변경 없음)

- `ProjectBoxItem`은 이미 `UNASSIGNED_BOX`를 특수 처리한다(실제 프로젝트가 아니므로 상세 링크 없이 이름표만, 회색). 단계 막대가 없으니 자연히 할일 칩만 담긴다.

## 변경 파일

- `src/components/features/my-work/my-work-calendar-layout.ts` — `active`에 `UNASSIGNED_BOX` 포함 + 맨 뒤 정렬, 낱개 배치 루프 삭제, 관련 주석 갱신.
- `src/components/features/my-work/my-work-calendar-source.ts` — 미배정이 있으면 `meta`에 "프로젝트 없음" 이름표 추가.

## 새 API / 스키마

- **없음.** 표시(그룹 박스) 변경만. 데이터·소속·생성 경로는 그대로.

## 결정 / 기본값

- 박스 이름표는 **"프로젝트 없음"**(회색). 그 안의 할일은 단계 막대가 없는 것으로 "단계 없음"을 표현한다(별도 "단계 없음" 라벨은 달지 않음). 필요하면 라벨 문구는 조정 가능.
- 실제 프로젝트 박스처럼 첫 항목~마지막 항목 구간을 잇는 연속 밴드(옅은 회색 틴트)로 그려진다 — 프로젝트 박스와 같은 방식.

## 검증

- `npm run lint` 통과 / `npx tsc --noEmit` 통과(exit 0).
- 배치 로직 검토: `UNASSIGNED_BOX`가 이 주에 칩이 있으면 항상 `active`에 포함돼(전역 range가 그 주를 지남) 모든 칩이 레인을 배정받음 — 낱개 배치 루프 삭제로 인한 미배정 칩 누락 없음.
- 프로젝트 없음 그룹 박스가 하단에 묶여 보이는지의 시각 확인은 머지 후 프로덕션/사용자 확인(태스크 ①의 정렬과 함께).
