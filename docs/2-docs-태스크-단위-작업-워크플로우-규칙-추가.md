# 2. 태스크 단위 작업 워크플로우 규칙 추가

- **예정 커밋 메시지**: `docs: 태스크 단위 작업 워크플로우 규칙 추가`
- **작업일**: 2026-07-22

---

## 1. 작업 내용 요약

`CLAUDE.md`의 "작업 결과 문서 워크플로우" 섹션을 "작업 워크플로우"로 확장 개정했다. 기존에는 문서 작성 규칙만 있었는데, **작업 진행 사이클 전체**를 규칙으로 명문화했다.

## 2. 배경 (왜)

사용자의 진행 방식 요구를 상시 규칙으로 고정하기 위함:

> 사용자가 작업을 러프하게 말하면 → 에이전트가 태스크로 분해하고 → 태스크는 커밋 1개 단위와 일치시키고 → 작업 후 결과를 히스토리 파일로 만들면 → 사용자가 검토하고 넘어가는 방식

`CLAUDE.md`는 에이전트가 매 세션 자동으로 읽는 파일이므로, 여기에 등록하면 이후 어떤 세션에서도 같은 방식으로 동작한다.

## 3. 확정된 워크플로우 사이클

1. **태스크 분해** — 러프한 요청을 태스크로 나눈다. 태스크 1개 = 커밋 1개 = 작업 결과 문서 1개. 각 태스크에 예정 커밋 메시지를 붙인 분해안을 먼저 보여준다.
2. **작업** — 한 번에 태스크 하나씩. 완료 전 빌드·린트 검증.
3. **작업 결과 문서 작성** — `docs/<번호>-<커밋-메시지-슬러그>.md`
4. **검토 대기** — 사용자 승인 없이 다음 태스크로 넘어가지 않는다.
5. **커밋** — 승인 시 작업 + 문서를 같은 커밋으로 찍고 다음 태스크 진행.

문서 규칙 보강 사항: "커밋에 포함되는 모든 변경이 문서에 기록되어야 하며, 작업했지만 최종 diff가 없는 것도 히스토리로 남긴다"를 명문화 (1번 커밋 작업 중 받은 피드백 반영).
