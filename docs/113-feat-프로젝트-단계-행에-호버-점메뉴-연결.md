## 작업 요약

docs/112에서 만든 `RowActions`를 남은 세 곳에 붙였다. 이로써 **프로젝트·단계·할일 6종 전부**가 같은 방식으로 호버 ⋮ → 삭제를 지원한다.

| 대상 | 위치 |
| --- | --- |
| 단계 | 보드 컬럼 헤더 |
| 프로젝트 (스탭) | 사이드바 플랫 목록 |
| 프로젝트 (마스터) | 사이드바 그룹별 목록 |

## 할일 행과 달랐던 점

앞 태스크의 할일 행은 세 곳 모두 구조가 같아 `group` 클래스와 `<RowActions>` 한 줄이면 끝났다. 프로젝트·단계는 각각 다른 제약이 있었다.

### 프로젝트 — `<Link>` 안에 버튼을 넣을 수 없다

프로젝트 행 전체가 `<Link>`다. 그 안에 `<button>`을 넣으면 **앵커 안의 인터랙티브 요소**가 되어 마크업이 유효하지 않고, 눌렀을 때 페이지 이동이 함께 일어난다.

그래서 ⋮ 를 `<li>`의 **형제**로 두고 절대 위치로 겹쳤다.

```tsx
<li className="group relative flex items-center">
  <ContextMenu>…<Link className="… pl-3 pr-9">…</Link>…</ContextMenu>
  <RowActions className="absolute right-1.5" … />
</li>
```

- `group`이 `<li>`에 있으므로 **행 어디에 올려도** ⋮ 가 나타난다(Link 위든 버튼 위든).
- `<Link>`에 `pr-9`를 줘 오른쪽 버튼 자리를 비웠다. 이게 없으면 긴 프로젝트 이름의 말줄임이 버튼 아래로 파고든다.

마스터 목록은 `pl-[34px]`(들여쓰기) + `pr-9`, 스탭 목록은 `px-3`을 `pl-3 pr-9`로 풀었다.

### 단계 — 컬럼 강조와 겹친다

보드 컬럼은 호버하면 전체에 링이 생기고, 안쪽 카드·버튼을 가리키는 동안에는 `has-[[data-column-child]:hover]:ring-0`으로 그 강조를 끈다. ⋮ 도 같은 취급을 받아야 하므로 `data-column-child`를 붙인 `<span>`으로 감쌌다.

```tsx
<span data-column-child className="flex shrink-0">
  <RowActions … />
</span>
```

`group`은 **헤더**에 붙였다. 컬럼 전체에 붙이면 카드를 훑을 때도 단계 ⋮ 가 계속 떠 있어 카드 자신의 ⋮ 와 둘이 보인다. 기존 코드가 "순서 변경 손잡이는 헤더로 한정한다"고 정해 둔 것과도 맞는다.

### 권한 — 마스터 목록에서만 노출

마스터 그룹 목록의 프로젝트 행은 원래 `isMaster`일 때만 컨텍스트 메뉴(삭제)를 단다. ⋮ 도 같은 조건으로 감쌌다.

스탭 플랫 목록은 다르다. 그 목록은 `staffProjects`(자기가 작업자인 프로젝트)만 담으므로 **걸리는 행이 전부 삭제 권한이 있다** — 기존 주석이 밝혀 둔 대로다. 그래서 조건 없이 붙였다.

`pr-9`는 마스터가 아닐 때도 적용된다. 버튼이 없으면 오른쪽 여백이 조금 넓어질 뿐이고, 목록 간 정렬은 오히려 통일된다.

## 색 변경은 우클릭 메뉴에만 남겼다

마스터의 프로젝트 우클릭 메뉴에는 **색 변경 팔레트 + 삭제**가 있다. ⋮ 메뉴에는 **삭제만** 넣었다.

`RowActions`의 `actions`는 `{label, onSelect}` 목록이라 팔레트 같은 임의 UI를 받지 못한다. 받게 만들면 6종이 공유하는 컴포넌트가 프로젝트 전용 사정을 알게 되고, 이번에 세운 "도메인 로직 없는 얇은 조합"이 무너진다.

결과적으로 **두 메뉴의 항목이 다르다** — ⋮ 는 삭제, 우클릭은 색 변경 + 삭제. 의도한 절충이며, 색 변경까지 ⋮ 로 옮기려면 `RowActions`에 커스텀 섹션을 받는 설계가 따로 필요하다.

### 육안 확인이 불가능한 이유를 확인했다

**이 앱은 데이터 행을 서버에서 렌더하지 않는다.** 이번에 사이드바로 확인했다.

```
사이드바 nav 항목("작업 현황"·"프리셋")   → HTML에 있음
DB의 프로젝트 4개                        → HTML에 0개
⋮ 버튼(aria-label="… 메뉴 열기")          → 0개
```

프로젝트 목록은 `useProjectStore`, 보드는 `useBoardState`로 그려지는데 두 스토어 모두 `getServerSnapshot()`이 빈 값이라(docs/78에 기록됨) **정적 셸만 SSR되고 행은 클라이언트에서 붙는다.** 따라서 `curl`로는 어느 화면에서도 ⋮ 를 확인할 수 없다.

확인이 필요한 것:

1. 행/헤더에 올렸을 때만 ⋮ 가 보이는가
2. 프로젝트 ⋮ 를 눌러도 **페이지 이동이 일어나지 않는가** (Link 형제 배치)
3. 단계 ⋮ 를 눌러도 단계 상세가 열리지 않는가 (컬럼 `onClick` 차단)
4. 단계 ⋮ 에 올렸을 때 컬럼 전체 링이 꺼지는가 (`data-column-child`)
5. 긴 프로젝트 이름이 버튼을 침범하지 않는가 (`pr-9`)
6. 헤더 드래그(단계 순서 변경)가 여전히 되는가

## 알려진 이슈

- **⋮ 와 우클릭 메뉴의 항목이 다르다** (프로젝트 한정). 위 "색 변경" 절 참조.
- **그룹 행에는 붙이지 않았다.** 그룹도 우클릭 삭제가 있지만 요청은 "프로젝트·할일·단계"였다. 같은 컴포넌트로 쉽게 확장할 수 있다.
- **상세 오버레이의 삭제 버튼은 그대로다.** 단계·할일 상세 오버레이 하단에 삭제 버튼이 따로 있는데, 이번 변경과 중복되지만 성격이 다른 진입점이라 두었다.
