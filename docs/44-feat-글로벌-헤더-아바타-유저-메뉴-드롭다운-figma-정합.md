# feat: 글로벌 헤더 아바타 유저 메뉴 드롭다운 Figma 정합

## 1. 작업 요약

글로벌 헤더 우측 아바타를 클릭하면 열리는 **유저 메뉴 드롭다운**을 Figma `Y.OS Shell — User Menu Open`(`74:65`) 안의 `User Menu Dropdown`(`187:821`) 스펙으로 정합했다.

기존에도 드롭다운은 있었으나(docs/4에서 셸 구현 시 최소 형태로 도입) 항목이 `설정`·`로그아웃` 2개뿐이고 컨테이너 스펙도 shadcn 기본값이었다. 이번 작업으로 **상단 프로필 블록(`_Me`) + 항목 5종(알림·내 정보·프리셋 관리·설정·로그아웃)** 구성과 컨테이너·타이포·위치 스펙을 디자인에 맞췄다.

DB·API·스키마 변경은 없다. 프로필 블록이 요구하는 `title`·`email`은 이미 `SessionUser`에 실려 오고 있었다.

## 4. 구현 상세 — 스펙 대조

측정은 dev 서버(1440×900 뷰포트)에서 실제 렌더된 DOM의 `getComputedStyle`/`getBoundingClientRect` 기준이다.

### 컨테이너 (`187:821`)

| 항목 | 기존 | Figma | 구현 결과(실측) |
|---|---|---|---|
| 너비 | 220px | 240px | **240px** ✓ |
| 패딩 | `p-1`(4px) | 6px | **6px** ✓ |
| 항목 간격 | 없음(기본) | 2px | **2px** ✓ |
| 라운딩 | `rounded-lg`(10px) | 8px (`brand/radius-base`) | **8px** ✓ |
| 테두리/그림자 | `ring-1` + `shadow-md` | `border` + `0 4px 8px rgba(0,0,0,.1)` | **1px solid + 0 4px 8px rgba(0,0,0,.1)** ✓ |
| 우측 위치 | 트리거 우측단(=화면 24px) | 화면 우측 4px | **4px** ✓ |
| 상단 위치 | 58px | 72px | **72px** ✓ |

`--radius`는 0.625rem(10px)이라 토큰 `rounded-lg`로는 8px이 안 나온다. 디자인이 별도 토큰(`brand/radius-base` = 8px)을 쓰므로 `rounded-[8px]`로 명시했다.

### 위치 오프셋 — 주의 지점

Figma의 패널 위치는 **트리거가 아니라 헤더 기준**이라 Radix 기본 앵커(트리거)와 어긋난다. 두 축 모두 보정이 필요했다.

- **가로**: 디자인은 화면 우측 4px. 헤더 좌우 패딩이 24px이라 트리거 우측단보다 20px 바깥 → `alignOffset={-20}`.
  - Radix 기본 `collisionPadding`이 10px이라 그대로 두면 충돌 회피가 패널을 다시 안쪽으로 되민다 → `collisionPadding={4}`로 낮춰야 4px가 유지된다.
- **세로**: 디자인은 `top: 72px` = 헤더 높이(64) + 8. 그런데 트리거는 헤더 안에서 세로 중앙 정렬이라 **하단이 50px**이다. `sideOffset={8}`이면 패널 상단이 58px이 되어 **헤더를 6px 파고든다**. → `sideOffset={22}`(= 72 − 50).
  - 처음엔 "sideOffset 8 = 디자인의 8px 간격"으로 보고 그대로 뒀는데, 브라우저 실측에서 `top: 58`이 나와 발견했다. 헤더 높이나 트리거 구성이 바뀌면 이 값도 같이 조정해야 한다(코드에 주석으로 근거를 남겼다).

### 프로필 블록 `_Me` (`187:835`)

`DropdownMenuLabel`을 오버라이드해 구성했다. 선택 가능한 항목이 아니므로 `DropdownMenuItem`을 쓰지 않았다(포커스·키보드 내비게이션 대상에서 제외되어야 한다).

- 컨테이너: `flex items-center gap-2.5 p-2` (Figma gap 10 / padding 8)
- 아바타: 32px 원형, `bg-muted` + `text-foreground` — 헤더 트리거 아바타(`bg-primary` + `text-primary-foreground`)와 **의도적으로 다르다**. 디자인이 그렇게 되어 있다.
- 이름: 13px medium `text-foreground`
- 부제: 11px regular `text-muted-foreground`, 내용은 `직책 · 이메일`
  - `title`이 없으면 권한명(마스터/스탭)으로 대체 — 트리거의 기존 규칙과 동일하게 `roleLabel()`로 통일
  - `email`이 없으면 ` · ` 구분자 없이 직책만 출력
  - 긴 이메일 대비 `truncate` + `min-w-0` 적용 (240px 고정 폭이라 넘칠 수 있다)

### 항목 (`187:842` 외)

공통 스펙을 `ITEM_CLASS` 상수로 묶었다 — 5개 항목에 동일 문자열을 반복하면 나중에 한 곳만 빠뜨리기 쉽다.

`gap-2 rounded-[6px] px-2 py-[7px] text-[13px] font-normal` (Figma: 13px regular / padding 8·7 / 라운딩 6)

| 알림 | 없음 | 우측 배지 — `Badge` 기본 variant(`bg-primary`)에 `h-auto rounded-[8px] px-1.5 py-px text-[10px]` 오버라이드 |
| 내 정보 | 없음 | 라우트 미구현 |
| 프리셋 관리 | 없음 | 라우트 미구현 |
| 설정 | 없음 | 라우트 미구현 (기존과 동일) |

## 5. 결정 사항과 이유

| 결정 | 선택 | 이유 |
|---|---|---|

| 가로 위치 | Figma 그대로(화면 우측 4px) | 헤더 px-24 라인과 어긋나지만 픽셀 정합 우선. 기존 정합 태스크(docs/18·27) 관례. 사용자 확인 완료 |
| 라우트 없는 항목 | 비활성화하지 않고 무동작으로 둠 | 디자인상 활성 상태다. `disabled` 처리하면 회색으로 죽어 보여 디자인과 어긋난다 |
| shadcn 원본 수정 | 하지 않음 | 재생성 충돌·타 화면 영향 회피 |

### 검증 환경 특성 (재현 시 참고)

headless Chrome의 **실제 마우스 클릭(CDP)은 Radix 트리거의 `pointerdown`을 발화시키지 못한다** — `page.click()` 후에도 `data-state="closed"` 유지. 합성 `PointerEvent('pointerdown')` 디스패치로 열어야 한다. docs/12·19에 기록된 것과 동일한 제약이며, 실제 브라우저 조작에는 영향 없다.

또한 로그인 직후에는 클라이언트 라우팅 상태라 셸이 서스펜스 경계만 남아 `header`가 DOM에 없다. 검증 스크립트는 로그인 후 `/`로 **전체 리로드**한 뒤 셀렉터를 기다려야 한다.

## 7. 알려진 사항 / 후속 과제

- **알림 배지 숫자는 자리표시(2)** — 알림 도메인 구현 시 API 경계 흐름(`lib/api` → `app/api` → `server`)으로 교체해야 한다. 사이드바 `문의함` 배지(4)도 동일 성격이라 함께 정리하는 편이 낫다.
- **`내 정보`·`프리셋 관리`·`설정`은 아직 무동작** — 대상 라우트가 없다. 프리셋 도메인은 docs/18에서도 후속 대상으로 남아 있던 항목(Figma 161:511·163:676·164:755)이라 연결 지점이 된다.
- **`sideOffset={22}`는 헤더 높이(64px)와 트리거 세로 중앙 정렬에 의존한다.** 헤더 레이아웃이 바뀌면 함께 조정해야 한다.
- 드롭다운 아바타(`bg-muted`)와 헤더 트리거 아바타(`bg-primary`)의 톤이 다른 것은 디자인 의도다. 추후 아바타를 공용 컴포넌트로 뽑는다면 variant로 구분해야 한다.
