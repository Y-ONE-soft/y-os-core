## 1. 작업 내용 요약

작업 현황 로드맵에 사용자 요구 3건을 반영했다.

1. **가로 스크롤** — 타임라인이 화면보다 넓어지면 가로 스크롤. 왼쪽 "프로젝트 · 단계" 라벨 열은 **sticky로 고정**되어 스크롤해도 어떤 행인지 잃지 않는다.
2. **연도 표시** — 헤더에 창 기간을 `2026.07.13 ~ 2026.08.09`로 명시하고, 눈금은 **첫 칸과 연도가 바뀌는 칸에만** 연도를 붙인다(`2026 7/13`, `2026 Q4` → `2027 Q1`).
3. **단계 바 → 상세** — 로드맵의 단계 막대(또는 단계명)를 클릭하면 **단계 상세 오버레이**가 열린다. 지금까지는 프로젝트 상세 페이지에서만 열 수 있었다.

## 2. 주요 구현 결정

| 항목 | 결정 | 이유 |
|---|---|---|
| 스크롤 폭 기준 | `LABEL_WIDTH(200) + 눈금수 × MIN_COLUMN_WIDTH(150)` | 칸이 과도하게 좁아지는 것을 막고, 그보다 좁은 화면에서만 스크롤이 생기게 함 (항상 스크롤 = 불필요한 잘림) |
| 라벨 열 고정 | 각 행의 라벨 셀에 `sticky left-0 z-20` + 행 배경색 | 스크롤 시 막대만 이동. 그룹 헤더 행은 `bg-muted`로 배경을 맞춰 막대가 비쳐 보이지 않게 함 |
| 연도 표기 방식 | 기간 텍스트(항상) + 눈금은 변화 지점만 | 모든 눈금에 연도를 붙이면 `2026 7/13 2026 7/20 …`처럼 소음이 큼. 분기 라벨은 `26 Q3` → `Q3`로 바꾸고 연도를 공통 규칙으로 통일 |
| 단계 상세 진입 | 막대 = `<button>`, 단계명도 클릭 가능 | 막대는 시각적으로 크고 직관적, 단계명은 막대가 없는(기간 미설정) 단계에서도 진입 가능 |
| 프로젝트 `전체` 막대 | 클릭 불가 유지 | 프로젝트 상세는 사이드바 경로가 이미 있고, 집계 막대라 대응하는 단일 상세가 없음 |
| 오버레이 재사용 | 기존 `StageDetailOverlay` 그대로 (props: projectId/name/color/stageId) | 상세 페이지와 동일 UX·동일 저장 경로(DB) |

## 5. 알려진 사항 / 후속 과제

- 프로젝트 상세 페이지 로드맵은 여전히 고정 창 + 자리표시 스위처 — 같은 모듈(`roadmap-window`)로 전환 필요
- 막대 드래그로 기간 조절은 미구현 (헤더 문구만 존재)
- 세로 스크롤은 페이지 전체 스크롤에 의존 — 행이 많아지면 타임라인 헤더 sticky도 검토 필요
