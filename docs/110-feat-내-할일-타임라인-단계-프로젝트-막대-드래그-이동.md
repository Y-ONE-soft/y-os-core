# 110. feat: 내 할일 타임라인 단계·프로젝트 막대 드래그 이동

## 작업 요약

docs/99에서 단계 행을 넣었지만 단계 막대는 클릭만 됐다. **단계와 프로젝트 막대를 끌어서 옮길 수 있게** 하고, 끌면 아래 계층이 함께 움직이도록 했다.

| 대상 | 동작 |
| --- | --- |
| 단계 막대 이동 | 단계 날짜 + **그 단계 할일의 예정일도 같은 일수만큼** 이동 |
| 단계 막대 양 끝 | 기간만 조절, 할일은 그대로 |
| 프로젝트 막대 이동 | 소속 **모든 단계 + 모든 할일**(백로그 포함) 같은 일수만큼 이동 |
| 프로젝트 막대 양 끝 | 조절 불가 |

## 왜 이 규칙인가

### 단계를 끌면 할일도 따라간다

계층 뷰에서 단계만 옮기고 할일이 제자리에 남으면 할일이 단계 밖으로 튀어나온다. 프로젝트 상세 로드맵은 단계 아래에 할일을 펼치지 않아 문제가 안 됐지만, 이 화면은 단계 아래에 할일이 붙어 있어 **블록이 통째로 움직여야** 말이 된다.

기간 조절(양 끝 손잡이)은 다르다. 단계를 늘리거나 줄이는 것은 할일 일정을 바꾸겠다는 뜻이 아니므로 단계만 바꾼다.

**이동과 기간 조절은 커밋 값으로 구분한다** — 시작일과 종료일이 같은 일수만큼 밀렸으면 이동, 아니면 기간 조절이다. `RoadmapBar`가 모드를 알려주지 않으므로 결과에서 되짚는다.

### 프로젝트 막대는 이동만

프로젝트 막대의 기간은 소속 단계·할일 날짜에서 계산한 **파생값**이다. 양 끝을 늘렸을 때 어느 단계가 늘어나야 하는지 정의되지 않는다. 그래서 손잡이를 없애고 통째 이동만 허용한다.

## 변경 내용

### `src/components/features/projects/roadmap-bar.tsx`

- `resizable?: boolean` prop 추가 (기본 `true`)
- `false`면 양 끝 손잡이를 렌더하지 않는다 — 본체 드래그(이동)는 그대로 동작

기본값이 `true`라 기존 사용처(프로젝트 상세 로드맵·작업 현황 로드맵)의 동작은 바뀌지 않는다.

### `src/components/features/my-work/my-work-timeline-panel.tsx`

**`moveStage(projectId, stage, patch)`**

1. `updateStage`로 단계 날짜 반영
2. 시작일 변화량(`startDelta`)이 0이면 종료 — 종료일만 끈 경우다
3. 종료일 변화량이 시작일과 다르면 종료 — 기간 조절이므로 할일은 건드리지 않는다
4. 같으면 이동이므로 그 단계 할일의 `scheduledDate`를 `shiftISO`로 같이 옮긴다

**`moveProject(projectId, delta)`**

소속 단계(시작일 있는 것)와 그 할일, 백로그 할일까지 전부 `delta`만큼 옮긴다. 종료일이 없는 단계는 열린 채로 이동한다.

프로젝트 막대의 `onCommit`에서 `dayOffset(patch.startDate, marks[0])`로 이동 일수를 구해 넘긴다.

**막대 연결**

- 단계 막대에 `onCommit` 추가, 툴팁을 `클릭하면 단계 상세, 끌면 할일까지 함께 이동`으로
- 프로젝트 막대에 `resizable={false}` + `onCommit` 추가, 툴팁 `끌면 단계·할일이 함께 이동`

## 알려진 이슈 / 후속

- **낙관적 업데이트가 항목마다 개별 호출이다.** 단계 하나에 할일이 여러 개면 `updateTask`가 그만큼, 프로젝트 이동이면 전체 항목 수만큼 호출된다. `boardActions`가 항목 단위 API라 지금 구조에서는 불가피하다. 한 프로젝트 규모에서는 문제되지 않지만, 일괄 이동 API(`PATCH /api/admin/projects/[id]/shift` 같은)로 묶는 것이 다음 단계다.
- 이동 중 미리보기는 끌고 있는 막대에만 나타난다. 아래 계층(할일 막대)은 손을 뗀 뒤에 움직인다 — `RoadmapBar`의 미리보기가 자기 막대 안에서만 동작하기 때문이다.
- 프로젝트 막대를 끌면 **날짜가 없는 항목은 그대로**다. 없던 일정이 생기지는 않는다.
