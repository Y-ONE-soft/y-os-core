이전에는 문자열 상수 3개(`"🔍 필터"`, `"담당자 1 ▾"`, `"프로젝트 ▾"`)를 버튼으로 그린 **자리표시**였다. 눌러도 아무 일도 일어나지 않았고 숫자 `1`도 하드코딩이었다.

## 3. 설계 결정

**모듈 스토어를 쓴 이유** — `MyWorkPage`는 서버 컴포넌트라 클라이언트 상태를 만들어 props로 내려줄 수 없다. 필터 바만 클라이언트로 떼고, 캘린더·타임라인 패널이 같은 모듈 스토어를 구독하게 했다. `useSyncExternalStore` 기반이라 한 곳에서 바꾸면 양쪽 뷰가 함께 갱신된다.

**빈 선택 = 제한 없음** — 체크를 모두 풀면 `null`로 되돌려 "아무것도 안 보이는" 상태를 만들지 않는다.

**담당자 기본값은 로그인 사용자** — "내 할일"의 원래 기준(내가 작업자인 프로젝트)이 기본이고, 담당자를 고르면 그 사람들 기준으로 바뀐다. 요약 라벨도 미선택 시 `담당자 <내 이름>`으로 기본값을 그대로 드러낸다.

| 필터 | 대상 | 비고 |
|---|---|---|
| 담당자 | 실제 사용자(`GET /api/admin/users`) | 프로젝트의 `ownerId`로 판정 — 현재 담당 개념이 프로젝트 소유자뿐이다 |
| 프로젝트 | 전체 프로젝트 | 선택 시 그 프로젝트만 |

두 필터는 **AND**로 걸린다. 선택이 있으면 `초기화` 버튼이 나타난다.

## 5. 알려진 사항 / 후속 과제

- **담당자 = 프로젝트 소유자**다. 할일 단위 담당자 필드가 없어서 프로젝트 `ownerId`로 판정한다. 할일별 담당 개념이 생기면 이 필터의 기준도 바꿔야 한다.
- **백로그 패널에는 필터가 적용되지 않는다.** 필터 바 옆 안내대로 "뷰에만 적용 · '내 할일'과 독립"이라 캘린더·타임라인만 좁힌다.
- `🔍 필터` 버튼은 되살리지 않았다 — 무엇을 여는 버튼인지 정의된 적이 없다. 검색·조건 추가가 필요해지면 그때 설계한다.
