# 117. feat: 담당자 보드 카드에 상태 라벨 표시

요청 사이클 `담당자-기본값-상태-라벨`의 **태스크 2/2**.

> 사용자 요청: "상태는 할 일, 완료 이 두개로 보여지면됨" (작업현황 > 담당자 뷰)

담당자 보드의 카드는 상태를 **체크박스와 취소선으로만** 구분했다. 글자로 명시된
상태가 없어 훑어볼 때 바로 읽히지 않았다.

## 상태는 둘뿐

카드 오른쪽에 `할 일` / `완료` 칩을 붙였다. `done` 하나에서 파생하므로 체크박스와
항상 같은 값을 가리킨다 — 별도 상태 필드를 만들지 않았다.

```tsx
{card.task.done ? "완료" : "할 일"}
```

## 지연은 상태가 아니라 색으로

docs/110에서 예정일이 지난 미완료 할일에 **`지연` 배지**를 따로 붙여 뒀었다.
그대로 두면 카드 하나에 `지연`과 `할 일`이 나란히 붙어 상태가 셋처럼 보인다.
요구가 "두 개"이므로 배지를 없애고, **같은 칩을 경고색으로 칠하는 방식**으로 옮겼다.

```tsx
{/* 상태는 할 일 / 완료 둘뿐이다. 예정일이 지난 할일은 상태를
    늘리는 대신 같은 칩을 경고색으로 칠해 알린다 */}
<span
  title={card.overdue ? `예정일 ${card.task.scheduledDate} — 지났습니다` : undefined}
  className={cn(
    "shrink-0 rounded-full border px-1.5 py-px text-[10.5px] font-medium",
    card.task.done
      ? "text-muted-foreground"
      : card.overdue
        ? "border-destructive/40 text-destructive"
        : "text-foreground",
  )}
>
  {card.task.done ? "완료" : "할 일"}
</span>
```

정리하면 이렇다. 상태 값은 둘, 시각 변형은 셋이다.

| 상태 | 칩 | 색 |
| --- | --- | --- |
| 할 일 | `할 일` | 기본 |
| 할 일 + 예정일 지남 | `할 일` | 경고색 + 경고 테두리, `title`에 지난 예정일 |
| 완료 | `완료` | 흐림 (카드 자체도 이미 흐려진다) |

컬럼 머리의 수치·부하 막대는 그대로다 — 지연이 있으면 여전히 경고색이라 컬럼
단위로도 눈에 띈다.

| 모든 카드에 상태 칩이 붙음 | PASS |

| 미완료 카드는 `할 일` | PASS |
| 완료 카드는 `완료` | PASS |
| 지연 카드는 상태 칩이 경고색 | PASS |
| 지연 아닌 미완료는 기본색 | PASS |
