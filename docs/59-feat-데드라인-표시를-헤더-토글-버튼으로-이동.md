# 59. 데드라인 표시를 헤더 토글 버튼으로 이동

- **예정 커밋 메시지**: `feat: 데드라인 표시를 헤더 토글 버튼으로 이동`
- **작업일**: 2026-07-22
- **작업 브랜치**: `데드라인-표시-체크박스` (워크트리 `.claude/worktrees/deadline-toggle`, base: main `39d415c`)

---

## 1. 작업 내용 요약

단계 오버레이의 **데드라인 표시**를 사이드 패널 구석의 스위치에서 **헤더 우측의 체크 토글 버튼**으로 옮겼다.

사용자 요청 경과: "체크박스처럼 잘 보이게" → 이어서 **"할일의 이슈 등록 같은 배치로"** 지정. 작업 상세 오버레이 헤더 우측의 `🐞 이슈 등록` 버튼과 **같은 자리·같은 규격**으로 배치했다.

| | 이전 | 이번 |
|---|---|---|
| 위치 | 사이드 패널 하단, 기간 필드 아래 | **헤더 우측** (작업 상세의 이슈 등록 자리) |
| 형태 | shadcn `Switch` + 2줄 라벨 | **`Button variant="outline"` 토글** — `☑ 데드라인 표시` / `☐ 데드라인 표시` |
| 상태 표시 | 스위치 on/off | 체크 글리프 + **켜짐일 때 primary 강조**(테두리·배경·글자) |
| 접근성 | `Switch` 기본 | `aria-pressed` + 설명 `title` |

- 두 오버레이(**단계 추가 / 단계 상세**) 모두 동일하게 적용해 생성·수정 경험을 맞췄다.
- 사이드 패널의 기존 데드라인 행은 제거해 중복을 없앴다(설명 문구는 버튼 `title`로 이전).

## 2. 주요 결정

| 항목 | 결정 | 이유 |
|---|---|---|
| 체크 표현 | 이모지 글리프 `☑`/`☐` | 작업 상세의 `🐞 이슈 등록`과 같은 "이모지 + 라벨" 패턴. 별도 아이콘 임포트 없이 상태가 한눈에 |
| 켜짐 강조 | `border-primary bg-primary/10 text-primary` | 아웃라인 버튼만으로는 on/off가 약해 보인다. 브랜드 컬러로 활성 상태를 분명히 |
| 컴포넌트 | `Switch` 제거하고 `Button` 사용 | 헤더의 다른 액션(이슈 등록·닫기)과 같은 높이·라운드로 정렬되어야 함 |
| 추가 오버레이 헤더 | `justify-between`으로 바꿔 우측 배치 | 기존에는 좌측 정렬만 있었음. 상세와 같은 레이아웃 규칙 적용 |

### 검증 중 오탐 1건

- 1차 테스트에서 "새로고침 후 유지: false"가 나왔으나, **프로젝트에 단계가 4개**여서 새로고침 뒤 테스트가 `[0]` 인덱스로 **다른 단계**를 연 것이 원인이었다. 단계 id를 고정해 API로 직접 확인하니 저장은 정상. 앱 결함 아님.

## 5. 참고 — 병렬 세션 조율

- 사용자가 함께 요청한 **단계 삭제**는 다른 세션(`보드-단계-우클릭-삭제`)이 이미 서버·API·클라이언트 3계층을 구현 중이어서 **이번 작업에서 제외**했다(사용자 지시로 보류 확정). 이 커밋은 데드라인 배치만 다룬다.

## 6. 알려진 사항

- 데드라인을 끄면 로드맵·보드에서 기간 막대와 기간 라벨이 사라진다(기존 동작 유지). 버튼 `title`로 그 인과를 설명한다.
