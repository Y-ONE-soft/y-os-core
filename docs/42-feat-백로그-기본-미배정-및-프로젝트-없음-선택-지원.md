## 1. 작업 요약

"내 작업에서 만들면 일단 프로젝트 없음, 원칙적으로 백로그에는 프로젝트 없음으로 있어야 한다"를 화면에 반영했다.

- 내 작업 백로그의 입력창으로 만든 작업은 **어떤 프로젝트에도 배정되지 않는다** (이전엔 첫 프로젝트 백로그로 들어갔다)
- 백로그 목록이 **미배정 작업 + 전 프로젝트 백로그**를 함께 보여준다 (미배정이 위)
- 행의 드롭다운 라벨과 작업 티켓 모두에서 **"프로젝트 없음"을 고를 수 있다** — 배정 해제도 가능

## 2. 변경 내용

### 2-1. 스토어: 미배정을 1급 상태로 (`board-store.tsx`)

| 액션 | 변경 |
|---|---|
| `addUnassignedTask(name)` | **신규** — `projectId: null`로 생성. 내 작업 백로그의 기본 생성 경로 |
| `assignTask(from, taskId, to, stageId)` | `from`·`to` 프로젝트를 **nullable**로 확장. 미배정 → 프로젝트/단계, 프로젝트 → 미배정 양방향 지원 |
| `toggleTask(projectId, …)` | projectId nullable — 미배정 작업의 완료 토글 |
| `updateTask(projectId, …)` | projectId nullable — 미배정 작업의 이름·내용 수정 |

미배정 작업은 `boards[projectId]`가 아니라 워크스페이스의 `unassigned` 배열에 있으므로, 낙관적 업데이트도 그 배열을 직접 갱신한다.

### 2-2. 백로그 패널 (`my-work-backlog.tsx`)

- 입력창 → `addUnassignedTask` 호출. **프로젝트 선택 없이 항상 활성**(이전엔 프로젝트가 없으면 비활성)
- 목록 = `미배정 작업 + 전 프로젝트 백로그`
- 라벨: 미배정이면 **"프로젝트 없음"**(속이 빈 점), 배정됐으면 기존대로 `프로젝트 · 백로그`(색 점)
- 드롭다운 **맨 위에 "프로젝트 없음" 항목** 추가 — 현재 미배정이면 체크 표시

### 2-3. 작업 티켓 (`task-detail-overlay.tsx`)

- 위치 해석에 **미배정 작업 탐색**을 추가 — `unassigned`를 먼저 보고, 없으면 기존대로 보드를 훑는다
- 헤더 브레드크럼: 미배정이면 `프로젝트 없음 · 미배정`
- 프로젝트 Select에 **"프로젝트 없음"** 옵션 추가 (센티널 `__unassigned__`)
- 단계 Select는 미배정일 때 **비활성 + "프로젝트를 먼저 선택하세요"** — 프로젝트 없는 작업이 단계에 들어가는 모순을 UI에서 차단 (서버도 같은 규칙을 강제한다, docs/41 3절)

## 5. 알려진 사항 / 후속 과제

- 예정일 자동 계산은 태스크 3에서 — 지금은 단계를 지정해도 `scheduledDate`가 비어 있다
- 캘린더는 여전히 자리표시 상수라 미배정/예정일이 화면에 반영되지 않는다 (이번 요청 범위 밖)
- 프로젝트 상세 화면의 백로그(`project-backlog.tsx`)는 그 프로젝트 소속 백로그만 다루므로 이번 변경과 무관 — 미배정 작업은 내 작업 페이지에서만 보인다
