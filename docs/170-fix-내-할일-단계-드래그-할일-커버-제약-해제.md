## 1. 작업 내용 요약

내 할일 캘린더에서 **단계 막대에 할일이 있으면 시작/끝 가장자리를 할일 너머로 끌 수 없던 제약을 없앴다.** 이제 단계 막대는 자유롭게 리사이즈되고, 할일은 제자리에 남는다(독립).

기존 모델은 "단계는 항상 자기 할일을 덮어야 한다"였다(`clampStageToTasks`). 그래서 시작 손잡이를 첫 할일보다 늦게 끌면 그 지점에서 멈췄다 — 사용자에겐 "할일이 있으면 드래그가 안 되는" 것으로 보였다.

새 모델: **단계 막대와 할일은 독립.**
- 단계 시작/끝 가장자리는 할일에 걸리지 않고 자유롭게 움직인다.
- 할일은 제자리에 남는다 — 단계 밖으로 밀려나도 된다.
- 단계 밖으로 나간 할일도 **프로젝트 바 안에는 유지된다**(레이아웃이 단계+할일 합집합으로 계산 — 기존 그대로).

## 2. 사용자 결정 반영

두 가지를 확인받고 진행했다:
- **단계 시작 가장자리 드래그 시 배정 할일** → "할일은 제자리, 막대만 자유로" (커버 제약 제거, 할일 미이동)
- (프로젝트 바 드래그는 태스크 ②)

## 3. 구현

`my-work-calendar-panel.tsx` 두 곳에서 커버 강제를 제거했다.

| 지점 | 변경 전 | 변경 후 |
|---|---|---|
| `nextStageDates` (단계 리사이즈) | `mode==="move" ? dragged : clampStageToTasks(dragged, taskDateRange(tasks))` | 항상 `dragged` — clamp 없음 |
| 할일 드래그 커밋 (`handleDrag` task 분기) | 할일이 단계 밖으로 나가면 `clampStageToTasks`로 단계를 늘려 덮음 → `updateStage` | 할일만 이동, 단계는 건드리지 않음 |

- 단계 **이동(move)** 은 종전대로 자기 할일을 함께 옮긴다(막대 본체를 통째로 잡는 제스처 — 요청 대상 아님). 변경한 것은 **가장자리 리사이즈**와 **할일 개별 드래그**뿐이다.
- `clampStageToTasks`·`taskDateRange` import가 이 파일에서 미사용이 되어 제거(다른 파일에서는 계속 쓰임 — roadmap-utils 원본 유지).

## 4. 검증

1. `npm run lint` ✓ · `npm run build` ✓
2. **브라우저 실제 포인터 드래그 검증** (puppeteer, master01, dev 3018) — 칸 폭을 DOM에서 실측해 정확히 이동. **자동 체크 7건 ALL PASS**

   검증 데이터: 단계 07-13~07-27, **완료 할일** 07-15(미완료면 '오늘'로 이월 저장되어 clamp 경계 검증이 무의미해지므로 done 할일 사용). 시작 손잡이를 오른쪽으로 ~5일 끌었다.

   | 항목 | 결과 |
   |---|---|
   | 시작일 07-13 → **07-18** (뒤로 이동) | ✓ clamp 안 걸림 |
   | **시작일이 할일(07-15)을 넘어섬** | ✓ (예전엔 07-15에서 멈췄음) |
   | 할일 예정일 07-15 그대로 | ✓ 제자리 |
   | 종료일 07-27 불변 | ✓ |
   | 페이지 에러 | 0 |

3. 프로젝트 바 합집합(req 3)은 레이아웃(`collectProjectRanges`, 미변경)이 단계+할일 최소~최대로 계산하므로 유지 — 07-18~07-27 단계 밖의 07-15 할일도 프로젝트 바 안에 남는다.
4. 검증용 프로젝트·단계·할일은 DB에서 삭제(잔여 0건)

### 검증 함정 메모

처음엔 미완료 할일 07-10을 심었는데 **이월 로직이 scheduledDate를 DB에서 오늘(07-24)로 바꿔 저장**해, 드래그가 clamp 경계(07-24)를 실제로 넘지 못한 채 통과처럼 보였다(위양성). 이월이 적용되지 않는 **완료(done) 할일**로 바꿔 clamp를 제대로 넘겨 재검증했다. 날짜 기반 드래그 검증은 이 이월 저장을 반드시 고려해야 한다.

## 5. 알려진 사항 / 후속

- 단계 **이동**은 여전히 할일을 함께 옮긴다(의도 — 본체 제스처). 이것도 분리하길 원하면 별도 조정
- 프로젝트 바 자체를 드래그(이동·리사이즈)하는 기능은 **태스크 ②**에서 스키마(`Project.startDate/endDate`)와 함께 구현
