# fix: 캘린더 마감 라벨을 실제 종료일에만 표시

## 작업 요약

내 할일 캘린더에서 **마감 라벨이 프로젝트의 실제 종료일이 아닌 날에 떴다.** 특히 일 뷰에서 "오늘 마감이 아닌데도 마감이 뜬다"는 형태로 드러났다. 원인은 배치 계산이 그리드에 잘린(클램프된) 데이터로 "그리드 밖으로 이어지는가"를 판정했기 때문이다. 소스가 **비클램프 기간**을 넘기도록 고쳤다.

## 버그 — 클램프된 데이터로 이어짐 여부를 판정

프로젝트 박스는 오른쪽으로 더 이어지지 않는 조각(`!box.continuesRight`)에 마감 라벨을 붙인다 ([`my-work-calendar.tsx:145`](../src/components/features/my-work/my-work-calendar.tsx#L145)).

`continuesRight`는 배치 모듈에서 이렇게 계산됐다 ([`my-work-calendar-layout.ts:268`](../src/components/features/my-work/my-work-calendar-layout.ts#L268)).

```ts
continuesRight: projectRange.end > weekEnd
```

문제는 `projectRange`가 **그리드에 클램프된 오버레이**에서 나온다는 것이다(`collectProjectRanges`). 단계 막대는 그려질 때 그리드 밖 부분이 잘린다(`stageSegments`의 `end = Math.min(totalDays - 1, rawEnd)`). 그래서 그리드 밖으로 이어지는 프로젝트도 `projectRange.end`가 그리드 끝으로 잘려, `projectRange.end > weekEnd`가 **false**가 되고 마감 라벨이 그리드 끝에 잘못 붙었다.

**일 뷰(1칸)에서 가장 심하게 드러난다.** 오늘을 지나가는 프로젝트(어제 시작~모레 종료)의 단계가 오늘 1칸으로 잘리면 `projectRange.end`가 오늘 다음 칸이 되고, 그게 곧 `weekEnd`라 `continuesRight=false` → 오늘이 마감이 아닌데 마감이 뜬다.

**같은 결함이 주·월 뷰의 그리드 오른쪽 끝에도 잠재했다.** 다음 달로 이어지는 프로젝트가 이번 달 마지막 날에 마감으로 표시됐다.

단계 배지(`stageSegments`의 `deadline: … && rawEnd === end`, [source:78](../src/components/features/my-work/my-work-calendar-source.ts))는 **비클램프 `rawEnd`를 확인**하기 때문에 멀쩡했다. 프로젝트 라벨만 클램프된 값을 봤던 것이 불일치의 핵심이다.

## 재현 — 실제 모듈로 회귀 테스트

수정 전, 실제 `buildCalendarSource` + `buildWeekLayouts`를 import한 스크립트로 5개 케이스를 돌려 버그를 고정했다. 판정은 **실제 렌더 조건과 동일**하게 "프로젝트 박스 조각 중 하나라도 `!continuesRight`면 마감이 뜬다"로 확인했다.

| 케이스 | 마감 떠야 하나 | 수정 전 | 수정 후 |
| --- | --- | --- | --- |
| 일 뷰: 오늘을 지나가는 프로젝트 | 아니오 | **뜸(버그)** | 안 뜸 |
| 일 뷰: 오늘이 실제 종료일 | 예 | 뜸 | 뜸 |
| 주 뷰: 이 주를 지나가는 프로젝트 | 아니오 | **뜸(버그)** | 안 뜸 |
| 월 뷰: 다음 달로 이어지는 프로젝트 | 아니오 | **뜸(버그)** | 안 뜸 |
| 월 뷰: 이 달에 끝나는 프로젝트 | 예 | 뜸 | 뜸 |

일·주·월 세 뷰 모두 그리드 오른쪽 끝에서 버그가 재현됐다. 통제 케이스(실제로 그 안에서 끝나는 경우)는 수정 전후 모두 정상이라 회귀가 없음을 확인했다.

## 수정 — 소스가 비클램프 기간을 넘긴다

### `my-work-calendar-source.ts`

- `DaySpan` 타입(그리드-일자 범위, `end` 배타적, **클램프 안 함** → 음수·초과 가능)과 `CalendarSource.projectSpans`를 추가했다.
- 오버레이를 만들며 프로젝트별 비클램프 기간을 누적한다. 단계는 `rawStageSpan`(그리드 밖까지의 실제 범위), 할일은 예정일 하루. 그리드에 걸치는(잘려서라도 보이는) 것만 반영한다.

```ts
// 단계: 잘리기 전 실제 범위
const raw = rawStageSpan(grid, stage);
if (raw && inGrid(raw)) extendSpan(projectSpans, project.id, raw);
```

### `my-work-calendar-layout.ts`

- `buildWeekLayouts`에 `projectSpans?` 인자를 추가했다. 넘기면 그것으로, 없으면 기존 `collectProjectRanges`(클램프 기반)로 폴백한다.
- `projectSpans`는 객체(`Record`)이고 기존 로직은 `Map`을 쓰므로 `new Map(Object.entries(...))`로 정규화한다.

박스의 보이는 col·span은 여전히 그리드로 클램프된다(`clippedStart/End`의 `max`/`min`). **`continuesLeft/Right`만** 비클램프 기간으로 판정이 정확해진다.

### `my-work-calendar-panel.tsx`

`buildWeekLayouts(...)` 호출에 `source.projectSpans`를 넘긴다.

## 곁들여 고쳐진 것 — 프로젝트 이름 라벨

이름 라벨은 왼쪽으로 이어지지 않는 조각(`!box.continuesLeft`)에만 붙는다. `continuesLeft`도 같은 방식으로 클램프된 값을 봤기 때문에, 그리드 이전에 시작한 프로젝트가 그리드 첫날에 이름을 다시 표시하는 문제가 있었다. `continuesLeft`도 비클램프 기간을 쓰게 되어 함께 정확해졌다.

## 알려진 이슈

- 폴백 경로(`collectProjectRanges`)는 여전히 클램프 기반이라, `projectSpans` 없이 `buildWeekLayouts`를 부르면 옛 버그가 남는다. 현재 유일한 호출부인 패널은 항상 `projectSpans`를 넘기므로 실사용 경로는 안전하다.
- 이 커밋은 마감 라벨 표시 조건만 고친다. 일 뷰에서 지나가는 단계 **막대 자체**를 숨기는 것은 다음 태스크(일 뷰 시간대 전환)에서 다룬다.
