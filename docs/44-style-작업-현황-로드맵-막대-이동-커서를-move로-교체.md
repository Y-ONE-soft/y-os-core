# style: 작업 현황 로드맵 막대 이동 커서를 move로 교체

## 작업 요약

작업 현황(`/projects`) 로드맵 단계 막대를 이동할 때 뜨던 **손 모양 커서**(`cursor-grab` → `active:cursor-grabbing`)를 **`cursor-move`(✥)** 로 바꿨다.

docs/42에서 드래그 기능을 넣을 때 관례적으로 `grab`/`grabbing`을 썼는데, 사용자 검토에서 손 아이콘이 투박하다는 지적을 받아 교체했다.

## 선택 이유

`grab`/`grabbing`은 **"물건을 집어 든다"** 는 은유다. 카드를 통째로 들어 옮기는 칸반 보드나 캔버스 패닝에는 맞지만, 타임라인에 붙어 좌우로만 미끄러지는 18px짜리 간트 막대에는 은유가 과하다. 커서 모양도 벌린 손 → 주먹으로 두 단계 변해 시선을 끈다.

후보를 놓고 비교했다.

| 후보 | 판단 |
| --- | --- |
| `cursor-move` (✥) | **채택.** "이 요소는 옮길 수 있다"는 가장 직접적인 표준 커서. MS Project·dhtmlxGantt 등 간트 도구의 관례 |
| `cursor-ew-resize` (↔) | 제외. 양 끝 손잡이가 이미 쓰고 있어 **이동과 리사이즈가 구분되지 않는다** |
| `cursor-col-resize` | 제외. 위와 같은 이유 + 열 경계를 끄는 의미라 막대 이동과 맞지 않는다 |
| 커스텀 SVG 커서 (data URI) | 제외. 핫스팟 지정·HiDPI 흐림·폴백 처리가 따라붙는데, 커서 하나 다듬자고 감수할 비용이 아니다 |

`move`가 4방향 화살표라 상하 이동도 되는 것처럼 읽힐 수 있다는 점은 인지하고 있다. 다만 막대는 행에 고정돼 있어 오해 여지가 크지 않고, 좌우 전용 표준 커서(`ew-resize`)는 손잡이와 충돌해서 쓸 수 없다. **이동 = `move`(✥) / 리사이즈 = `ew-resize`(↔)** 라는 구분을 지키는 쪽을 택했다.

## 변경 파일

### `src/components/features/projects/workload-roadmap.tsx`

`Bar`의 `className` 한 줄만 바뀐다.

```diff
- draggable && "cursor-grab touch-none select-none active:cursor-grabbing",
+ draggable && "cursor-move touch-none select-none",
```

- `active:cursor-grabbing`이 함께 사라졌다. `move`는 누르는 동안 모양이 변하지 않는 커서라 상태 분기가 필요 없다. 드래그 중 시각 피드백은 기존 `draft && "z-10 shadow-sm"`(막대를 이웃 위로 띄우고 그림자)이 그대로 담당한다.
- `touch-none`·`select-none`은 유지했다. 각각 터치 드래그 시 가로 스크롤 방지, 드래그 중 텍스트 선택 방지 목적이라 커서와 무관하다.
- 선택 이유를 코드 주석으로 남겨 다음 사람이 다시 `grab`으로 되돌리지 않게 했다.

양 끝 손잡이의 `cursor-ew-resize`는 **변경 없다.**

`grep`으로 `src/` 전체에 `cursor-grab`/`cursor-grabbing` 잔존이 없음을 확인했다(이 두 곳이 유일한 사용처였다).
