# 55. 내 작업 캘린더에 할일 칩 표시

- **예정 커밋 메시지**: `feat: 내 작업 캘린더에 할일 칩 표시`
- **작업일**: 2026-07-22
- **작업 브랜치**: `캘린더-드래그` (워크트리 `.claude/worktrees/calendar-drag`, base: main `36d6641`)
- **요청 사이클 태스크 1/4** — 캘린더에서 프로젝트·단계·할일을 드래그로 조작하는 요청의 첫 단계. 이후 단계 드래그(2) → 할일 드래그(3) → 프로젝트 기간 필드·박스 드래그(4)가 이어진다.

---

## 1. 작업 요약

예정일(`Task.scheduledDate`, docs/49)이 잡힌 할일을 캘린더의 해당 날짜 칸에 **칩으로 표시**한다. 할일을 드래그해 옮기려면 먼저 화면에 있어야 하므로 이번 요청의 전제 작업이다.

렌더러(`my-work-calendar.tsx`)와 배치 모듈(`my-work-calendar-layout.ts`)에는 **이미 `kind: "task"` 칩 처리와 할일 레인 계산이 준비돼 있었다**(다른 세션이 "Task에 예정일이 생기면 여기서 함께 만든다"는 주석과 함께 열어둔 자리). 이번 변경은 소스 빌더가 그 오버레이를 만들어 주는 것뿐이다.

## 2. 변경 내용

`buildCalendarSource`에 `taskChip()`을 추가하고, 프로젝트를 훑을 때 단계 막대와 함께 할일 칩을 만든다.

| 항목 | 규칙 |
|---|---|
| 대상 | `scheduledDate`가 있는 할일 |
| 위치 | 예정일 칸 하루짜리(`span: 1`) |
| 색 | **소속 단계 색** — 어느 단계의 할일인지 한눈에 보이게 |
| 그리드 밖 | 보고 있는 달 범위를 벗어나면 만들지 않는다 |
| 백로그 | 예정일이 없는 게 정상이지만(단계를 벗어나면 해제, docs/51) 방어적으로 함께 훑고, 있으면 프로젝트 색으로 표시 |

`placed` 판정에 할일도 포함시켜, **단계 막대는 이 달에 없지만 할일만 있는 프로젝트도 박스가 그려지도록** 했다.

## 5. 알려진 사항 / 후속 과제

- 칩은 아직 **읽기 전용** — 드래그 이동은 태스크 3
- 할일 칩에는 클릭 동작이 없다(단계 막대는 상세 오버레이가 열린다). 칩 클릭 → 작업 상세 연결은 이 요청 범위 밖이며 필요하면 별도 태스크
- 검증용으로 만든 프로젝트·단계·할일은 요청 사이클 마지막에 정리한다 (드래그 태스크에서 계속 사용)
