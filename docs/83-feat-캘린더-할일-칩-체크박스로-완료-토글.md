## 1. 작업 내용 요약

내 할일 캘린더의 할일 칩에서 **체크박스를 누르면 그 자리에서 완료가 되도록** 했다. 기존에는 칩 어디를 눌러도 상세 오버레이가 열려, 완료 하나 체크하려면 상세를 열고 → 체크하고 → 닫아야 했다.

- **체크박스 영역** → 완료 토글 (상세는 열리지 않음)
- **나머지 영역(제목)** → 종전대로 상세 오버레이
- 다시 누르면 완료 해제. 완료 시 취소선·흐린 색은 기존 표시 그대로

## 2. 구현 메모

### 버튼 안에 버튼을 넣을 수 없다

기존 칩은 `<button>` 하나가 칩 전체였고 체크박스는 그 안의 `aria-hidden` 장식 `<span>`이었다. 버튼 중첩은 유효하지 않은 HTML이라, 칩을 **컨테이너 `<div>` + 버튼 2개**(체크박스 / 제목)로 나눴다.

### 드래그와 충돌하지 않게

칩은 드래그로 날짜를 옮길 수 있다. 체크박스에서 시작한 포인터가 드래그로 넘어가면 *칩을 옮기려다 완료가 눌리거나, 체크하려다 일정이 움직이는* 사고가 난다. 체크박스에서 `onPointerDown`·`onClick` 모두 `stopPropagation()`으로 끊었다 — 드래그 시작은 제목 버튼에만 걸린다.

### 접근성

장식용 `aria-hidden` 스팬을 **`role="checkbox"` + `aria-checked` + `aria-label`("〈할일명〉 완료")** 버튼으로 바꿨다. 키보드 포커스·스크린리더로도 완료 조작이 가능해졌고, 검증 스크립트도 이 역할을 기준으로 칩을 찾는다.

### 어느 단계의 할일인지 찾기

`boardActions.toggleTask(projectId, stageId, taskId)`는 위치를 요구하는데 캘린더 오버레이에는 `taskId`만 있다. 패널이 자기 프로젝트의 단계·백로그를 훑어 위치를 찾아 넘기도록 `handleToggleTask`를 추가했다(기존 `nextTaskSchedule`과 같은 방식). 단계에 속하지 않은 백로그 할일도 함께 처리한다.

- 완료 시각(`completedDate`)은 서버가 `done` 전환에 맞춰 자동 기록한다(기존 규약) — 이 경로에서도 동일하게 동작
