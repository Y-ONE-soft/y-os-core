## 1. 작업 내용 요약

단계 상세 오버레이에서 **그 단계에 속한 할일들을 지라(에픽→하위 이슈)처럼 목록으로** 볼 수 있게 했다. 목록에서 완료 토글·상세 열기·할일 추가까지 된다.

기존에는 단계 상세에 내용·산출물·연결·댓글만 있고 정작 그 단계의 할일은 보이지 않았다. 데이터(`stage.tasks`)는 이미 있어 UI만 붙였다.

## 3. 구현

**위치**: 본문 컬럼의 `내용` 바로 아래(산출물 위). 지라 에픽이 설명 다음에 하위 이슈를 두는 순서와 같다.

**섹션 구성**:
- 헤딩 `할일 · {완료}/{전체}` — 진행도를 한눈에.
- 각 할일 행: 체크박스(완료 토글) + 이름 버튼. 완료면 취소선.
- 빈 단계면 "이 단계에 할일이 없습니다."
- 하단에 **항상 보이는 입력**(`＋ 할일 이름 입력 후 Enter`) — 버튼을 눌러 열 필요 없이 바로 쳐서 Enter로 추가하고, 추가 후 입력을 비워 연속 입력이 편하다. 내 할일 백로그와 같은 방식.

## 3-1. 할일 추가 UX 통일 (직접 입력)

리뷰 피드백으로, **"버튼을 눌러 입력을 여는" 방식을 없애고 "상시 보이는 입력에 바로 쳐서 Enter"**로 통일했다. 두 곳을 바꿨다.

| 위치 | 이전 | 이번 |
|---|---|---|
| 단계 상세 할일 섹션 | `＋ 할일 추가` 버튼 클릭 → 입력 등장 | 상시 입력, 바로 Enter |
| 프로젝트 보드 컬럼 | `＋ 할일` 버튼 클릭 → 입력 등장(`addingStageId`) | 상시 입력, 바로 Enter |

내 할일 백로그는 이미 상시 입력이라 그대로 두고, 프로젝트 쪽(보드·단계 상세)을 여기에 맞췄다. 보드 컬럼은 전체가 클릭 대상(단계 상세 열기)이라 입력 클릭은 `stopPropagation`으로 막았다. 쓰이지 않게 된 `addingStageId` 상태도 제거했다.

**동작 배선** — 새 액션을 만들지 않고 기존 보드 액션을 그대로 썼다:
- 완료 토글: `boardActions.toggleTask(projectId, stage.id, task.id)`
- 추가: `boardActions.addTask(projectId, stage.id, name)`
- 상세 열기: 행 이름을 누르면 `<TaskDetailOverlay taskId onClose>`를 **단계 상세 위에 겹쳐** 띄운다. 닫으면 단계 상세로 복귀. (이미 `CollaboratorRequestDialog`를 겹쳐 띄우던 것과 같은 중첩 Dialog 패턴. `TaskDetailOverlay`는 `taskId`만으로 스토어에서 위치를 찾아 독립 동작한다.)

## 5. 알려진 사항 / 후속 과제

- **할일 상세를 겹쳐 여는 방식**이라 배경에 단계 상세가 남는다. 지라처럼 "패널 교체"가 필요하면 별도 사이클로 네비게이션을 손봐야 한다.
- 목록에 담당자·유형·난이도 같은 메타는 표시하지 않는다 — 이름·완료만. 필요해지면 지라 하위 이슈 행처럼 칼럼을 늘릴 수 있다.
- 정렬은 스토어의 할일 순서를 그대로 따른다(생성순).
