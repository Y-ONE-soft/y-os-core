# 96. 작업 현황 캘린더 뷰 구현

- **예정 커밋 메시지**: `feat: 작업 현황 캘린더 뷰 구현`
- **작업일**: 2026-07-22
- **작업 브랜치**: `작업현황-캘린더-뷰` (워크트리 `.claude/worktrees/status-calendar`)

## 1. 작업 내용 요약

작업 현황 화면의 뷰 스위처에서 **캘린더**를 실제로 동작하게 했다. Figma **`Task Status Layout — Master · Calendar`(210:969)** 기준.

- 로드맵이 기간을 가로 막대로 본다면, 캘린더는 **날짜에 걸린 일정**을 월 달력으로 본다
- 헤더 문구 그대로 **"마감일 기준으로 작업을 봅니다"** — 단계의 **시작일에 이름 칩**, **종료일에 "이름 마감" 칩**을 얹는다
- **마스터는 전체, 스탭은 자기 프로젝트만** — 페이지가 이미 권한별로 계산해 둔 `visibleProjects`를 그대로 넘겨 분기가 자동으로 따라온다(별도 권한 로직 없음)

| 헤더 | 캘린더 · 설명 · 오늘 · ‹ 2026년 7월 › | 동일 (로드맵 카드 헤더와 같은 골격) |

## 3. 구현 메모

- **월 그리드는 기존 `buildMonthGrid`(my-work-month.ts) 재사용** — 내 할일 캘린더와 같은 계산을 쓴다. 다만 이 헬퍼는 이월 칸을 `null`로 주므로, 디자인대로 앞뒤 달 날짜를 보여주기 위해 `gridStart + index`로 실제 날짜를 따로 계산했다(공유 헬퍼는 건드리지 않았다 — 다른 세션이 쓰는 파일)
- **단계 색은 스토어 파생값 그대로** — `useBoardState()`가 프로젝트 색에서 파생한 색을 주므로(docs/72) 캘린더도 같은 색 체계를 자동으로 따른다
- 칩을 누르면 **단계 상세 오버레이**가 열린다 (로드맵 막대와 동일한 진입)
- 월 이동(‹ ›)과 **오늘** 버튼으로 이번 달 복귀

### 담당자 뷰

뷰 스위처를 상태로 바꾸면서, 아직 구현되지 않은 **담당자**는 `disabled` + `title="준비 중입니다"`로 막았다. 누를 수 있게 두면 빈 화면이 되기 때문이다. 구현 시 이 목록에서 빼면 된다.

### 검증 중 만난 것

시드 스크립트가 `PrismaClientValidationError`로 실패해 칩이 하나도 없었고 검증 4건이 FAIL로 나왔다(UI 자체는 통과). 원인은 병렬 세션이 추가한 **`Stage.order` 필수 필드** — 기존 시드 코드에 없던 컬럼이다. `order`를 채워 재시드하니 전부 통과했다. **공유 스키마가 자주 바뀌므로 시드 실패를 UI 실패로 오인하지 않도록 오류 메시지를 먼저 확인해야 한다.**

## 6. 알려진 사항 / 후속 과제

- 칩은 **단계 기준**이다. 할일(Task)의 예정일은 표시하지 않는다 — 디자인의 칩 라벨이 모두 단계명이고 "마감일 기준"이라는 설명과도 맞다. 할일까지 얹을지는 별도 요청 대상
- 한 칸에 일정이 많으면 칸을 넘어가는 부분은 잘린다(`overflow-hidden`). "+N개" 같은 접힘 표시는 디자인에 없어 넣지 않았다
- **담당자 뷰**는 여전히 미구현 — 비활성 상태
- 필터(그룹·프로젝트 칩)는 캘린더에도 그대로 적용된다 — `visibleProjects`를 공유하기 때문
