## 1. 작업 내용 요약

이미 만든 프로젝트의 색을 **사이드바 우클릭 메뉴에서 바꿀 수 있게** 했다. 단계·할일 색이 프로젝트 색 파생(docs/72)이므로, **색 하나만 바꾸면 그 프로젝트의 모든 단계 막대·할일 칩이 로드맵·보드·캘린더에서 함께 바뀐다.**

기존 우클릭 메뉴(프로젝트 삭제) 위에 "색 변경" 스와치 8종을 얹고 구분선으로 나눴다. 마스터의 그룹 트리 행과 스탭의 플랫 리스트 행 **양쪽 모두** 동일하게 적용된다.

## 2. 권한 — 삭제와 동일한 범위

색 변경은 삭제와 같은 가드를 쓴다.

- **마스터**: 전체 프로젝트
- **스탭**: 자기가 작업자(`ownerId`)인 프로젝트만

서버에서 `updateMany`의 where에 `ownerId`를 함께 걸어 **쿼리 레벨에서** 막는다(스탭이 남의 프로젝트 id로 직접 호출해도 0건). 0건이면 403 — 존재 여부를 흘리지 않기 위해 "없음"과 "권한 없음"을 구분하지 않는 기존 삭제 정책을 그대로 따랐다.

색 값은 `#rrggbb` 정규식으로 검증한다. 스타일에 그대로 들어가는 값이라 형식을 좁게 잡았다.

## 5. 알려진 사항 / 후속 과제

- 색 선택은 팔레트 8색 고정이다(생성 UI와 동일). 임의 색 입력은 범위 밖
- 프로젝트 상세 화면에는 색 변경 진입점을 두지 않았다 — 사이드바 우클릭 한 곳으로 통일. 필요해지면 상세 헤더에 추가 가능
