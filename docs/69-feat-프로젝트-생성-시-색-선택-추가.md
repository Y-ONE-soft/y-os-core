## 1. 작업 내용 요약

프로젝트를 만들 때 **색을 직접 고를 수 있게** 했다. 기존에는 팔레트에서 순번대로 자동 배정되어 사용자가 관여할 수 없었다.

사이드바의 "＋ 프로젝트 추가" 인라인 행에 **8색 스와치 팔레트**를 추가했다. 마스터(그룹별 추가)·스탭(플랫 리스트 추가) 양쪽 진입점에 동일하게 적용된다.

- 행을 열면 기존 자동 배정 규칙(전체 프로젝트 수 기준 순번)이 **기본 선택값**으로 제시된다 — 아무것도 안 고르면 종전과 같은 색
- 입력창 왼쪽에 현재 선택 색 점을 표시해 무엇이 적용될지 바로 보인다
- 선택된 스와치는 링으로 표시

## 2. 구현 메모

| 항목 | 내용 |
|---|---|
| 서버·API | **변경 없음** — `POST /api/admin/projects`가 이미 `color`를 받고 있었다 |
| 스토어 | `addProject(groupId, name, color?)`로 확장. 미지정 시 종전 자동 배정 유지(하위 호환) |
| 팔레트 | 기존 `PROJECT_COLORS` 8색 재사용 — 새 색을 도입하지 않아 기존 화면과 톤이 어긋나지 않는다 |

### 스와치 클릭이 입력을 닫아버리는 문제

인라인 입력은 `onBlur={onCancel}`로 포커스를 잃으면 추가를 취소한다. 스와치를 누르면 입력이 blur되어 **색을 고르는 순간 행이 사라지는** 문제가 있었다. 스와치 버튼에 `onMouseDown={(e) => e.preventDefault()}`를 걸어 포커스가 이동하지 않도록 해 해결했다(검증 항목으로도 넣었다).

### 검증 방법론 메모

첫 실행에서 "스와치 클릭 시 선택 반영"이 실패했는데, `click()` 직후 동기적으로 `aria-pressed`를 읽어 **React 리렌더 전 값**을 본 테스트 타이밍 문제였다(요청 본문·저장 색은 정상이었다). 클릭 후 대기 뒤 읽도록 고쳤다. 상태 변화를 단언할 때는 리렌더를 기다려야 한다.

## 5. 알려진 사항 / 후속 과제

- 팔레트는 고정 8색이다. 임의 색(color picker) 입력은 범위 밖 — 필요해지면 별도 태스크
- **이 시점에는 단계 색이 여전히 프로젝트 색과 무관하다**(팔레트 순번 배정). 태스크 ②에서 프로젝트 색 파생으로 통일한다
- 그룹 추가 행에는 색 개념이 없어 그대로 두었다
