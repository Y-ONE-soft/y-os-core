# 133. fix: 내 할일 캘린더를 잘림 없이 세로 스크롤로

## 증상

내 할일 캘린더 뷰가 뷰포트 높이에 고정돼 있어, 한 달에 단계·할일이 많으면 **아래쪽 주가 잘리거나** 칸이 눌려 보였다.

## 원인

캘린더는 페이지의 고정 높이 flex 체인 안에 있다.

```
페이지 h-full → 본문 flex-1 → 좌측 열 flex-1 → 캘린더 flex-1 overflow-clip
```

캘린더 루트가 `flex-1`(남는 높이를 채움) + `overflow-clip`(넘치면 잘라냄)이고, 주 행은 `flex-1` + `minHeight`다. 한 달의 주 높이 합이 캘린더에 배정된 높이를 넘으면 `overflow-clip`이 초과분을 **잘라버렸다.** 주마다 최소 높이(`minHeight`)는 지켜지지만, 전체가 화면보다 커지면 아래가 사라졌다.

## 변경 내용

### `src/components/features/my-work/my-work-calendar.tsx`

**1. 루트를 세로 스크롤로**

`overflow-clip` → `overflow-x-clip overflow-y-auto`. 가로는 그대로 잘라 둥근 모서리를 유지하고, 세로는 넘치면 스크롤한다.

- 내용이 배정 높이보다 작으면 — 주가 `flex-1`로 자라 화면을 채운다 (기존 동작 유지)
- 내용이 배정 높이보다 크면 — `minHeight`가 유지된 채 넘쳐서 스크롤바가 생긴다. **더 이상 잘리지 않는다**

주 행은 그대로 두었다. `flex-1` + `minHeight`가 그대로 "여유 있으면 채우고, 부족하면 넘친다"를 만든다.

**2. 요일 헤더 고정**

루트가 스크롤되면 요일 헤더(일·월·화…)도 함께 밀려 올라간다. 헤더에 `sticky top-0 z-20 bg-card`를 줘 스크롤 중에도 위에 남게 했다. `bg-card`로 아래를 지나가는 내용을 가린다.

스크롤바가 헤더와 날짜 칸을 **같은 컨테이너 안에서** 줄이므로 열 정렬이 어긋나지 않는다(헤더를 스크롤 밖에 두면 스크롤바 폭만큼 틀어진다).

## 왜 내부 스크롤인가 (페이지 성장 대신)

캘린더를 페이지째 늘려 문서 전체가 스크롤되게 하는 방법도 있었지만, 그러면 **요청 알림·백로그 aside까지 밀려 내려가** 함께 스크롤해야 한다. 캘린더 안에서만 스크롤하면 요청·백로그는 제자리에 남는다. "잘리지 않게"라는 요구를 충족하면서 주변 레이아웃을 건드리지 않는다.
