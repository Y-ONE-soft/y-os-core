# feat: 드래그로 기간을 고르는 미니 캘린더 컴포넌트

## 작업 요약

프로젝트 생성 다이얼로그에서 쓸 날짜 선택 캘린더 `RangeCalendar`를 만들었다. 한 컴포넌트가 두 모드를 처리한다.

| 모드 | 쓰이는 곳 | 조작 |
| --- | --- | --- |
| `single` | 프리셋 적용 — 기준일 하나 | 클릭 한 번에 확정 |
| `range` | 직접 만들기 — 기간 | **드래그**로 시작~종료, 클릭 한 번은 하루 |

이 태스크는 컴포넌트까지다. 다이얼로그 결합은 후속 태스크(3·4)다.

## 설계 판단

### 1. 기존 캘린더를 확장하지 않고 새로 만든 이유

`내 할일`의 월 캘린더(`my-work-calendar`)에는 **이미 단계·할일을 옮기는 드래그가 걸려 있다.** 여기에 "기간 선택 드래그"를 얹으면 포인터 이벤트마다 모드를 갈라야 하고, 두 드래그 로직이 서로를 침범한다. 사용자와 상의해 **다이얼로그 안 별도 미니 캘린더**로 정했고, 그래서 이 컴포넌트는 선택 외에 하는 일이 없다.

### 2. 그리드를 직접 계산한 이유

`my-work-month.ts`의 `buildMonthGrid`를 재사용할 수 있어 보였지만 쓰지 않았다. **`my-work`가 이미 `projects`를 참조하고 있어**(`roadmap-utils`, `stage-detail-overlay` 등) 반대 방향으로 끌어오면 피처 간 순환 의존이 된다. 그리드 계산은 10줄 남짓이라 자체 구현이 싸다.

대신 날짜 연산은 태스크 1에서 만든 `@/lib/stage-plan`의 `addDaysISO`를 쓴다 — 서버 생성 로직과 **같은 날짜 규칙**을 공유하게 된다.

### 3. 이월 칸도 실제 날짜를 갖는다

`buildCells`는 앞뒤 이월 칸을 `null`로 비우지 않고 실제 날짜를 채운다. 월 경계를 걸치는 기간(예: 7/28~8/3)을 드래그로 잡으려면 이월 칸도 선택 대상이어야 하기 때문이다. 표시만 흐리게 처리한다.

### 4. `todayLocalISO`는 UTC를 쓰지 않는다

`stage-plan`의 날짜 연산은 UTC 기준이지만(DST 보정 회피), **"오늘"은 로컬 기준이어야 한다.** `new Date().toISOString()`으로 찍으면 한국 시간대(UTC+9) 오전 9시 이전에 하루 전 날짜가 나온다. 그래서 로컬 연·월·일을 직접 조합한다.

### 5. 드래그가 잠기지 않게

셀 밖에서 마우스를 떼면 `pointerup`이 셀에 오지 않아 드래그 상태가 영원히 남는다. 컨테이너의 `onPointerUp`/`onPointerLeave`에서도 확정·해제하도록 했다.

역방향 드래그(뒤 → 앞)는 `normalize()`로 앞뒤를 바꿔 정규화한다.

## 검증 중 잡은 실제 결함 — 범위 밴드가 보이지 않았다

브라우저에서 8/4 → 8/19을 드래그했을 때 **반환값은 정확했지만(`2026-08-04 ~ 2026-08-19`) 화면에는 양끝 두 칸만 찍히고 사이 날짜가 전혀 강조되지 않았다.**

원인은 사이 날짜에 준 `bg-accent`가 이 패널 배경과 명도가 거의 같아 묻힌 것이었다. 사용자 입장에서는 기간이 잡혔는지 알 수 없는 상태다.

```diff
- selected && !isEdge && "bg-accent text-accent-foreground",
+ selected && !isEdge && "bg-primary/15 text-foreground",
```

`primary`의 옅은 틴트로 바꿔 밴드가 분명히 드러나게 했다. **값만 확인하고 넘어갔으면 놓쳤을 결함**이라 화면을 캡처해 눈으로 본 것이 결정적이었다.

## 변경 파일 내역

### 신규 `src/components/features/projects/range-calendar.tsx`

| 요소 | 내용 |
| --- | --- |
| `RangeCalendar` | `mode`(single/range), `value`, `onChange`, `initialMonth` |
| `todayLocalISO` | 로컬 기준 오늘 (export — 후속 태스크에서 기본값으로 쓴다) |
| `DateRange` | `{ startDate, endDate }` |
| `buildCells` | 이월 칸 포함 월 그리드 |
| `normalize` | 역방향 드래그 정규화 |

드래그 중에는 확정값 대신 진행 중인 범위를 그린다(`shown`). 브라우저 기본 텍스트 선택이 끼어들지 않도록 `onDragStart` 기본 동작을 막고 `select-none`을 준다.

## 알려진 이슈 / 후속

- **키보드 조작이 없다.** 날짜 칸이 `button`이라 Tab·Enter로 개별 선택은 되지만, 화살표 키로 날짜를 옮기거나 Shift+화살표로 범위를 늘리는 조작은 없다. 드래그가 요구사항의 핵심이라 이번 범위에 넣지 않았으나, 접근성 관점에서 후속으로 다룰 만하다.
- **터치 지원은 확인하지 않았다.** 포인터 이벤트를 써서 원리상 동작해야 하지만 실제 터치 기기로 검증하지 않았다.
- 남은 태스크: (3) 생성 버튼 + 프리셋 적용 모드, (4) 직접 만들기 모드.
- docs 번호는 작성 시점 기준으로 87을 썼다. 병렬 작업이 많아 머지 순서에 따라 조정이 필요할 수 있다.
