## 1. 작업 내용 요약

보드의 작업(제목) 클릭 시 열리는 **작업 상세 오버레이**를 Figma `Task Detail Overlay` 디폴트 + 변형 4종 기준으로 구현했다. 디폴트 레이아웃(헤더/본문/세부 사항 사이드), 산출물 업로드 상태, 연결 티켓·위키 상태, 공동 작업자 지정 요청·도움 요청 모달까지 포함 — 상태 변형은 별도 화면이 아니라 **하나의 컴포넌트의 로컬 상태 전환**으로 재현된다.

## 4. 주요 결정 사항

| 항목 | 결정 | 이유 |
|---|---|---|
| 오버레이 컨테이너 | shadcn `Dialog` (1280×780, rounded-16, 기본 닫기 버튼 숨김·디자인의 ✕ 사용) | 접근성(포커스 트랩·ESC)·프로젝트 규약. 요청 모달은 중첩 Dialog |
| 스토어 연동 | 제목/완료 체크(toggleTask)·**단계 셀렉트(moveTask 신규)**는 board-store 실동작. 단계는 taskId로 역탐색해 이동 후에도 현재 단계가 갱신됨 | 이미 있는 로컬 도메인은 실제로 동작시키는 편이 화면 검증에 유리 |
| 산출물 | 파일 선택(찾아보기)·드래그 앤 드롭 → 로컬 목록 행(아이콘·형식·크기·방금 전), ⋯ 메뉴로 삭제 | Artifacts Uploaded 변형 상태를 실제 인터랙션으로 재현. 업로드 백엔드는 미도입 — 세션 로컬 |
| 연결 티켓·위키 | 고스트 행 → 인라인 입력, `ABC-123` 패턴이면 티켓 칩+진행 중 배지, 아니면 위키 칩 | Links Added 변형 재현. 실제 티켓/위키 검색은 해당 도메인 태스크에서 |
| 댓글 | 로컬 목록 + 입력/Enter/등록 (세션 사용자 아바타·이름) | 디자인의 빈 상태 + 등록 흐름 재현 |
| 요청 모달 | 멤버 4명 자리표시(TEAM_MEMBERS) 체크 선택, 선택 요약 문구("N명 선택됨"), 변형별 메시지 placeholder, 보내기=닫기 | 발송 대상 시스템(알림) 미도입. 디자인의 선택 예시 상태 대신 빈 선택으로 시작 |
| 작업 키 칩 | `YOS-124`처럼 task id 해시 기반 자리표시 키 | 티켓 키 체계 미도입 — DB 태스크에서 실키로 교체 |
| 유형·난이도 | shadcn Select, 로컬 상태 자리표시 옵션 | 도메인 필드 미도입 |
| 생성·수정 시각 | 정적 자리표시 ("생성 2026-07-22 · 수정 방금 전") | 작업에 타임스탬프 필드 없음 |

- 이슈 등록(🐞) 버튼은 시각 전용 — 이슈 도메인 태스크에서 연결
