# 143. fix: 예상 단계 수 입력의 앞자리 0 표시 제거

## 증상

프로젝트 생성 > 직접 만들기의 **예상 단계 수** 입력에서 앞의 `0`이 사라지지 않았다. 필드가 `0`을 보여준 채로 남고, 그 뒤에 `1`을 치면 `01`처럼 찍혔다.

## 원인

입력이 `value={stageCount}`(숫자)인데, 필드를 비우면 `onChange`의 `Number("") || 0`으로 **stageCount가 0**이 된다. 그러면 `value={0}`이라 입력칸에 `0`이 그대로 표시되고, 그 뒤에 숫자를 이어 치면 `01`이 된다.

## 변경 내용

### `src/components/features/projects/project-create-dialog.tsx`

`value={stageCount}` → **`value={stageCount || ""}`**. 0일 때는 빈 칸으로 보여준다. 빈 칸에 `1`을 치면 앞자리 0 없이 `1`이 된다.

내부적으로는 여전히 0이라 `spans`가 비고 제출도 막히므로(기존 `canSubmit` 로직 그대로) 동작에는 영향이 없다. 표시만 고친 것이다.
