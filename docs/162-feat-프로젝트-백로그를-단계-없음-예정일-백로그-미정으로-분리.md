# feat: 프로젝트 백로그를 단계 없음(예정일)·백로그(미정)로 분리

## 요청 (사이클 A 태스크 1)

> 백로그랑 단계없음이랑 구분하자. 백로그는 날짜도·단계도 없는 것, 단계없음은 날짜는 있으나 단계에 속하지 않은 것. 프로젝트 상세에도 디폴트로 "단계 없음"이 표시되어야 함.

## 개념

`Task`에서 `stageId=null`인 할일(지금까지 통칭 "백로그")을 **예정일 유무로** 둘로 가른다. 스키마 필드 추가 없이 예정일(`scheduledDate`)로 판정하는 **가상 구분**이다.

- **단계 없음** = `stageId=null` + **예정일 있음**. 캘린더에 이 항목들이 뜬다.
- **백로그** = `stageId=null` + **예정일 없음(미정)**.

## 변경 — `src/components/features/projects/project-backlog.tsx`

프로젝트 상세 우측 패널을 **두 섹션**으로 재구성했다(둘 다 항상 노출 = "디폴트로 생성").

1. **단계 없음** 섹션(상단) — `backlog` 중 예정일 있는 할일. 비면 안내 문구.
2. **백로그** 섹션(하단) — 예정일 없는 할일. 할일 추가 입력(`addBacklogTask` = 단계·예정일 없이 생성)은 여기 둔다.

세부:
- 한 줄 티켓을 `BacklogRow` 컴포넌트로 추출해 두 섹션이 공유한다(기존 카드 동작 — 클릭 상세, 체크박스, 소속 드롭다운, 우클릭·삭제 — 그대로).
- **단계 없음 행엔 예정일 배지**를 붙여 백로그(미정)와 한눈에 구분한다(`formatShort`).
- 소속 드롭다운 트리거 라벨은 현재 상태를 반영: 예정일 있으면 "단계 없음", 없으면 "백로그". "백로그" 항목의 체크는 미정일 때만 표시.
- **드롭 존은 패널 전체 그대로** — 단계 카드를 이 패널에 놓으면 `assignTask(..., null)`로 단계에서 빼고 **예정일도 해제**되어 백로그(미정)로 간다. #143의 "칩→백로그 드래그(날짜 해제)"와 일관.
- 항목이 늘 수 있어 패널에 `overflow-y-auto`를 추가.

## 데이터/동작 영향

- **스키마·서버·저장 로직 변경 없음.** 순수 표시 구분(같은 `board.backlog` 데이터를 예정일로 나눠 그림).
- 예정일 있는 stageless 할일은 이제 백로그 섹션이 아니라 **단계 없음** 섹션에 보인다. (캘린더에는 원래대로 뜬다.)
