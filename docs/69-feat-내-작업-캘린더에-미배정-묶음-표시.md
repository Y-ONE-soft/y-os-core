## 1. 작업 요약

예정일이 잡힌 **미배정 작업(프로젝트 없음)** 을 캘린더에 **"미배정" 묶음**으로 그린다.

캘린더의 박스는 `project` 키로 묶이는데 미배정 작업은 소속 프로젝트가 없어 **어느 박스에도 들어갈 자리가 없었다.** 다음 태스크에서 백로그를 날짜 칸에 떨어뜨릴 수 있게 되면 미배정 작업도 예정일을 갖게 되므로, 그것들이 놓일 자리를 먼저 만든 것이다.

## 2. 구현

가짜 프로젝트 키 `__unassigned__`(`UNASSIGNED_BOX`)로 묶는다. 배치·렌더 모듈은 `project`를 그냥 문자열 키로 다루므로 **박스·레인 로직을 건드리지 않고** 같은 흐름을 탄다.

| 항목 | 값 |
|---|---|
| 묶음 이름 | `미배정` |
| 색 | `#71717a` (뮤트 계열 — 프로젝트 색과 구분) |
| 대상 | `projectId = null` 이면서 예정일이 있는 작업 |

`taskChip()`의 인자를 `Project` 객체 → **박스 키 문자열**로 바꿔 프로젝트 작업과 미배정 작업이 같은 함수를 쓰게 했다.

## 5. 알려진 사항 / 후속 과제

- 지금은 **API로 예정일을 넣어야만** 미배정 칩이 생긴다. 사용자 조작으로 만드는 경로(백로그 → 날짜 드래그)는 태스크 2
- 미배정 칩도 기존 할일 칩과 같이 **드래그로 날짜 이동**이 된다(캘린더 드래그는 taskId 기준이라 소속과 무관)
- 백로그 작업(프로젝트는 있고 단계는 없는 경우)도 예정일이 있으면 그 프로젝트 박스 안에 칩으로 뜬다 — 주석을 실제 동작에 맞게 고쳤다
