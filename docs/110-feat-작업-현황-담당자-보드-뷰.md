## 권한별 범위

| 권한 | 컬럼 |
| --- | --- |
| 마스터 | **전체 직원** + `미배정`(맨 뒤) |
| 스탭 | **본인 컬럼만** |

스탭에게 미배정 컬럼도 주지 않았다 — "자기 것만"이라는 요구를 그대로 따랐다.
남의 할일이 새어 나가지 않는지는 검증 항목으로 따로 뒀다(9d).

컬럼 목록은 `useUsers()`가 주는 전체 직원이고, 할일이 하나도 없는 직원도 빈
컬럼으로 나온다 — "전체 직원 + 미배정"으로 정한 사용자 결정에 맞췄다.

## 수치와 부하 막대 — 디자인이 모호해 정한 것

디자인 헤더에는 `9/6`이 빨간색으로 적혀 있는데 **그 컬럼의 카드는 6장**이고, 부하
막대는 22.81%만 차 있다. 셋이 서로 안 맞아 자리표시로 판단했다. 사용자도 의미를
특정하지 않았다("몰라").

**같은 디자인 파일의 로드맵 단계 라벨이 `0/3`, `0/2` 형식으로 이미 `완료/전체`를
쓰고 있다**(`87:1189`, `87:1197` 등). 같은 파일 안의 확립된 규약을 따르는 것이
가장 근거 있는 해석이라 보고 이렇게 정했다.

| 요소 | 의미 |
| --- | --- |
| `N/M` | 완료 / 담당 할일 총계 |
| 막대 폭 | 완료 비율 |
| 경고색(빨강) | **지연**이 하나라도 있을 때 — 예정일이 지났는데 끝나지 않은 할일 |

디자인의 빨간 상태를 살리되, 없는 필드(용량)를 지어내지 않고 **있는 데이터로 설명
가능한 조건**을 골랐다. 지연 건수는 수치에 `title`로 붙고, 해당 카드에는 `지연`
배지가 붙어 어느 할일 때문인지 바로 보인다.

> 다른 의미였다면 바꾸기 쉬운 자리다 — `ColumnHead`의 `alert`/`ratio` 두 값만
> 다시 계산하면 된다.

## 화면

컬럼은 `bg-border` 위에 헤더(아바타·이름·직책·수치) → 진척 막대 → 카드 목록 순이다.
아바타 색은 `avatarColor(id)`로 사용자 id에서 결정적으로 파생한다(User에 색 컬럼이
없다). 미배정은 muted 배경에 `—`.

카드는 **체크박스 + 프로젝트 색 점 + 이름**이다. 프로젝트 점을 넣은 이유는 이 보드가
여러 프로젝트를 한데 모으기 때문 — 어느 프로젝트 할일인지 구분이 필요하다.

카드 상호작용은 docs/94에서 정한 규약을 그대로 따랐다. **카드 전체가 클릭 대상**이고
호버하면 링으로 선택된 것처럼 보이며, 체크박스만 예외로 완료를 토글한다.
`role="button"`·`tabIndex`·Enter/Space로 키보드도 받는다.

### 컬럼 너비 — 처음에 틀렸다가 고친 것

프로젝트 보드의 컬럼(`w-[260px] shrink-0`)을 그대로 가져왔더니 화면 오른쪽이 크게
비었다. 디자인은 `flex-[1_0_0]`으로 **너비를 균등하게 채운다**. 스크린샷으로 확인하고
고쳤다.

```
min-w-[220px] flex-1
```

다만 그대로 `flex-1`만 두면 인원이 늘수록 무한정 좁아지므로 하한을 두고, 하한에
걸리면 컨테이너의 `overflow-x-auto`로 넘긴다. 디자인(4명)에서는 균등 분할로,
인원이 많아지면 가로 스크롤로 동작한다.

## 검증

`localhost:3051`에서 puppeteer로 확인. 마스터·스탭 각각 실제 로그인해 분기를 봤다.

| 검증 | 결과 |
| --- | --- |
| 담당자 탭이 활성화 가능(비활성 해제) | PASS |
| 마스터: 전체 직원 + 미배정 컬럼 수 | PASS |
| 마지막 컬럼이 `미배정` / 부제 `담당자 없음` | PASS |
| 수치 = 완료/전체 (마스터 `1/3`, 스탭 `0/1`) | PASS |
| 미배정 컬럼이 기존 담당자 없는 할일까지 모음 | PASS |
| 담당자별로 자기 할일만 담김 (남의 것 없음) | PASS |
| 진척 막대 폭 = 완료 비율(1/3 → 33%) | PASS |
| 지연 있으면 막대·수치가 경고색 | PASS |
| 지연 없으면 정상색 | PASS |
| 지연 카드에 `지연` 배지 | PASS |
| 카드 클릭 → 할일 상세 | PASS |

| 스탭: 컬럼 1개(본인)뿐 | PASS |
| 스탭에게 미배정 컬럼 없음 | PASS |
| 스탭에게 남의 할일 안 보임 | PASS |

22개 항목 전부 PASS. 태스크 1의 검증 8개도 다시 돌려 전부 통과(회귀 없음).
`npx tsc --noEmit`, `npm run build`, `npm run lint` 통과. 콘솔 에러 없음.

### 검증에서 헛짚은 것

처음 4건이 FAIL이었는데 전부 테스트 쪽 문제였다.

1. **`main section`이 필터 영역까지 잡았다.** 페이지의 필터 블록도 `<section>`이라
   컬럼 수가 하나 더 세어졌다. → `[role="progressbar"]`를 가진 section으로 좁혔다.
2. **아바타도 `text-[11px]`이라** 부제 선택자가 아바타 글자(`—`)를 집었다.
   → 이름 블록의 자식 순서로 잡았다.
3. **미배정 컬럼 기대값을 잘못 잡았다.** 내가 만든 1건만 셌는데, 실제로는 **이전부터
   있던 담당자 없는 할일이 전부 여기 모인다**(정상 동작). → 기대값을 API로 계산하게
   바꿨다.

## 알려진 것

- 부하 막대의 정상색으로 `bg-emerald-500`(디자인의 `#10b981`)을 썼다. `globals.css`에
  성공/정상을 뜻하는 토큰이 아직 없어 Tailwind 팔레트 이름을 그대로 쓴 것이다.
  토큰이 생기면 갈아 끼울 자리다.
- 담당자 지정은 **할일 상세에서만** 가능하다(docs/109). 이 보드에서 카드를 다른
  담당자 컬럼으로 드래그해 옮기는 건 아직 없다 — 디자인에도 그 상호작용은 없다.
