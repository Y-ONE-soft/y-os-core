# feat: 캘린더 드래그 중 기간 경계에서 자동으로 이전/다음 기간 전환

## 요청

> 캘린더 뷰에서 단계·할일을 드래그할 때, 월(기간) 경계를 지나가면 같이 넘어갈 수 있도록 해줘.

기존에는 드래그 좌표가 보이는 그리드로 클램프돼(`pointerDay`가 `[0, lastDay]`) 보이는 달 밖으로는 끌 수 없었다. 다음 달로 옮기려면 손을 떼고 달을 넘긴 뒤 다시 잡아야 했다.

## 변경 내용

드래그 중 포인터가 그리드 **가장자리**에 머물면 자동으로 이전/다음 기간으로 넘기고, 끌던 항목이 새 기간에서도 끊김 없이 이어지게 한다.

### `src/components/features/my-work/my-work-calendar.tsx`

1. **절대 날짜 기준 델타** — 드래그 델타를 그리드 상대값이 아니라 **절대 날짜**로 계산한다. `dragDelta(drag, x, y) = round( dayOffset(현재 gridStart, 누를 때 gridStart) + pointerDay(x,y) − 누를 때 pointerDay )`. 가운데 항이 기간 전환으로 gridStart가 바뀐 만큼을 보정하므로, 달이 바뀌어도 델타가 이어진다. `dragRef`에 `startGridStart`·`lastX`·`lastY`를 추가.
2. **가장자리 감지 + 자동 전환** — `edgeDirection`이 포인터가 흐름 방향 가장자리 밴드(월 보기=위/아래, 주·일 보기=좌/우) 안이면 방향을 돌려준다. 가장자리에 들어오면 `setInterval`로 `onEdgeTurn`을 주기(550ms)마다 호출한다. **즉시** 넘기지 않고 첫 발화를 간격만큼 미뤄, 마지막 줄에 그냥 놓으려다 스치는 것으로는 안 넘어가게 했다.
3. **전환 시 미리보기 갱신** — `gridStart`가 바뀌면(effect) 드래그 중이면 마지막 포인터 좌표로 델타를 다시 계산해 `onDrag("move")`를 재발신한다. 그래서 전환 순간 끌던 막대·칩이 새 기간의 제자리로 따라온다.
4. 릴리즈·언마운트 시 타이머 정리(`stopEdgeTurn`).
5. `onEdgeTurn?: (direction) => void` prop 추가.

### `src/components/features/my-work/my-work-calendar-panel.tsx`

- `MyWorkCalendar`에 `onEdgeTurn={shiftPeriod}` 연결.
- **`shiftPeriod`를 함수형 업데이트(`setAnchorISO(prev => …)`)로 변경.** 자동 전환 인터벌 콜백은 타이머를 걸 때의 `shiftPeriod` 클로저를 붙잡는데, 그게 그때의 `anchorISO`를 읽으면 매 발화가 "같은 달 + 1"만 계산해 한 번밖에 안 넘어간다. `prev`를 읽어 항상 현재 기준에서 넘긴다.

## 결정 이유

- **절대 날짜 델타**: 기간 전환으로 그리드가 재기준(gridStart 변경)되면 그리드 상대 델타는 끊긴다. 절대 날짜로 두면 전환과 무관하게 "누른 날 대비 며칠"이 일관된다.
- **머무름(dwell) 후 전환**: 즉시 전환하면 마지막 주에 정상적으로 놓으려는 동작을 방해한다. 가장자리에 550ms 머물러야 넘어가 오작동을 줄였다.
- **함수형 업데이트**: 타이머 콜백의 스테일 클로저 문제를 근본적으로 없앤다.

## 검증 (실제 앱 · 실데이터 · 깨끗한 빌드)

로컬 dev(최신 main 워크트리)에서 실사용자(김주웅) 세션으로 실제 드래그:

- Y.OS core 2단계(7/20~)를 잡아 캘린더 **하단 가장자리**로 끌고 머무름 → 제목이 **7월 → 8월 → 9월**로 연속 전환되고, 끌던 단계 막대가 계속 따라옴.
- 안쪽으로 옮겨 손을 떼자 **단계 PATCH + 할일 PATCH**가 나가고 DB `startDate`가 **07-20 → 09-14**로 반영됨(월 경계를 넘어 저장). 검증 후 07-20으로 원복.
- **실앱 검증으로 잡은 버그 2건(그대로 배포됐으면 기능이 죽었을 것):**
  - 자동 전환이 **한 번만** 되던 문제 → `shiftPeriod` 함수형 업데이트로 해결.
  - 릴리즈 시 델타가 **항상 0**이라 저장이 안 되던 문제 → `handlePointerUp`이 `dragRef`를 비운 뒤 `dragDelta`를 부르는데 `dragDelta`가 `dragRef.current`를 읽어 0을 반환. `dragDelta`가 `drag`를 인자로 받게 고쳐 해결.
- `npm run lint` 통과 · `npm run build` 성공(exit 0).

## 알려진 이슈 / 참고

- 가장자리 홀드는 550ms마다 **한 기간씩** 넘어간다(월 보기=한 달, 주=한 주, 일=하루). 오래 머물수록 빨리 이동하며, 목표 기간에서 안쪽으로 옮겨 정확히 놓는다.
- 병렬로 `my-work-calendar.tsx`를 미커밋 수정 중인 세션(`캘린더-가로-간격`)이 있어 머지 순서·리베이스 주의.
- docs 번호 165는 작성 시점 기준(최대 164). 병렬 머지 충돌 시 나중 머지 쪽이 조정.
