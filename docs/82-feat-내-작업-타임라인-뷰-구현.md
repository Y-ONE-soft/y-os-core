## 1. 작업 요약

내 작업 페이지의 **타임라인 뷰**를 만들고, 캘린더/타임라인 탭이 실제로 전환되게 했다. 기준 디자인은 Figma `My Work Layout — Timeline`(node `199:1139`)이며, **표·막대·기간 축은 작업 현황 로드맵과 같은 컴포넌트를 재사용**해 두 화면의 조작감을 맞췄다(사용자 요구: "현재 서비스랑 통일감 있는 컴포넌트").

## 2. 화면 구성

| 영역 | 내용 |
|---|---|
| 헤더 | `타임라인` + 표시 기간(`formatPeriod`) · **오늘** 버튼 · 범위 스위처(일/주/개월/분기) |
| 라벨 컬럼 | `프로젝트 · 작업` (200px, 가로 스크롤 시 고정) |
| 프로젝트 행 | 색 점 + 이름 + `· N`(작업 수), **`전체 N%` 막대** (단계 기간과 할일 예정일을 모두 덮는 범위) |
| 작업 행 | 체크박스 + 작업명(클릭 → 작업 상세), 예정일 위치에 **마커 + 날짜**(완료면 `✓`) |
| 축 | 주/월 눈금선, **오늘선**, 가로 스크롤(진입 시 오늘 위치로) |

## 3. 재사용한 것

| 재사용 | 출처 |
|---|---|
| `buildTimeline` / `RANGE_OPTIONS` / `formatPeriod` / `boundsOfStages` | `roadmap-window.ts` (작업 현황 로드맵) |
| `RoadmapBar` (이동·드래그·미리보기) | `roadmap-bar.tsx` |
| 라벨 컬럼 + 눈금 + 오늘선 마크업, 오늘로 스크롤 | `workload-roadmap.tsx` 구조를 따름 |
| `taskTone` 색 규칙 | `project-palette.ts` |
| 단계 커버 규칙(`clampStageToTasks`) | 캘린더 드래그와 동일 |

작업 막대를 끌면 **예정일이 바뀌고, 단계 밖으로 나가면 단계가 늘어나 덮는다** — 캘린더 뷰와 같은 규칙이다.

## 4. 뷰 전환은 URL로

탭은 `<Link>`이고 상태는 `?view=timeline`에 둔다.

- 페이지가 **서버 컴포넌트로 남는다** (CLAUDE.md: 클라이언트 경계는 트리 아래쪽에)
- 링크 공유·뒤로가기가 자연스럽다
- Next 16 규약대로 `searchParams`는 `Promise`라 `await` 후 읽는다

### 구현 중 고친 것 2가지

- **React Compiler 린트**(`preserve-manual-memoization`)가 `useCallback`을 보존하지 못해 오류가 났다. 캘린더 패널과 같은 방식으로 `myProjects`·`timeline`을 `useMemo`로 고정하니 해소됐다. ref로 우회하는 방법도 시도했으나 `react-hooks/refs`에 걸려, 코드베이스 관례(useMemo)를 따르는 쪽이 맞았다
- **하루짜리 막대 안에서 날짜가 잘렸다**(`07/`). `RoadmapBar`는 최소 폭 26px에 `overflow-hidden`이라 글자가 들어가지 않는다. 디자인처럼 **막대 오른쪽에 날짜를 따로** 그려 해결했다(막대는 라벨 없는 마커로 두고, 위치는 `barRange`로 계산)

## 7. 알려진 사항 / 후속 과제

- 헤더의 범위 스위처는 로드맵 규격(일/주/개월/분기)을 따랐다. 디자인의 `기간·시작일·종료일` 세그먼트는 캘린더 뷰 헤더의 것으로, 타임라인에서는 로드맵과 통일하는 쪽을 택했다
- 프로젝트·작업 행만 그린다. 디자인처럼 **단계 행은 넣지 않았다** — 작업 현황 로드맵이 이미 프로젝트·단계 축이라 역할이 겹친다
- 필터 행(🔍 필터·담당자·프로젝트)은 여전히 시각만 — 캘린더 뷰와 동일
- 미배정 작업은 타임라인에 없다(프로젝트 기준 표라서). 캘린더 뷰에는 "미배정" 묶음으로 나온다
