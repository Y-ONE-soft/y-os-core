## 1. 작업 내용 요약

프로젝트 상세 보드(#3에서 정적 자리표시로 구현됨)를 **실제 동작**으로 전환했다. 사용자 지시("이미 있으면 하지 말고")에 따라 기존 시각 요소는 그대로 재사용하고, 없던 것만 추가:

1. **단계 추가 다이얼로그** — Figma `Stage Add Overlay`(116:545): 단계명·시작일/종료일·공동 작업자·데드라인 표시 토글·취소/추가
2. **보드 스토어** — 프로젝트별 단계/작업/백로그 상태 (localStorage 영속, 기존 자리표시 데이터를 YOS 시드로 이관)
3. 기존 `＋ 단계`/`＋ 단계 추가` 버튼(로드맵) → 다이얼로그 연결, 생성 시 로드맵 행·기간 막대·보드 컬럼 자동 반영
4. 보드 `＋ 작업` → 인라인 입력 동작화 (Enter 생성·Esc/blur 취소), 백로그 입력 동작화
5. **점선 "＋ 단계 추가" 컬럼** (디자인 116:452 — 구현에 없던 요소) 추가
6. 체크박스 완료 토글 → 단계별 done/total·로드맵 %·상단 진행률 실시간 계산

## 3. 주요 구현 결정

| 항목 | 결정 | 이유 |
|---|---|---|
| 데이터 계층 | `board-store.tsx` — project-store와 동일한 `useSyncExternalStore` + localStorage 패턴 (`yos.board.v1`). Provider 불필요한 모듈 스토어 | 확립된 패턴 재사용. 단계/작업 도메인 DB 전환 시 API 경계 흐름으로 교체 |
| 시드 | 기존 정적 데이터(96:166 예시)를 `BOARD_SEED["p-yos"]`로 이관, 타 프로젝트는 빈 보드 | 화면 유지하면서 동작화 |
| 날짜 입력 | shadcn Input `type="date"` | 디자인은 텍스트 필드지만 네이티브 피커가 실용적. 종료일 < 시작일이면 추가 버튼 비활성 |

| 기간 막대 | 단계 날짜를 로드맵 창(2026-07-15 + 28일) 기준 % 위치로 계산, 창 밖은 클램프. 색은 단계 색에서 파생(hexToRgba 12%/80%) | 정적 좌표 하드코딩 제거 |

| 공동 작업자 필드 | 현재 사용자 아바타 + "작업자 추가" 비활성 자리표시 | 작업자 도메인 미구현 — 인증/멤버 태스크에서 연결 |
| 완료 토글 | 체크 시 취소선 + 카운트·% 갱신 | 디자인엔 없지만 카운트 표시(0/3)가 실데이터임을 성립시키는 최소 동작 |
| shadcn `switch` 추가 | 다이얼로그 데드라인 토글용 | CLI 설치 |

## 6. 알려진 사항 / 후속 과제

- 드래그 인터랙션(막대 기간 조절, 백로그↔단계 카드 이동)은 미구현 — 디자인 문구만 존재, 별도 태스크 권장
- 단계 컬럼 `⋯` 메뉴(이름 변경·삭제 추정), 로드맵 범위 스위처(오늘/일/주/개월/분기)는 시각만 — 후속 태스크
- 보드 데이터는 localStorage — 단계/작업 DB 모델링 태스크에서 API 경계로 교체
- 로드맵 창이 2026-07-15+4주 고정(자리표시) — 실데이터 전환 시 동적 창(오늘 중심) 필요
- 진행 중 발견: 이전 태스크(db-foundation)의 dev 서버 프로세스가 3003 포트와 워크트리 폴더 잠금을 유지하고 있었음 — 프로세스 종료로 포트 회수 (폴더 잔여물은 잠금 해제 후 삭제 예정)
