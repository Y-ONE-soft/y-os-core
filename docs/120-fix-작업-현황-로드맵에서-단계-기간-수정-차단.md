## 1. 작업 내용 요약

작업 현황(`/projects`)의 로드맵에서 **단계 막대를 끌어 기간을 바꾸던 동작을 껐다.** 클릭해서 단계 상세를 여는 것은 그대로다.

작업 현황은 여러 프로젝트를 한눈에 훑는 **조회 화면**인데, 막대가 프로젝트 상세 로드맵과 같은 컴포넌트라 드래그 편집까지 딸려 있었다. 넓은 화면에서 스크롤하다 막대를 살짝 끌면 의도치 않게 일정이 바뀐다.

## 3. 구현 방식

`RoadmapBar`는 **이미 읽기 전용을 지원**한다 — prop 주석에 "드래그로 기간이 바뀌면 호출 — 넘기지 않으면 드래그 비활성"이라 적혀 있고, 내부적으로 `const draggable = Boolean(onCommit)`으로 갈린다.

따라서 컴포넌트를 고칠 필요 없이 **호출부에서 `onCommit`을 넘기지 않는 것**으로 끝났다. 리사이즈 손잡이·커서·포인터 핸들러가 한꺼번에 빠진다.

`title`도 함께 바꿨다. 이전 문구는 "클릭하면 단계 상세, **양 끝을 끌면 기간 조절, 가운데를 끌면 이동**"이라 되지 않는 동작을 안내하고 있었다 → "클릭하면 단계 상세".

## 4. 영향 범위

프로젝트 상세(`/projects/[id]`)의 로드맵은 **그대로 편집 가능**하다. 거기서는 `ProjectRoadmap`이 별도로 `onCommit`을 넘긴다. 기간 수정은 프로젝트 상세 로드맵과 단계 상세 오버레이에서만 한다.
