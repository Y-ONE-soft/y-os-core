# feat: 로그인 후 첫 화면을 프로젝트로 변경

- **예정 커밋 메시지**: `feat: 로그인 후 첫 화면을 프로젝트로 변경`
- **작업일**: 2026-07-22
- **작업 브랜치**: `로그인-첫화면-프로젝트` (워크트리 `.claude/worktrees/login-landing-projects`)

---

## 1. 작업 내용 요약

로그인하면 **운영(`/`) 대신 프로젝트(`/projects`)로 도착**하도록 바꿨다. 상단 메뉴의 활성 항목도 자연히 `프로젝트`가 되고, 사이드바도 프로젝트 섹션(작업 현황·내 할일·작업 분석·프리셋)으로 열린다.

로그인한 사용자를 `/`로 보내던 지점이 **두 곳**이었다.

| 위치 | 상황 |
|---|---|
| `src/components/features/auth/login-form.tsx` | 로그인 성공 직후 `router.replace("/")` |
| `src/proxy.ts` | 세션이 있는 채로 `/login`에 접근했을 때 |

두 곳이 각자 문자열을 들고 있으면 한쪽만 고쳤을 때 어긋난다. **`AFTER_LOGIN_PATH` 상수를 `src/lib/constants.ts`에 두고 양쪽이 같은 값을 참조**하게 했다. `constants.ts`는 이미 `SESSION_COOKIE`를 proxy(edge)와 서버 코드가 함께 쓰는 자리라 위치가 맞다.

## 2. 변경 파일 내역

| 구분 | 파일 | 내용 |
|---|---|---|
| 수정 | `src/lib/constants.ts` | `AFTER_LOGIN_PATH = "/projects"` 추가 |
| 수정 | `src/components/features/auth/login-form.tsx` | `router.replace("/")` → `router.replace(AFTER_LOGIN_PATH)` |
| 수정 | `src/proxy.ts` | 로그인 상태의 `/login` 리다이렉트 대상을 상수로 |
| 추가 | `docs/109-feat-로그인-후-첫-화면을-프로젝트로-변경.md` | 본 문서 |

> 문서 번호는 작성 시점에 107이었으나 머지 직전 다른 브랜치들이 107(중복)·108을 선점해, "나중에 머지되는 쪽이 번호를 올린다"는 규칙에 따라 **109**로 조정했다.

## 3. 건드리지 않은 것과 그 이유

- **`/`(운영 대시보드)는 그대로 둔다.** 상단 메뉴의 `운영` 탭이 여전히 `/`를 가리키고 정상 동작한다. 로그인 랜딩만 바뀐 것이지 운영 섹션을 없앤 게 아니다.
- **상단 메뉴 순서도 그대로다** (`운영 · 프로젝트 · 위키`). 요청은 "로그인하면 프로젝트가 먼저 나오게"였고, 메뉴 순서를 바꾸는 것은 다른 화면(운영에서 시작하는 흐름)에도 영향이 커서 범위 밖으로 봤다. 순서 변경을 원하면 `top-nav.tsx`의 `SECTIONS` 배열 순서만 바꾸면 되는 한 줄 작업이다.
- **비로그인 가드는 그대로다.** 보호 경로에 세션 없이 접근하면 여전히 `/login`으로 간다.

## 5. 작업 중 발견한 것 — 랜딩 화면이 비어 보일 수 있다

검증 중 로그인 직후 화면의 **콘텐츠 영역이 완전히 비어 있었다.** 원인을 확인해 보니 제 변경과 무관한 **데이터 조건**이었다.

- 공유 개발 DB에 프로젝트가 0개다 (docs/46에서 시드를 그룹 골격만 남기고 축소).
- `/projects`가 렌더하는 `TaskStatusPage`는 **프로젝트가 없을 때 빈 상태 안내 없이 아무것도 그리지 않는다.**
- 프로젝트를 1개 만들고 같은 경로를 다시 열자 `작업 현황` 화면(그룹 선택·로드맵)이 정상 렌더됐다.

즉 기능은 정상이지만, **워크스페이스가 비어 있는 계정은 로그인 첫 화면이 백지로 보인다.** 이전에는 `/`(운영 대시보드)로 떨어져 최소한 "대시보드 콘텐츠는 이후 태스크에서 구현됩니다" 문구라도 있었기 때문에, 이번 변경으로 이 공백이 더 눈에 띄게 됐다.

`TaskStatusPage`에 빈 상태(프로젝트가 없을 때의 안내·생성 유도)를 넣는 것은 이번 요청 범위 밖이라 손대지 않고 후속 과제로 남긴다.

## 6. 알려진 사항 / 후속 과제

- **프로젝트가 0개인 계정의 랜딩이 백지다.** `TaskStatusPage` 빈 상태 추가가 필요하다 (위 5절).
- 랜딩 경로가 전 사용자 공통이다. 권한(마스터/스탭)이나 개인 설정에 따라 다르게 하려면 `AFTER_LOGIN_PATH`를 함수로 바꿔야 한다 — 지금은 그런 요구가 없어 상수로 뒀다.
- 검증용 임시 프로젝트는 스크립트가 만들고 끝나면 삭제했다 (삭제 status 200 확인).
