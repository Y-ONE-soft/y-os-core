## 1. 작업 내용 요약

프로젝트 상세의 **단계 로드맵**을 작업 현황 로드맵과 동일한 방식으로 전환했다 (사용자 요청: "바를 움직이고 옮기고 할 수 있게").

| 항목 | 이전 | 이번 |
|---|---|---|
| 타임라인 | 고정 창(2026-07-15 + 4주), 화면 폭에 압축 | **지라식 배율 타임라인** — 오늘 앞뒤로 길게 펼쳐지고 가로 스크롤 |
| 일·주·개월·분기 | 시각 자리표시(동작 없음) | **실제 배율 전환** |
| 오늘 | 없음 | **오늘 위치로 스크롤** |
| 막대 | 정적 표시 | **드래그로 이동 + 양 끝 손잡이로 기간 조절**, 놓는 순간 DB 저장 |
| 단계 번호 | 없음 | 막대에 순번 배지 (작업 현황과 동일) |
| 연도 | 없음 | 헤더 기간 + 눈금 연도 변경 지점 표시 |

## 2. 주요 구현 결정

| 항목 | 결정 | 이유 |
|---|---|---|
| 막대 컴포넌트 | `workload-roadmap.tsx` 내부의 `Bar`(≈200줄)를 **`roadmap-bar.tsx`로 추출**해 두 화면이 공유 | 드래그 로직(포인터 캡처·임계값·미리보기·커밋)을 복제하면 한쪽만 고쳐지는 사고가 난다. 실제로 docs/14에서 겪은 교차 의존 문제와 같은 부류 |
| 작업 현황 쪽 변경 | import만 교체 (동작·스타일 동일) | 회귀 위험 최소화. 추출 직후 빌드로 확인 |
| 라벨 열 폭 | 프로젝트 상세는 180px 유지 (작업 현황 200px) | 기존 화면 폭 유지 — 상세는 프로젝트명이 없어 좁아도 된다 |
| `프리셋 저장` 버튼 | 제거 | 동작 정의가 없던 자리표시. 헤더에 기간 표기가 들어가며 공간 정리 |
| 저장 시점 | 드래그 중에는 미리보기, **손을 뗄 때 1회 저장** | 드래그마다 API를 때리지 않기 위함 (공유 로직) |
| 드래그 vs 클릭 | 3px 미만 이동은 클릭(단계 상세 열기)으로 처리 | 공유 로직의 기존 규칙 그대로 |

## 5. 알려진 사항 / 후속 과제

- 세로 드래그(단계 순서 변경)는 미구현 — 가로 이동·기간 조절만
- 막대 드래그 시 스크롤 경계에서 자동 스크롤(auto-pan) 없음 — 창 밖으로 끌려면 스크롤 후 다시 잡아야 함
- 작업 현황·프로젝트 상세가 이제 같은 막대를 쓰므로, 드래그 관련 개선은 `roadmap-bar.tsx` 한 곳만 고치면 양쪽에 반영된다
