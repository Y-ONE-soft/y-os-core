## 1. 작업 내용 요약

Figma **Stage Detail Overlay**(`130:414`)와 **Collaborator Request** 변형(`142:1209`)을 구현했다. 프로젝트 상세의 보드 컬럼 단계명 / 로드맵 단계 행을 클릭하면 대형 오버레이가 열리고, 편집은 **즉시 저장**(디자인 문구 그대로)으로 보드 스토어에 반영된다.

### 구성

- **헤더**: [단계] 칩 + `{프로젝트명} · 로드맵` 브레드크럼 + 닫기
- **본문 좌측**: 제목(체크박스 = 단계 완료, 완료 시 취소선) · 내용 텍스트영역(blur 시 저장) · 산출물 드롭존(자리표시) · 연결 티켓·위키(자리표시) · **댓글(실동작 — 등록·목록·상대시간·작성자=세션 사용자)**
- **우측 세부 사항**: 프로젝트(도트+이름) · 시작/종료일(변경 즉시 저장 → 로드맵·보드 반영) · **공동 작업자 지정 요청 버튼**(요청 수 배지) · 데드라인 표시 토글 · "변경사항은 즉시 저장됩니다 / 생성·수정 시각"
- **공동 작업자 지정 요청**(중첩 다이얼로그): 멤버 체크 선택(자리표시 4명 — 김서연·박지훈·이민아·최현우) + 선택 요약("N 외 M명 선택됨") + 메시지 + 요청 보내기 → 스토어에 요청 멤버 기록

## 2. 주요 결정

| 항목 | 결정 | 이유 |
|---|---|---|
| 데이터 | `BoardStage` 확장: `done/description/comments/requestedCollaborators/createdAt/updatedAt` + `updateStage`/`addComment` 액션 | 기존 보드 스토어 단일 소스 유지 — 날짜 수정이 로드맵 막대에 즉시 반영 |
| 즉시 저장 | 토글·체크·날짜는 onChange, 내용은 onBlur | 디자인 명세("변경사항은 즉시 저장됩니다") |
| 내용 편집 상태 | `DescriptionEditor` 서브컴포넌트 + `key={stage.id}` 리마운트 | effect 내 setState 린트 금지 회피 (v13과 동일 규칙 준수) |
| 멤버 목록 | `TEAM_MEMBERS` 상수 자리표시 | 멤버 도메인 미구현 — 디자인의 4명 그대로 |
| 산출물·티켓 연결 | 비주얼만 | 파일/위키 도메인 부재 — 후속 태스크 |
| 진입점 | 보드 컬럼 단계명 + 로드맵 단계 행 (hover 밑줄) | 상세 페이지 컨텍스트에서만 (작업 현황 로드맵 연결은 후속) |
