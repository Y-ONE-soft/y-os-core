- **예정 커밋 메시지**: `refactor: 작업 현황 캘린더를 내 할일 캘린더와 동일화 (읽기 전용)`
- **작업일**: 2026-07-22
- **작업 브랜치**: `작업현황-캘린더-내작업-동일화` (워크트리 `.claude/worktrees/status-cal-reuse`)

## 1. 작업 내용 요약

직전(docs/96)에 작업 현황 캘린더를 Figma `Task Status Layout — Master · Calendar`(210:969)의 **단순 월 달력**(날짜 칸 + 시작/마감 칩)으로 만들었는데, 사용자 요청이 **"내 할일 캘린더와 거의 동일하게, 수정만 못하게"** 로 바뀌었다.

그래서 자체 구현을 걷어내고 **`MyWorkCalendar` 컴포넌트를 그대로 재사용**한다.

| | 내 할일 캘린더 | 작업 현황 캘린더 |
|---|---|---|
| 화면 | 프로젝트 박스 + 단계 막대 + 할일 칩 | **동일** |
| 월 이동·오늘·"이 달 N건" | 있음 | **동일** |
| 대상 범위 | 내가 작업자인 프로젝트 | 페이지가 계산한 범위 (마스터 = 전체, 스탭 = 자기 것) |
| 단계 드래그(이동·기간 조절) | 가능 | **불가** |
| 할일 드래그(날짜 이동)·백로그 드롭 | 가능 | **불가** |
| 체크박스 완료 | 가능 | **비활성** |
| 단계·할일 상세 열기 | 가능 | **가능** (조회는 막지 않는다) |

## 2. 구현 메모

### 읽기 전용은 "핸들러를 주지 않는 것"으로 표현된다

`MyWorkCalendar`는 편집 동작을 전부 **선택적 콜백**으로 받는다. `onDrag`·`onDropTask`·`onToggleTask`를 넘기지 않으면

- `draggable = Boolean(onDragStart)` → 드래그 커서·리사이즈 손잡이가 아예 렌더되지 않고
- 백로그 드롭 핸들러도 붙지 않는다

즉 **컴포넌트 수정 없이 읽기 전용이 된다.** 다만 체크박스만은 예외였다 — `onToggleTask`가 없어도 버튼이 그대로 렌더되어 **눌러도 아무 일이 없는 컨트롤**이 됐다. 그래서 `disabled={!onToggleTask}`를 추가했다(고장으로 읽히는 것을 막는 최소 변경).

### 데이터

`buildCalendarSource(grid, projects, boards)` → `buildWeekLayouts` 파이프라인을 그대로 쓴다. 넘기는 `projects`만 `visibleProjects`(페이지가 권한·필터를 반영해 계산한 값)로 바뀐다. **권한 분기 코드는 여기에 없다.**

미배정 할일(`unassigned`)은 넘기지 않는다 — 작업 현황은 프로젝트 범위를 보는 화면이라 프로젝트 없는 할일은 대상이 아니다.

### 삭제된 것

docs/96에서 만든 자체 월 달력(날짜 칸 + `시작일/마감` 칩) 구현은 제거했다. Figma 210:969의 단순 달력 형태는 더 이상 쓰지 않는다 — 사용자 요청이 우선.

## 5. 알려진 사항 / 후속 과제

- **상세 오버레이 안에서는 수정이 가능하다.** "수정만 못하게"는 캘린더 조작(드래그·완료 체크)을 막는 것으로 해석했고, 상세는 조회 진입점으로 열어 뒀다. 상세까지 읽기 전용으로 원하면 별도 요청 필요
- 필터(그룹·프로젝트 칩)는 캘린더에도 그대로 적용된다 — `visibleProjects`를 공유
- 담당자 뷰는 여전히 미구현(비활성) — docs/96에서 정한 그대로

## 6. 병렬 작업
