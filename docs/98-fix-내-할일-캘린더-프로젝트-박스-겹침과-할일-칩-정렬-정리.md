# 98. fix: 내 할일 캘린더 프로젝트 박스 겹침과 할일 칩 정렬 정리

내 할일 캘린더 뷰에서 보고된 두 가지 어긋남을 고친다. 둘 다 브라우저에서 실제 좌표를 재서 원인을 확인하고, 고친 뒤 같은 방법으로 다시 쟀다.

| 증상 | 수정 전 | 수정 후 |
|---|---|---|
| 위아래 프로젝트 박스 사이 세로 간격 | **-4px** (겹침) | **+4px** |
| 할일 칩의 체크박스 ↔ 제목 세로 중심 차 | **+2.5px** (제목이 아래) | **0px** |

## 1) 프로젝트 박스가 4px씩 겹쳐 있었다

"간격이 좁다"가 아니라 **실제로 포개져 있었다.** 박스 기하가 이랬다.

```tsx
top: laneTop(box.lane) - 3,
height: box.lanes * LANE_PX + 4,
```

`laneTop(lane) = DATE_ROW_PX + lane * LANE_PX`이므로, 레인 `L`부터 `n`칸을 쓰는 박스의 아래끝은

```
laneTop(L) - 3 + n * LANE_PX + 4 = laneTop(L + n) + 1
```

인데, 바로 아래 박스는 `laneTop(L + n) - 3`에서 시작한다. **박스 높이가 자기 레인 블록보다 커서 아래 박스를 4px 침범**하고 있었다.

근본 원인은 여유가 없는 레인 높이였다. 레인 26px에 할일 칩 20px·단계 막대 22px가 들어가면 남는 4~6px로 위아래 박스가 각자 안쪽 여백을 낼 수 없다. 그래서 두 박스가 서로의 여백을 나눠 쓰다가 겹쳤다.

**수정** — 레인 높이를 28px로 올리고, 여백·간격을 이름 있는 상수로 드러냈다.

```tsx
const LANE_PX = 28; // 26 → 28
const BOX_PAD_PX = 2; // 테두리와 내용 사이 여백
const BOX_GAP_PX = 4; // 위아래 프로젝트 박스 사이 간격

top: laneTop(box.lane) - BOX_PAD_PX,
height: box.lanes * LANE_PX - BOX_GAP_PX,
```

이제 박스 높이가 레인 블록보다 `BOX_GAP_PX`만큼 **작아서** 다음 박스와 그만큼 벌어진다.

```
아래끝   = laneTop(L) - 2 + n * 28 - 4 = laneTop(L + n) - 6
다음 위끝 = laneTop(L + n) - 2          → 간격 4px
```

레인 28px = 할일 칩 20 + 여백 2 + 간격 4 + 여백 2로 맞춘 값이다. 마지막 레인에 든 할일 칩 아래로 2px 여백이 남아 위쪽 여백과 대칭이 된다. **이 다섯 상수(`LANE_PX`, `TASK_PX`, `STAGE_PX`, `BOX_PAD_PX`, `BOX_GAP_PX`)는 함께 움직인다** — 하나만 바꾸면 박스가 다시 겹치거나 내용이 테두리에 닿는다. 코드에도 주석으로 남겼다.

단계 막대(22px)는 항상 박스의 첫 레인에만 놓이고(`MIN_BOX_LANES = 2`이므로 마지막 레인이 될 수 없다) 아래로 6px이 남아 여유가 있다.

## 2) 할일 칩의 제목이 체크박스보다 2.5px 내려가 있었다

칩은 `flex items-center`라 두 자식이 모두 가운데 정렬되어야 하는데, 체크박스 중심은 10px(칩 높이 20의 정확한 절반)인 반면 제목 중심은 12.5px이었다.

원인은 제목을 감싼 버튼이었다.

```tsx
<button className="min-w-0 flex-1 truncate text-left ...">
  <span className="truncate text-[10px] font-medium">{overlay.label}</span>
</button>
```

**버튼에는 글자 크기가 없다.** 그래서 버튼의 라인박스는 상위에서 물려받은 큰 폰트 기준(측정값 line-height 15px 이상)으로 잡히고, 그 안의 10px 제목은 라인박스 가운데가 아니라 **베이스라인에 걸려** 아래로 밀린다. 인라인 `<span>`이 라인박스 안에서 베이스라인 정렬되는 기본 동작이라, 바깥 `items-center`로는 잡히지 않는다.

**수정** — 버튼 자체를 세로 가운데 정렬하는 flex 컨테이너로 만든다. 체크박스 버튼이 이미 쓰는 방식과 같다.

```tsx
<button className="flex min-w-0 flex-1 items-center text-left ...">
  <span className="min-w-0 flex-1 truncate text-[10px] font-medium">…</span>
</button>
```

- 버튼에서 `truncate`를 떼고 제목 `<span>`으로 옮겼다. 이제 span이 flex 아이템이라 블록화되어 말줄임이 정상 동작한다.
- span에 `min-w-0`을 함께 준 이유: **flex 아이템은 기본 `min-width: auto`라 내용보다 좁아지지 않는다.** 이게 없으면 긴 제목이 칩 밖으로 삐져나온다.

## 검증

로컬 dev 서버(포트 3177, `Win32_Process` CommandLine으로 이 워크트리 소유임을 확인)에서 puppeteer로 DOM 좌표를 직접 측정했다.

실제 데이터의 캘린더가 비어 있어(모든 할일이 예정일 없는 백로그 상태) 측정 대상이 없었다. **검증 전용 그룹·프로젝트 2개를 만들어** 기간이 겹치도록 배치해 박스가 세로로 쌓이게 한 뒤 측정하고, 끝나고 그룹째 삭제했다 (docs/94의 사고 재발 방지).

**수정 전**

```
박스간격: [{주:1, 위:"검증프로젝트 가", 아래:"검증프로젝트 나", 세로간격:-4},
          {주:2, 미배정↔미배정, 세로간격:-4}, {주:3, 세로간격:-4}]
할일칩  : 칩높이 20, 체크박스중심 10, 제목중심 12.5, 어긋남 2.5  (×5건 전부 동일)
```

**수정 후**

```
박스간격: 세로간격 4 (3주 모두)
할일칩  : 체크박스중심 10, 제목중심 10, 어긋남 0  (×5건 전부 동일)
```

**말줄임 회귀 확인** — `truncate`를 버튼에서 span으로 옮겼으므로 긴 제목이 여전히 잘리는지 따로 확인했다. 칩 너비를 넘는 긴 이름의 할일을 추가해 측정.

```
{할일:"아주아주긴할일이름으로칩너비", 칩너비:109, 제목너비:86, 넘침:-4, 말줄임:true}
```

`말줄임: true`(scrollWidth > clientWidth)이고 `넘침: -4`로 칩 안에 머문다. 짧은 이름들은 `말줄임: false`로 그대로 표시된다.

3배 확대 촬영으로 체크박스와 제목이 한 줄에 맞는 것도 눈으로 확인했다.

`npm run lint`, `npm run build` 통과. 콘솔 에러 0건.

## 알려진 이슈

- **주 행이 레인당 2px씩 높아진다.** 레인 3줄짜리 주라면 6px, 6주 달력이면 최대 수십 px 늘어난다. 캘린더는 `flex-1`이라 자연스럽게 늘어나지만, 화면이 작으면 아래 "요청 알림" 영역이 더 밀린다.
- **`MIN_BOX_LANES = 2` 때문에 단계만 있고 할일이 없는 프로젝트 박스는 아래 절반이 빈다.** 기존 의도("단계 막대 두 개 두께")를 그대로 뒀고 이번 수정 범위 밖이다.
- 단계 막대 라벨(`overlay.label`)도 같은 인라인 span 구조지만, 그쪽 버튼은 이미 `flex ... items-center`라 같은 증상이 없다.
