# 137. feat: 프로젝트 생성 그룹칸을 이름 위로 올리고 권한별 표시

## 작업 요약

내 할일 > 프로젝트 생성 다이얼로그를 손봤다.

- **그룹 칸을 프로젝트 이름 위로** 올렸다 (어느 그룹에 만들지 먼저 정하는 흐름).
- **그룹 칸을 스탭·마스터 모두**에게 보여준다. 이전엔 마스터에게만 있었다.
- **스탭**은 담당 소속이 정해져 있어 **비활성(disabled)** 으로 표시한다.
- **마스터**는 편집 드롭다운 + 설명을 둔다.

## 변경 내용

### `src/components/features/projects/project-create-dialog.tsx`

**순서 교체** — `그룹` → `프로젝트 이름` 순으로. (이전: 이름 → 그룹)

**스탭** — 자기 소속 그룹을 비활성 `<select>`로 보여준다. 값도 못 바꾸고, 서버도 세션 그룹으로 강제하므로 표시 전용이다. 그룹 이름은 프로젝트 스토어에서 찾고(`myGroupName`), 로드 전에는 "내 소속 그룹" 자리표시를 쓴다. 아래에 안내: "담당 소속으로 지정됩니다. 소속은 관리자가 관리합니다."

**마스터** — 기존 편집 드롭다운 그대로(기본값은 내 정보에서 지정한 대표 그룹, 다른 그룹도 선택 가능). 라벨 아래에 설명을 추가했다:

> 마스터 권한이라 내 정보에서 지정한 대표 그룹으로 기본 설정됩니다. 필요하면 아래 목록에서 바꿔주세요.

**마스터 권한 때문에** 대표 그룹이 기본으로 잡힌다는 점을 문구에 명시했다.

## 바꾸지 않은 것

- 서버(`api/admin/projects/route.ts`) — 스탭은 세션 그룹, 마스터는 선택 그룹을 쓰는 로직 그대로. 스탭의 비활성 칸은 순수 표시라 서버 판정과 무관하다.
- 제출 조건(`sharedReady`) — 마스터는 그룹이 잡혀 있어야 하고(기본값으로 채워짐), 스탭은 그룹 값을 제출하지 않는다. 변화 없음.
