# fix: 내 할일 캘린더 단계 드래그 대칭 이동 및 이동 시 변형 방지

## 요청

> 내 할일 캘린더 뷰에서 단계 드래그가 잘 안되는 경우가 있다. 자연스럽지 않고, 특정 날짜에서 더 뒤로 가거나 안 움직인다. 전반적으로 자연스럽고 문제 없게 해달라.

## 원인 진단

내 할일 캘린더(`my-work` 캘린더 패널)의 단계 막대 드래그에서 두 가지가 "부자연스러움 / 특정 날짜에서 뒤로 감"을 만들고 있었다.

### 원인 1 — 포인터→날짜 델타가 비대칭 (`my-work-calendar.tsx`)

`dayFromPointer`가 `Math.floor(((clientX - rect.left) / rect.width) * 7)`로 **포인터가 놓인 절대 칸 인덱스**를 구하고, 델타 = `지금칸 − 누른칸`으로 계산했다.

- floor는 "칸 경계를 넘는 순간" 값이 바뀌므로, 막대의 **어느 위치를 잡았느냐**에 따라 앞/뒤로 한 칸 넘어가는 데 필요한 이동량이 달라진다.
- 예: 칸 왼쪽 끝을 잡으면 조금만 왼쪽으로 가도 즉시 −1로 튀고(뒤로 쉽게 감), 오른쪽으로는 거의 한 칸을 끌어야 +1이 된다.
- 이 비대칭이 "자연스럽지 않다 / (잡은 위치에 따라) 뒤로 손쉽게 튄다"로 드러났다. 또 near-click(살짝 눌러 뗌)에서도 경계 근처면 우연히 ±1 이동이 커밋되는 부작용이 있었다.

### 원인 2 — 할일이 있는 단계를 "이동"하면 막대가 변형됨 (`my-work-calendar-panel.tsx` + `roadmap-utils.ts`)

이동(move) 처리가 `clampStageToTasks`를 써서 결과를 **늘려서** 할일을 덮었다.

```
startDate = dates.startDate > cover.min ? cover.min : dates.startDate;  // 앞으로 끌면 시작이 첫 할일에 붙어 뒤로 당겨짐
endDate   = end < cover.max ? cover.max : end;                          // 뒤로 끌면 끝이 마지막 할일에 붙어 안 나감
```

- 할일 예정일(= "특정 날짜")을 지나 **앞으로** 끌면 시작이 그 날짜에 붙어 **뒤로 당겨지고** 끝만 늘어나 막대가 변형됐다.
- **뒤로** 끌면 반대로 끝이 그 날짜에 붙어 안 움직였다.
- 즉 막대가 통째로 미끄러지지 않고 할일 날짜에 "낚여" 늘어나거나 멈췄다. 이게 "특정 날짜에서 더 뒤로 가거나, 그렇지 않아(안 움직임)"의 실체다.
- 참고: `clampStageToTasks`의 주석은 이미 *"이동: 길이를 유지한 채 덮개를 깨지 않는 범위로 이동량을 제한한다"* 라고 **rigid 이동을 의도**하고 있었으나, 구현은 stretch였다. 이번 수정으로 이동 경로가 그 의도와 일치하게 된다.

## 변경 내용

### 1. `src/components/features/projects/roadmap-utils.ts` — `clampMoveDelta` 추가

이동 전용 헬퍼. 막대를 늘리지 않고 **길이를 유지한 채** 할일 덮개를 깨지 않는 선까지 **이동량 자체**를 제한한다.

```ts
export function clampMoveDelta(
  startDate: string,
  endDate: string | undefined,
  delta: number,
  cover: { min: string; max: string } | null,
): number {
  if (!cover) return delta;
  const end = endDate ?? shiftISO(startDate, OPEN_ENDED_DAYS - 1);
  const maxForward = Math.max(0, dayOffset(cover.min, startDate)); // 시작이 첫 할일까지 전진
  const maxBackward = Math.min(0, dayOffset(cover.max, end));      // 끝이 마지막 할일까지 후퇴(≤0)
  return Math.min(maxForward, Math.max(maxBackward, delta));
}
```

- `cover`(할일 예정일 최소~최대)가 없으면 제한 없음.
- 이미 할일을 덮고 있지 않은(데이터 드리프트) 경우 그 방향을 `0`으로 막아 **강제로 끌려가지 않게** 한다(`Math.max(0, …)` / `Math.min(0, …)`).
- 기존 `clampStageToTasks`는 **그대로 유지** — 할일 이동 시 단계가 늘어 덮는 경로(task-detail-overlay, my-work-timeline-panel, 캘린더의 할일 이동)와 start/end 조절 경로는 stretch가 옳으므로 건드리지 않았다.

### 2. `src/components/features/my-work/my-work-calendar-panel.tsx` — 이동 분기를 rigid로

`nextStageDates`의 `mode === "move"`만 `clampMoveDelta`로 이동량을 제한하고 `dragStageDates("move", …)`로 **변형 없이** 옮긴다. start/end 조절은 기존 `clampStageToTasks` 유지.

```ts
const cover = taskDateRange(stage.tasks);
if (mode === "move") {
  const delta = clampMoveDelta(stage.startDate, stage.endDate, deltaDays, cover);
  return { projectId: project.id, dates: dragStageDates("move", stage.startDate, stage.endDate, delta) };
}
const dragged = dragStageDates(mode, stage.startDate, stage.endDate, deltaDays);
return { projectId: project.id, dates: clampStageToTasks(dragged, cover) };
```

`clampMoveDelta` import 추가.

### 3. `src/components/features/my-work/my-work-calendar.tsx` — 델타 대칭 반올림

`dayFromPointer`(floor 절대 칸) → `pointerDay`(연속값)로 바꾸고, 델타는 `(지금 − 누른 위치)`를 **한 번만 반올림**해 구한다.

- `pointerDay`: 세로는 행 사각형으로 주 인덱스를 찾고(주 높이 제각각 대응), 가로는 `colFraction`을 **반올림 없이** 그대로 돌려준다. `[0, lastDay]` 클램프는 유지(그리드 밖 방지, 좌우로 끌면 앞뒤 주로 연속).
- `handlePointerMove` / `handlePointerUp`: `const delta = Math.round(pointerDay(...) - drag.startDay);`
  - 막대의 어느 지점을 잡든 **앞뒤로 반 칸을 넘겨야 하루가 바뀐다(대칭)**. floor의 잡은-위치 의존 문턱이 사라진다.
- `handlePointerUp`: 끌었다가 **같은 날로 돌아오면(델타 0) `commit` 대신 `cancel`** — 미리보기만 되돌리고 불필요한 저장(no-op 쓰기)을 하지 않는다.
- 클릭/드래그 구분(3px 임계값)과 루트 `setPointerCapture`(막대가 주 경계를 넘어 언마운트돼도 캡처 유지)는 그대로 둔다.

## 결정 이유

- **왜 stretch가 아니라 이동량 제한(rigid)인가**: "단계는 자기 할일을 항상 덮는다"는 불변식은 유지하되, **이동**은 막대 모양이 변하지 않아야 자연스럽다. 늘려 덮으면 시작이 뒤로 당겨져 사용자가 "뒤로 갔다"고 느낀다. 이동량만 경계에서 멈추면 막대가 통째로 미끄러지다 딱 멈춘다.
- **왜 floor가 아니라 이동량 반올림인가**: 절대 칸(floor)은 잡은 위치에 따라 문턱이 비대칭이라 방향 편향이 생긴다. 이동량 기준 단일 반올림은 대칭이라 "잡은 곳과 무관하게 반 칸 = 하루".
- **왜 `clampStageToTasks`를 안 건드렸나**: 할일 이동(할일이 단계 밖으로 나가면 단계가 늘어 덮음)과 start/end 조절에는 stretch가 정답이라 공유 함수를 그대로 두고, 이동 경로만 새 헬퍼로 분기했다. 영향 범위를 이동 드래그로 한정.
- **범위**: 요청이 "내 할일 캘린더"라 캘린더 패널만 고쳤다. 프로젝트 로드맵 막대(`roadmap-bar.tsx`)·내 할일 타임라인(`my-work-timeline-panel.tsx`)도 이동 시 같은 stretch 변형이 있으나, 별도 요청 시 `clampMoveDelta` 재사용으로 확장 가능(이번 범위 아님).
