# 85. fix: 프리셋 저장 버튼을 단계 로드맵 카드 헤더로 이동

## 작업 요약

프리셋 저장 버튼을 **페이지 헤더 제목 줄 오른쪽 → 단계 로드맵 카드 헤더**로 옮기고, 사이드바 `프리셋` 메뉴와 같은 아이콘을 붙였다.

## 그동안 무엇을 놓쳤나

이 버튼을 세 번 만졌고 그중 두 번은 크기만 바꿨다.

| 문서 | 조치 | 결과 |
| --- | --- | --- |
| docs/70 | 페이지 헤더 우측, `default`(h-8) | — |
| docs/78 | `lg`(h-9) — Figma 36px에 맞춤 | "혼자 크다" |
| docs/81 | `sm`(h-7) — 화면 밀도에 맞춤 | "더 이상해졌다" |

**크기 축에서 위아래로 움직인 것뿐이라 나아질 수 없었다.** 문제는 크기가 아니라 두 가지였다.

1. **자리** — 페이지 폭이 넓어(약 1,250px) 제목과 버튼 사이가 1,600px 가까이 벌어졌다. Figma 프레임(1184px)에서는 붙어 보이지만 실제 화면에서는 완전히 분리된 요소로 읽힌다.
2. **정체성** — 이 버튼만 아이콘이 없었다. 사이드바 워크스페이스 메뉴(작업 현황·내 할일·작업 분석·프리셋), 로드맵 `＋ 단계`, 백로그 입력 모두 시각적 표식을 갖는데 이것만 텍스트 든 회색 테두리 상자였다.

## 왜 단계 로드맵 카드 헤더인가

- **저장 대상이 그 카드의 내용이다.** 프리셋은 단계 구성과 각 단계의 할일을 담는다. 섹션에 작용하는 액션은 그 섹션 헤더에 두는 것이 통상적이다
- `＋ 단계`·레인지 스위처와 **형제 관계**가 된다 — 셋 다 단계 구성을 다루는 컨트롤이다
- 카드 폭 안이라 **넓은 화면에서 혼자 떠 있는 문제가 구조적으로 사라진다**
- docs/18에서 원래 있던 자리이기도 하다 (이후 로드맵 헤더 재구성 때 사라졌었다)

## 배운 것

같은 버튼을 네 번 손봤는데, 세 번째까지는 전부 **"어디에 있어야 하는가"를 묻지 않고 "얼마나 커야 하는가"만 조정**했다. 배치가 틀린 요소는 크기를 어떻게 잡아도 어색하다.

Figma의 좌표를 그대로 따르는 것도 함정이었다. 디자인 프레임 폭(1184px)과 실제 렌더 폭이 다르면 **같은 좌표가 다른 관계를 만든다.**
