## 1. 작업 내용 요약

프로젝트 상세 화면에서 로드맵 아래 **단계 보드(칸반) 컬럼이 카드 내용만큼만 작게** 표시되던 것을, Figma `Project Detail Layout`(96:166)처럼 **보드 영역 전체 높이를 채우는 패널**로 정합했다.

- 디자인: 로드맵 카드는 위쪽에 컴팩트하게(822×187), 칸반 컬럼은 그 아래 남은 높이를 모두 차지(260×**448**), 우측 백로그는 전체 높이(300×649)
- 기존 구현: 칸반 컨테이너에 `items-start`가 걸려 있어 각 컬럼이 **내용 높이로 수축** — 카드가 1~2개인 단계는 얇은 띠처럼 보였고, 단계마다 높이가 제각각이라 보드로 읽히지 않았다

## 2. 원인과 수정

| 항목 | 변경 전 | 변경 후 |
|---|---|---|
| 칸반 컨테이너 | `flex min-h-0 flex-1 items-start …` — `items-start`가 컬럼을 내용 높이로 수축 | `items-start` 제거 → 기본 `stretch`로 남은 높이를 꽉 채움 |
| 컬럼 `<section>` | 높이 제약 없음 | `min-h-0` 추가 — 카드가 많아도 컬럼이 보드 밖으로 밀려나지 않음 |
| 컬럼 헤더 | 수축 가능 | `shrink-0` — 카드가 많아도 헤더 높이 유지 |
| 카드 영역 | `<section>` 직계 자식 | `flex min-h-0 flex-1 flex-col gap-1.5 overflow-y-auto` 래퍼로 분리 — **카드 영역만 세로 스크롤**, 컬럼 자체는 고정 높이 |
| 카드·추가 버튼 | 기본(수축 가능) | `shrink-0` — 스크롤 컨테이너 안에서 카드가 눌리지 않도록 |

핵심은 `items-start` 제거 1줄이지만, 그것만으로는 카드가 많을 때 컬럼 내용이 보드 밖으로 넘치므로 **내부 스크롤 컨테이너**를 함께 도입했다.
