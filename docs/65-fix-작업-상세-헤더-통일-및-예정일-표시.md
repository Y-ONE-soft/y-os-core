- **작업 브랜치**: `티켓-헤더-예정일` (워크트리 `.claude/worktrees/ticket-header-schedule`, base: main `3a2c1c1`)

## 1. 작업 요약

사용자 지적 두 가지를 고쳤다.

1. **예정일이 어디에도 안 보였다** — `Task.scheduledDate`(docs/49)를 저장하고 캘린더에서 드래그로 옮기게까지 해놓고, 정작 **작업 상세에서는 값을 볼 수도 고칠 수도 없었다.** 세부 사항에 예정일 입력을 추가했다.
2. **헤더의 `YOS-202`가 뜻이 없었다** — 키 체계가 없어 작업 id 해시로 만들어 쓰던 가짜 티켓 번호다. 걷어내고 **단계 상세와 같은 표기**로 맞췄다.

## 2. 변경 내용

### 2-1. 헤더를 단계 상세와 통일

| | 단계 상세 | 작업 상세 (이전) | 작업 상세 (이후) |
|---|---|---|---|
| 칩 | `단계 3` | `YOS-202` (해시 기반 가짜 키) | **`작업 1`** |
| 브레드크럼 | `프로젝트명 · 로드맵` | `프로젝트명 · 단계명` | 그대로 (위치를 알려주는 같은 역할) |

칩은 **소속 안에서의 순번**이다 — 단계에 편성된 작업은 그 단계 안 순번, 백로그·미배정 작업은 각 목록 안 순번. 해시 키를 만들던 `taskKeyNumber()`는 삭제했다.

> 브레드크럼은 단계 상세의 `프로젝트 · 로드맵`과 같은 자리·같은 역할(이 항목이 어디 있는지)이라 유지했다. 표기가 `프로젝트 · 백로그`처럼 보이는 건 그 작업이 실제로 백로그에 있기 때문이다.

### 2-2. 세부 사항에 예정일

단계 상세의 시작일·종료일과 **같은 형태**(`Input type="date"`, 라벨 위)로 프로젝트·단계 아래에 넣었다.

- 값을 바꾸면 **캘린더 드래그와 같은 규칙**이 적용된다 — 단계 밖 날짜를 고르면 **단계가 그 날짜까지 늘어나 덮는다**(`clampStageToTasks` 재사용)
- **단계가 없는 작업**(백로그·미배정)은 입력이 비활성이고 "단계에 편성하면 예정일이 잡힙니다" 안내를 띄운다 — 단계를 벗어나면 예정일이 해제되는 규칙(docs/51)과 맞춘 것

## 5. 알려진 사항 / 후속 과제

- 작업 유형·난이도는 여전히 로컬 state 자리표시 (docs/19 범위 그대로)
