# feat: 캘린더 할일 추가 시 겹친 단계 중 선택

## 요청

> 같은 프로젝트 단계가 **중복이면 어떤 단계에 포함할건지 선택하게** 해줘.

캘린더의 단계 막대는 기간이 겹쳐도 나누지 않고 한 줄에 겹쳐 그린다(레이아웃 규칙 4). 그래서 한 날짜를 **여러 단계가 동시에 덮을 수 있다.** 앞 커밋(#156)은 이 경우 최상단(첫) 단계를 기본으로 넣었는데, 이 커밋은 **사용자가 직접 고르게** 한다.

## 동작

- 날짜 칸을 눌러 뜬 인라인 입력창에서, 그 날짜를 **덮는 단계가 둘 이상이면** 입력창 아래에 **단계 선택 칩**이 뜬다.
- 칩은 각 단계의 색·이름을 보여주고, 선택된 칩은 그 색으로 강조된다. 기본 선택은 최상단(첫) 단계.
- 다른 칩을 누르면 선택이 바뀌고, 입력창 위 점 색·플레이스홀더(`"<단계명>에 추가"`)도 그 단계로 갱신된다.
- 이름 입력 후 Enter → **선택된 단계**로 할일이 들어간다(예정일=그 날).
- 덮는 단계가 0~1개면 칩은 안 뜬다(선택할 게 없다).

## 구현 세부

파일: `src/components/features/my-work/my-work-calendar.tsx` — `AddTaskInput`만 수정.

- 선택 상태를 `useState`로 관리: `const [stageId, setStageId] = useState(stages[0]?.stageId ?? null)` (앞 커밋은 setter 없이 첫 단계 고정이었다).
- 팝오버를 세로(`flex-col`)로 바꿔 위엔 입력 행, 아래(단계 2개+일 때만)엔 칩 행을 둔다.
- **입력창 포커스 유지가 핵심**: 칩 버튼은 `onMouseDown`에서 `preventDefault()`로 기본 포커스 이동을 막고 `setStageId`만 한다. 이렇게 하면 칩을 눌러도 입력창이 blur(=취소)되지 않아, 고른 뒤 바로 이어서 타이핑·Enter할 수 있다.
- 선택 강조: 선택된 칩은 단계 색으로 테두리·배경 틴트(`hexToRgba`)와 글자색(`shade`)을 준다. 안 뜬 칩은 회색 텍스트 + 색 점만.

## 알려진 이슈 / 한계

- 칩은 팝오버 폭(최대 3칸) 안에서 줄바꿈된다. 겹친 단계가 아주 많고 이름이 길면 여러 줄이 될 수 있다(드문 경우).
- 팝오버의 빈 여백(칩 사이 gap·패딩)을 누르면 입력창이 blur되어 닫힌다 — 칩·입력 위 클릭은 정상 동작하므로 실사용엔 영향이 작다.
- #3(프로젝트 없음 → "단계 없음")은 `미배정-영역-제거` 머지 후 별도 사이클.
