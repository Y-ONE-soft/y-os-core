## 1. 작업 내용 요약

사용자 피드백("페이지가 아니라 지라처럼 하단 가로 스크롤로 움직여야 한다")에 따라 로드맵의 **탐색 모델을 근본적으로 바꿨다.**

| | 이전 (docs/25·28) | 이번 |
|---|---|---|
| 범위(일·주·개월·분기) | 화면 폭에 **딱 맞추는 창** | **확대 배율**(하루당 픽셀) |
| 타임라인 폭 | 항상 화면 폭 (창 밖은 안 보임) | 오늘 앞뒤로 길게 펼쳐짐 (2,881~5,352px) |
| 이동 방식 | ‹ › 페이지 버튼으로 창 점프 | **하단 가로 스크롤**로 연속 이동 |
| 오늘 | 페이지를 0으로 리셋 | **오늘 위치로 스크롤** (부드럽게) |

## 2. 주요 구현 결정

| 항목 | 결정 | 이유 |
|---|---|---|
| 범위 = 배율 | `dayWidth` 픽셀로 정의 (일 56px, 주 20px, 개월 6px, 분기 2.4px) | 지라 로드맵과 동일한 모델. 확대할수록 같은 기간이 넓게 펼쳐짐 |
| 타임라인 범위 | 배율별로 오늘 기준 과거·미래를 넉넉히 (일 −1/+2개월, 주 −2/+4, 개월 −6/+12, 분기 −12/+24개월) | 스크롤로 탐색할 여지를 확보. 모든 배율에서 화면보다 넓어 항상 스크롤 가능 |
| 위치 계산 | % → **px 기반** (`offsetDays × dayWidth`) | 스크롤 컨테이너와 픽셀 단위가 일치해야 눈금·막대가 어긋나지 않음 |
| 페이지 버튼 | **제거** | 스크롤이 그 역할을 대체. 버튼과 스크롤이 공존하면 위치 개념이 이중화됨 |
| 초기 스크롤 | ref 콜백에서 오늘 위치로 설정(왼쪽 120px 여백) | effect 없이 처리 → `set-state-in-effect` 린트 회피, 첫 페인트부터 올바른 위치 |
| 눈금 시작 정렬 | 주=월요일, 개월=1일, 분기=분기 첫날로 잘라냄 | 배율 전환 시 눈금이 어긋나지 않게 |
| 연도 경계 | 1월·Q1 눈금은 굵은 세로선 + 연도 재표시 | 긴 타임라인에서 연도 전환 지점을 즉시 인지 |
| 라벨 열 | `sticky left-0` 유지 + 우측 경계선 추가 | 멀리 스크롤해도 어떤 프로젝트/단계인지 유지 |
| 타임라인 헤더 | `sticky top-0` 추가 | 세로로 긴 목록에서도 날짜 눈금 유지 |

## 5. 작업 중 발생한 프로덕션 장애 (별건, 조치 완료)

검증 도중 **프로덕션 로그인이 500**으로 실패하는 것을 발견했다. 원인은 이 작업과 무관:

- 병렬 세션(`스탭-프로젝트-생성`)이 자기 브랜치에서 `prisma migrate dev`를 실행 → **공유 Neon DB에 `ALTER TABLE "User" DROP COLUMN "group"` 적용**
- 프로덕션도 같은 DB를 쓰는데 main 코드는 여전히 `User.group`을 읽음 → `P2022 ColumnNotFound` → 로그인 전면 실패
- **핫픽스**: 사용자 승인 하에 `ALTER TABLE "User" ADD COLUMN IF NOT EXISTS "group" TEXT` + 시드 계정 소속값 복구 SQL 실행 → **프로덕션 로그인 200 확인**. nullable 컬럼이라 해당 세션 작업에 영향 없고, 그 PR이 머지되면 정식 마이그레이션이 다시 정리한다.

### 후속 권고 (중요)

- **개발/프로덕션 DB 분리 필요.** 지금은 한 Neon DB를 공유해 개발 중 스키마 변경이 즉시 실서비스를 깨뜨린다. Neon 브랜치로 개발 DB를 분리하면 구조적으로 차단된다 — 별도 태스크 권장.
- CLAUDE.md의 "DB 스키마 변경은 동시에 한 브랜치만" 규칙(docs/22)이 이번 사고로 실증됐다. 규칙에 **"공유 DB = 프로덕션 DB이므로 마이그레이션은 사용자 승인 후 실행"**을 덧붙일지 검토 필요.

## 6. 알려진 사항 / 후속 과제

- 프로젝트 상세 페이지 로드맵은 아직 고정 창 + 자리표시 스위처 — 같은 모듈로 전환 필요
- 막대 드래그로 기간 조절 미구현
- 세로 스크롤은 페이지 전체에 의존 (타임라인 헤더는 sticky 처리됨)
