# fix: 단계 편입·생성 시 과거 시작이면 예정일·마감일을 오늘로

## 배경 — 직전 #143의 과교정

바로 앞 PR(#143)에서 단계 편입·생성 시 예정일을 "단계 시작일 그대로(과거여도)"로 잡았다. 그런데 과거에 시작한 단계에 할일을 **방금** 넣으면, 예정일이 과거 시작일로 잡히고 마감일도 그 과거 날짜가 되어 — 넣자마자 "N일 미뤄짐"으로 표시됐다. 방금 만든 할일이 이미 지연으로 뜨는 건 어색하다.

## 규칙 (사용자 협의)

마감일은 완료된 날이 아니라 **예정일의 원래 날짜**이고, 예정일=마감일이 같은 순간에는 미뤄짐이 없다. 방금 단계에 넣은 할일은 예정일=마감일이어야 하므로, **과거 시작 단계면 둘 다 오늘**로 잡아야 한다. 이후 하지 않고 하루가 지나야 예정일만 밀리며 미뤄짐이 드러난다.

## 수정

`roadmap-utils.ts`의 `scheduleFor`를 "시작일과 오늘 중 **더 늦은 날**"로 되돌렸다.

```diff
- return stageStartDate ?? todayISO();
+ const today = todayISO();
+ return stageStartDate && stageStartDate > today ? stageStartDate : today;
```

이 값이 예정일이자 마감일(#143에서 `deadline = scheduledDate`)이므로, 결과는:

| 단계 | 예정일 | 마감일 | 미뤄짐 |
| --- | --- | --- | --- |
| 미래 시작 | 단계 시작일 | 단계 시작일 | 없음 |
| **과거 시작** | **오늘** | **오늘** | **없음** (방금 넣었으니) |
| 캘린더 드래그 | 놓은 날 | 놓은 날 | 없음 |

`scheduleFor` 한 줄만 바뀌고, #143에서 만든 나머지(`addTask` 예정일 세팅, `createTask`의 `scheduledDate`/`deadline`, `assignTask` 낙관 반영)는 그대로 이 값을 쓴다.

## 검증

dev(`localhost:3034`)에서 실제 API로 확인. **7건 전부 통과.**

| 항목 | 결과 |
| --- | --- |
| 미래 단계 생성 → 예정일=마감일=단계 시작일(`07-28`) | ✅ |
| **과거 단계 생성** → 예정일=오늘(`07-23`), 시작일(`07-13`)로 안 잡음 | ✅ |
| 과거 단계 생성 → 마감일=오늘 | ✅ |
| **과거 단계 생성 → 안 미뤄짐 (예정=마감)** | ✅ |
| 백로그 → 과거 단계 → 예정=마감=오늘, 안 미뤄짐 | ✅ |
| 캘린더 드래그 → 놓은 날짜(`08-02`) | ✅ |

핵심은 과거 단계 경로 — 예정일·마감일이 모두 오늘이라 방금 넣은 할일이 미뤄짐으로 뜨지 않는다. 검증 데이터 삭제 후 잔여 0. lint(경고 0)·tsc·build 통과.
