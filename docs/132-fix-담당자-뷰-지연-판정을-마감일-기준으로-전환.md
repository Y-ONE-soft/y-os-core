# fix: 담당자 뷰 지연 판정을 마감일 기준으로 전환

## 배경 — 자동 이월이 만든 회귀

태스크 1에서 미완료 할일의 `scheduledDate`를 매일 오늘로 이월하도록 만들었다. 그런데 담당자 뷰(`task-status-assignees`)는 지연을 이렇게 판정하고 있었다.

```ts
overdue: !task.done && !!task.scheduledDate && task.scheduledDate < today
```

`scheduledDate`가 오늘로 이월되면 `scheduledDate < today`는 **항상 거짓**이 된다. 즉 자동 이월을 켜는 순간 담당자 뷰의 "지연 N건"이 전부 0이 되는 회귀다. 태스크 3 문서를 쓰다 발견해 이번 사이클에서 고친다.

## 변경 파일 내역

### `src/components/features/projects/task-status-assignees.tsx`

**판정을 마감일 기준으로.**

```diff
- overdue: !task.done && !!task.scheduledDate && task.scheduledDate < today,
+ overdue: !task.done && !!task.deadline && task.deadline < today,
```

`deadline`은 이월돼도 고정이라, "마감일이 지난 미완료"를 이월과 무관하게 잡는다. 이게 원래 "지연"의 의미이기도 하다.

**카드 툴팁도 마감일로.**

```diff
- title={card.overdue ? `예정일 ${card.task.scheduledDate} — 지났습니다` : undefined}
+ title={card.overdue ? `마감 ${card.task.deadline} — 지났습니다` : undefined}
```

`scheduledDate`를 그대로 두면 "예정일 2026-07-23(오늘) — 지났습니다"처럼 오늘 날짜가 "지났다"고 뜨는 모순이 생긴다. 마감일을 보여줘야 말이 맞는다.

dev 서버(`localhost:3025`)에서 마스터로 로그인해 작업 현황 → 담당자 뷰를 확인했다. **5건 전부 통과.**

| 항목 | 결과 |
| --- | --- |
| 검증 프로젝트/할일 생성 | ✅ |
| 예정일이 오늘로 이월됨 | ✅ `2026-07-23` |
| 마감일은 7일 전으로 고정 | ✅ `2026-07-16` |
| 지연 카드 툴팁이 마감일 기준("마감 2026-07-16 — 지났습니다") | ✅ |
| 이월된 예정일을 "지났다"고 잘못 표시하지 않음 | ✅ |

화면에서도 지연이 있는 담당자 컬럼이 붉은 카운트로 표시되는 것을 확인했다(마감 기준 판정이 기존 데이터의 지연도 다시 잡는다). 검증 데이터는 삭제 후 잔여 0건.

## 알려진 이슈 / 후속

- 담당자 뷰의 "지연"과 캘린더 칩의 "미뤄짐"이 이제 **같은 기준**(`deadline`)을 쓴다. 다만 판정 로직이 두 곳에 각각 있다(`task-status-assignees`의 인라인, `my-work-calendar-source`의 `late`). 공통 헬퍼로 뽑을 수 있으나 각기 다른 데이터 형태를 다뤄 이번엔 인라인으로 뒀다.
- docs 번호는 이 브랜치의 네 문서를 129~132로 재조정했다(125·126 선점).
