## 1. 작업 내용 요약

내 작업 캘린더에서 사용자가 지적한 3가지를 고쳤다. 셋 다 **한 프로젝트/한 단계가 여러 개로 쪼개져 보이는** 문제다.

| # | 문제 | 변경 |
|---|---|---|
| ① | 할일이 늘어날수록 프로젝트 박스가 가로선으로 나뉘어 **여러 프로젝트처럼 보임** | 박스 내부 줄 구분선 제거 — 박스는 한 덩어리 |
| ② | 주 경계를 넘는 단계가 **주마다 이름·개수 배지를 반복** 표시 | 이름·배지는 **시작 주에만**. 막대 자체는 기간만큼 계속 이어짐 |
| ③ | 단계에 호버하면 **커서가 놓인 조각만** 강조 | 같은 단계의 **모든 주 조각이 함께** 강조 |

②는 "프로젝트 이름은 범위가 시작하는 주에만 표시"라는 기존 규칙과 동일한 처리다.

## 2. 구현 메모

### ③ CSS `:hover`로는 안 된다

단계 막대는 주 경계에서 잘려 **조각마다 별개 DOM**이다. `hover:ring-1`은 커서가 놓인 조각에만 걸린다. 그래서 캘린더 루트에 `hoveredStageId` 상태를 두고 조각들이 `stageId`로 자신을 대조하도록 바꿨다.

- `onPointerEnter/Leave`로 상태를 올리고, `onFocus/Blur`도 함께 물려 **키보드 이동에서도 같은 강조**가 걸린다
- 강조는 테두리(ring)뿐 아니라 **배경 농도도 0.18 → 0.3**으로 올려, 떨어진 조각들이 한 덩어리로 읽히게 했다

### 변경하지 않은 것

- 레이아웃 계산(`my-work-calendar-layout.ts`)은 **손대지 않았다**. 박스 높이·레인 배치 규칙은 그대로고, 시각 표현만 바꿨다
- `my-work-calendar-source.ts`도 미변경 — `startsHere` 플래그가 이미 있어 그대로 활용했다. (이 파일은 병렬 세션 PR #68이 수정 중이라 의도적으로 피했다)

### 검증 방법론 메모

처음엔 `dispatchEvent(new PointerEvent("pointerenter"))`로 호버를 흉내 냈는데 **아무 반응이 없어 구현 실패로 보였다.** 원인은 React가 `onPointerEnter`를 위임된 `pointerover`로 처리하는데 `pointerenter`는 **버블링되지 않아** 루트 리스너에 닿지 않는 것. `page.mouse.move()`로 실제 커서를 옮기니 정상 동작했다. **위임 기반 이벤트(enter/leave 계열)는 합성 이벤트가 아니라 실제 포인터 이동으로 검증해야 한다.**

## 5. 알려진 사항 / 후속 과제

- 단계 막대의 "마감" 배지는 종료 조각에 그대로 둔다(시작 주에만 다는 이름과 별개 — 마감은 끝나는 위치에서 보여야 의미가 있다)
- 이름이 빠진 뒷 조각도 `title` 속성은 유지되어 마우스를 올리면 툴팁으로 단계명을 확인할 수 있다
- 프로젝트 박스가 최소 2줄(`MIN_BOX_LANES`)을 유지하는 규칙은 그대로 — 이번 요청 범위 밖
