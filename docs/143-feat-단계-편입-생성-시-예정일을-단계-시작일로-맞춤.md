# feat: 단계 편입·생성 시 예정일을 단계 시작일로 맞춤

## 작업 요약

할일이 단계에 속하게 될 때(단계 보드에서 생성 / 백로그를 단계로 이동) 예정일을 **그 단계의 시작일**로 잡는다. 캘린더 날짜 칸에 드래그해 놓은 경우는 그 놓은 날짜를 쓴다.

## 확정된 규칙 (사용자 협의)

- 기준 날짜 = **그 할일이 들어가는 단계의 시작일** (3단계에 넣으면 3단계 시작일).
- 시작일이 이미 지난 **과거여도 그대로** 쓴다 — 미완료면 방금 붙은 자동 이월로 예정일이 오늘로 당겨지고, 마감일은 그 시작일로 남아 상세에 "N일 미뤄짐"이 표시된다(마감일 기능과 일관).
- **캘린더 드래그**는 예외 — 사용자가 명시적으로 놓은 날짜를 쓴다.

## 경로별 현재 → 목표

| 경로 | 현재 | 목표 |
| --- | --- | --- |
| 단계 보드에서 생성 (`addTask`) | 예정일 **안 잡음** | 단계 시작일 |
| 백로그 → 단계 (`assignTask`) | `scheduleFor` = 시작일·오늘 중 늦은 날 | 단계 시작일 그대로 |
| 캘린더 드래그 | 놓은 날짜 | 그대로 |

## 변경 파일 내역

### `roadmap-utils.ts` — `scheduleFor`

"단계 시작일과 오늘 중 더 늦은 날" → **단계 시작일 그대로**.

```diff
- return stageStartDate && stageStartDate > today ? stageStartDate : today;
+ return stageStartDate ?? todayISO();
```

`scheduleFor`는 `board-store`의 `addTask`·`assignTask` 두 곳에서만 쓰이므로(확인함), 이 한 줄로 두 경로가 같은 규칙을 갖는다. 단계에 시작일이 없으면(진행형·미정) 잡을 근거가 없어 오늘로 둔다.

### `board-store.tsx`

- **`addTask`(단계 생성)**: 지금은 예정일을 안 잡았다. 그 단계의 시작일로 `scheduleFor`를 계산해 낙관적 캐시와 `createTaskApi` 양쪽에 넘긴다. `deadline`도 같은 값으로 낙관 반영.
- **`assignTask`(편입)**: `scheduleFor` 수정으로 자동 반영. 낙관적 `moved`에 `deadline`도 함께 넣어, 편입 직후 상세에서 마감·미뤄짐이 바로 맞게 보이도록 했다(서버 `withDeadline`과 같은 규칙).

### `service.ts` `createTask` + 라우트 + `createTaskApi`

`createTask`가 `scheduledDate`(선택)를 받는다. 값이 있으면 `deadline`도 같은 값으로 세팅한다 — `updateTask`의 `withDeadline`과 같은 규칙이라, 단계에서 생성된 할일도 이월/미뤄짐과 자연히 맞물린다.

```ts
const data = scheduledDate
  ? { ...rest, scheduledDate, deadline: scheduledDate }
  : rest;
```

라우트(`POST /api/admin/tasks`)는 `isISODate`로 거른 `scheduledDate`만 넘긴다. `deadline`은 클라이언트가 못 보내고 서버가 파생한다(마감일 기능의 위조 차단 규칙 유지).

## 검증

dev(`localhost:3033`)에서 세 경로와 과거 이월을 실제 API로 확인했다. **10개 전부 통과.**

| 경로 | 결과 |
| --- | --- |
| **단계 보드 생성** → 예정일 = 단계 시작일 (`07-28`) | ✅ |
| 단계 생성 → 마감일도 단계 시작일 | ✅ |
| 백로그 할일은 예정일 없음 (편입 전) | ✅ |
| **백로그 → 단계** → 예정일 = 그 단계 시작일 (`08-07`) | ✅ |
| 백로그 → 단계 → 마감일도 그 단계 시작일 | ✅ |
| **캘린더 드래그** → 놓은 날짜(`08-02`, 단계 시작일 아님) | ✅ |
| **과거 시작 단계** → 마감일 = 과거 시작일 (`07-13`) 그대로 | ✅ |
| 과거 → 예정일은 조회 시 오늘(`07-23`)로 이월 | ✅ |
| → 마감 대비 "미뤄짐" 성립 | ✅ |

과거 시작 단계 경로가 핵심이다 — 예정일을 과거 시작일로 잡으면 조회 시 오늘로 이월되고 마감일은 과거로 남아, 방금 붙인 이월/미뤄짐 기능과 정확히 맞물린다. 검증 데이터는 삭제 후 잔여 0.

- `npm run lint` 통과 (경고 0)
- `npx tsc --noEmit` 통과
- `npm run build` 통과

## 알려진 이슈 / 후속

- 미배정 할일(프로젝트 없이 내 할일에서 만든 것)을 캘린더에 드롭하면 그 날짜로 예정일이 잡히는데, 이건 드래그 경로라 그대로다(단계 개념 없음).
- 검증 중 `even-stages` API가 다른 세션(PR #128)에서 `spans` 방식으로 바뀐 것을 발견해 테스트를 맞췄다. 이 기능과는 무관하다.
- docs 번호는 작성 시점 main 최대(142) + 1로 143을 썼다.
